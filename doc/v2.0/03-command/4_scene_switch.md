# 场景切换机制

> **所属模块**：指挥模式
> **使用端**：PC 大屏
> **前置阅读**：`1_common.md`、`2_merge_analysis.md`

## 一、场景定义

```typescript
type CommandScene =
  | 'congestion'   // 日常拥堵场景
  | 'emergency';   // 应急场景
```

## 二、场景切换触发条件

### 进入日常拥堵场景
- 拥堵指数 > 4 且无停航/台风预警
- 从总览模式手动切换

### 进入应急场景
- 港口停航通知
- 台风橙色/红色预警
- 手动切换（导航栏点击"应急"按钮，但隐藏后用其他方式触发）
- 应急预案启动

### 应急场景退出
- 港口已复航 + 滞留 < 200
- 拥堵指数 < 4 且无台风预警
- 手动切换

## 三、场景切换的影响

### 摘要条（CommandSummaryBar）

| 元素 | 日常拥堵 | 应急场景 |
|------|---------|---------|
| 背景色 | 橙色渐变 | 红色渐变 |
| 主文字 | "进港大道拥堵指数 5.2" | "台风预警 + 港口停航" |
| 副文字 | "持续 12 分钟，趋势加剧" | "预计滞留 3,200 辆，应急 II 级" |
| 操作按钮 | "执行策略" | "启动预案" |

### 策略面板（StrategyCommandPanel）

| 元素 | 日常拥堵 | 应急场景 |
|------|---------|---------|
| 顶部入口 | 隐藏 | 显示"启动预案"按钮 |
| 策略列表 | AI 推荐策略 | AI 推荐策略 + 预案任务 |
| 推荐顺序 | 按拥堵原因 | 按预案阶段 |

### 执行资源面板（ExecutionResourcePanel）

| 元素 | 日常拥堵 | 应急场景 |
|------|---------|---------|
| 人员资源 | 显示 | 显示 |
| 物资资源 | 简化（拖车/警力/锥桶） | 完整（增加冷链车物资、应急电源） |
| 特殊车辆卡片 | 隐藏 | 显示（折叠卡片） |
| 物资需求卡片 | 隐藏 | 显示（折叠卡片） |
| 任务板 | 隐藏 | 显示（整合到底部） |

### 地图（CommandMap）

| 图层 | 日常拥堵 | 应急场景 |
|------|---------|---------|
| 拥堵热力 | 显示 | 显示 |
| 分流路线 | 显示（执行策略时） | 显示 |
| 视频监控点 | 显示 | 显示 |
| 临时停车区 | 隐藏 | 显示 P-1/P-2/P-3 |
| 无人机巡查路线 | 隐藏 | 显示 |
| 物资发放点 | 隐藏 | 显示 |
| 警力部署点 | 显示 | 增加部署密度 |

### 通信面板（CommandCommPanel）

| 元素 | 日常拥堵 | 应急场景 |
|------|---------|---------|
| 默认频道 | 全部 | 应急频道 |
| @提及部门 | 公安交警 | 公安交警/民政局/交通局/港口/城管/应急 |
| 快捷指令 | 5 条 | 增加应急专用 6 条 |
| 发起会商按钮 | 隐藏 | 显示 |

## 四、状态管理设计

### Store 字段

```typescript
interface CommandState {
  scene: CommandScene;

  // 场景切换原因（便于追溯）
  sceneChangedReason: string;
  sceneChangedAt: number;

  // 应急专属字段
  specialVehicles: SpecialVehicle[];
  supplyDemand: SupplyDemandItem[];
  emergencyTasks: EmergencyTask[];
  activePlan: ActivePlan | null;
}
```

### Actions

```typescript
interface CommandActions {
  // 切换场景
  setScene: (scene: CommandScene, reason: string) => void;

  // 启动预案（自动切换到应急场景）
  activatePlan: (planId: string) => void;

  // 退出应急（切换回日常）
  exitEmergency: (reason: string) => void;
}
```

## 五、场景切换的视觉反馈

### 切换动画
- 切换时摘要条颜色渐变（300ms）
- 新增面板从右侧滑入（300ms）
- 移除面板淡出（200ms）

### 用户提示
- 进入应急场景时：弹出 toast "已进入应急场景，启用预案功能"
- 退出应急场景时：弹出 toast "应急状态已解除，返回日常指挥"

### 不需要二次确认
- 自动切换不需要用户确认（避免漏掉关键时机）
- 手动切换需要二次确认（避免误操作）

## 六、关键时序

### 应急场景启动时序
```
1. 检测到触发条件（停航/预警）
   ↓
2. setScene('emergency', '港口停航')
   ↓
3. 摘要条样式切换为红色
   ↓
4. 执行资源面板展开应急卡片
   ↓
5. 地图叠加应急图层
   ↓
6. 通信面板切换到应急频道
   ↓
7. 弹出 toast 提示
   ↓
8. 触发虚拟人播报（如果总览有虚拟人）
```

### 应急场景退出时序
```
1. 检测到退出条件（复航 + 滞留低）
   ↓
2. 提示"应急状态已解除，是否退出？"
   ↓
3. 用户确认 → setScene('congestion', '应急已解除')
   ↓
4. 应急专属面板淡出
   ↓
5. 摘要条样式切换为橙色（如果还有拥堵）或绿色（如果已正常）
   ↓
6. 地图移除应急图层
```

## 七、边界情况处理

### 同时满足拥堵和应急条件
- 优先进入应急场景
- 拥堵态势作为应急场景的次要信息显示

### 应急条件解除但拥堵未解决
- 切换到日常拥堵场景
- 保留特殊车辆等数据（隐藏面板，不清空数据）

### 用户在应急场景手动切换到总览
- 提示"当前处于应急场景，确认切换？"
- 切换不影响应急 store 数据

## 八、验收标准

- [ ] 港口停航通知触发应急场景
- [ ] 台风橙色预警触发应急场景
- [ ] 拥堵指数 > 4 触发日常拥堵场景
- [ ] 摘要条根据场景显示不同样式
- [ ] 应急专属面板在应急场景下显示
- [ ] 日常场景下不显示应急专属面板
- [ ] 切换有平滑动画
- [ ] 切换时弹出 toast 提示
- [ ] 手动切换需要二次确认
- [ ] 自动切换不需要确认
