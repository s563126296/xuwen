# 徐闻智慧交通大屏 v2.0 — 需求分析（2026-04-27）

> 基于产品讨论整理，覆盖总览/指挥/AI分析三个模式的 v2.0 需求

---

## 一、整体 AI 价值定位

v2.0 的核心目标不是"加功能"，而是让用户感受到：

> **AI 在主动工作、AI 的判断靠谱、AI 在持续进化**

三个模式各承担不同的 AI 角色：

| 模式 | AI 角色 | 核心价值 |
|------|---------|---------|
| 总览模式 | AI 值班助手 | 主动感知、主动播报、随时可问 |
| 指挥模式 | AI 参谋长 | 给策略、盯执行、发现偏差、主动纠偏 |
| AI 分析模式 | AI 策略实验室 | 推演未来、复盘过去、展示进化 |

---

## 二、总览模式 v2.0

### 2.1 虚拟人"阿琼"播报增强

**播报场景：**

| 场景 | 触发条件 | 播报内容 |
|------|---------|---------|
| 每日启动 | 当天首次打开系统 | 昨日整体情况 + 今日态势预判 + 重点关注项 |
| 每周一 | 周一首次打开 | 上周整体情况 + 本周预计压力 |
| 异常插播 | 关键指标突变 | 异常内容 + 原因 + 建议 |
| 整点播报 | 每小时 | 关键指标摘要 |

**播报控制：**
- 优先级：异常 > 每日 > 整点
- 频率控制：同一事件 10 分钟内不重复
- 可静音、可点击停止

### 2.2 虚拟人对话能力

**定位：** 徐闻交通专家，形象亲切、专业

**能力边界：**

| 能回答 | 不能回答 | 不确定时 |
|--------|---------|---------|
| 大屏所有数据 | 政策解读 | "这个信息我暂时没有，建议联系 XX 确认" |
| 历史事件和策略 | 人事安排 | |
| 徐闻路网知识 | 预算审批 | |
| 交通规则常识 | 非交通领域 | |
| 联网搜索相关信息 | | |

**原则：** 不编造信息，不确定就说不确定

**技术路线：**
- 第一版：规则匹配（30+ 条 QA）
- 第二版：接入 LLM（Claude/讯飞星火）+ RAG 知识库
- 语音唤醒（"小闻小闻"）：P2，需评估浏览器兼容性

### 2.3 AI 预测指标突出

**视觉层面：**
- AI 预测指标加紫色 AI 图标
- 预测数值用脉冲动画
- hover 显示"AI 预测"提示

**内容层面：**
- "有 AI vs 无 AI"对比：如"AI 预测 1h 后 5.2，不干预预计 7.5"
- 历史战绩："过去 30 天准确率 84%"
- 预测失败时主动说明："上次偏差较大，本次已调整"

**交互层面：**
- 点击预测值查看依据
- 显示预测时间范围和置信度

### 2.4 AI 摘要面板重设计

**按角色分层：**
- 第一层（折叠态）：给领导看的一句话结论
- 第二层（展开态）：给值班员看的异常清单 + 影响因子
- 第三层（点击详情）：给指挥员看的归因分析 + 策略建议

**AI 感强化：**
- 背景改为紫色渐变
- 显示"已分析 12 个数据源"
- 显示"基于 126 个历史场景学习"
- 展开时有"AI 思考"动画

## 三、指挥模式 v2.0

### 3.1 应急合并（已确认方案）

- 保留：特殊车辆追踪、物资需求、预案快捷入口
- 移除：5 阶段状态机、滞留预测、台风路径
- 场景感知：congestion / emergency 动态切换
- 弱化应急场景复杂度

### 3.2 策略执行闭环监控（核心功能）

**完整流程：**

```
策略推荐 → 预期效果 → 执行监控 → 偏差检测 → 主动询问 → 决策建议 → 更新预期 → 用户反馈 → AI 学习
```

**阶段 1：策略推荐 + 预期效果**
- AI 推荐策略（已有）
- 新增：明确的预期效果时间线
  - 1h 后拥堵指数 6.8 → 5.2
  - 2h 后拥堵指数 6.8 → 4.3
  - 预计分流 500 辆/小时

**阶段 2：执行监控**
- 实时/定时收集关键数据
- 预期曲线（虚线）vs 实际曲线（实线）对比图
- 偏差区域红色填充

**阶段 3：偏差检测**
- 偏差超过 15% 触发预警
- 自动分析偏差原因（数据不足时进入询问）

**阶段 4：主动询问**
- 两个通道：
  - 大屏弹窗：问指挥员
  - 通信面板推送：问现场人员
- 提供快捷回答选项（S376 事故 / 路段畅通 / 新增车辆 / 其他）

**阶段 5：决策建议 + 更新预期**
- 根据反馈生成新建议
- 更新预期效果曲线
- 保留调整版本记录：
  ```
  v1 原始预期：1h → 5.2
  v2 调整：1h → 5.8（原因：S376 事故，分流效率降低）
  v3 调整：1h → 5.5（原因：启动 S207 备用分流）
  ```

**阶段 6：用户反馈**
- 策略执行完毕后弹出评价
- 👍 建议有效 / 👎 建议无效 / 💬 补充说明
- 反馈用于调整策略推荐权重

**需要覆盖的场景：**

| 场景 | AI 行为 |
|------|---------|
| 效果达预期 | 提示"策略生效，建议继续执行" |
| 效果超预期 | 提示"效果优于预期，建议提前结束，减少对分流路段影响" |
| 效果不达预期 | 偏差检测 → 主动询问 → 决策建议 |
| 多策略叠加 | 归因拆分，区分各策略贡献度 |
| 外部因素干扰 | 区分"策略没效果"和"外部因素干扰" |

### 3.3 AI 策略自我进化

**进化触发条件：**
- 策略执行偏差 > 15%
- 用户反馈"建议无效"
- 发现新的影响因子

**进化内容：**
- 补充之前未考虑的因子（天气、车型、路况、汇入车流等）
- 调整因子权重
- 更新预测模型参数

**进化记录：**
```
v1.0 基础模型（6 个因子：港口积压/流量/时段/道路容量/信号灯/历史数据）
v1.1 +天气因子（大雾/雨天分流效率降低 30%）
     ← 来源：4-22 S376 分流偏差，大雾天实际分流 350 辆 vs 预期 500 辆
v1.2 +车型因子（大货车占比 >40% 时分流速度降低 25%）
     ← 来源：4-25 进港大道分流偏差，大货车占比 55%
v1.3 +路况因子（分流道路拥堵时自动降低预期）
     ← 来源：4-26 S376 事故反馈，分流道路本身拥堵
v1.4 +汇入车流因子（其他路口汇入量 >200 辆/h 时降低分流效果预期）
     ← 来源：4-27 用户反馈，X699 县道汇入车流未纳入计算
```

**进化展示位置：** AI 分析模式（策略进化记录面板）

| 策略推荐引擎不支持动态因子 | 先看效果再决定是否重构 |
| 没有"有 AI vs 无 AI"对比数据 | aiSummaryEngine 同时生成干预/不干预预测 |
