# 徐闻智慧交通大屏 v2.0 — 需求分析（2026-04-27）

> 基于产品讨论整理，覆盖总览/指挥/AI分析三个模式的 v2.0 需求
>
> **最后更新：2026-04-29** — 根据实际开发完成情况同步更新各功能状态

---

## 一、整体 AI 价值定位

v2.0 的核心目标不是"加功能"，而是让用户感受到：

> **AI 在主动工作、AI 的判断靠谱、AI 在持续进化**

三个模式各承担不同的 AI 角色：

| 模式 | AI 角色 | 核心价值 |
|------|---------|---------|
| 总览模式 | AI 值班助手 | 主动感知、主动播报、随时可问 |
| 指挥模式 | AI 参谋长 | 给策略、盯执行、发现偏差、主动纠偏 |
| AI 分析模式 | AI 策略实验室 | 推演未来、复盘过去、展示进化 |

## 二、总览模式 v2.0

### 2.1 虚拟人"小语"播报增强 ✅ 已完成

**播报场景：**

| 场景 | 触发条件 | 播报内容 |
|------|---------|---------|
| 每日启动 | 当天首次打开系统 | 昨日整体情况 + 今日态势预判 + 重点关注项 |
| 每周一 | 周一首次打开 | 上周整体情况 + 本周预计压力 |
| 异常插播 | 关键指标突变 | 异常内容 + 原因 + 建议 |
| 整点播报 | 每小时 | 关键指标摘要 |

**播报控制：**
- 优先级：异常 > 每日 > 整点
- 频率控制：同一事件 10 分钟内不重复
- 可静音、可点击停止

**实现说明：**
- 虚拟人名称从原讨论的"阿琼"改为"小语"（P0 开发阶段确认）
- 16:9 横向播报面板（288x162px），支持隐藏/收起
- 浮动动画 + TTS 播报逻辑

### 2.2 虚拟人对话能力 ❌ 未实现（P2）

**定位：** 徐闻交通专家，形象亲切、专业

**能力边界：**

| 能回答 | 不能回答 | 不确定时 |
|--------|---------|---------|
| 大屏所有数据 | 政策解读 | "这个信息我暂时没有，建议联系 XX 确认" |
| 历史事件和策略 | 人事安排 | |
| 徐闻路网知识 | 预算审批 | |
| 交通规则常识 | 非交通领域 | |
| 联网搜索相关信息 | | |

**原则：** 不编造信息，不确定就说不确定

**技术路线：**
- 第一版：规则匹配（30+ 条 QA）

**未实现原因：** 需接入 LLM 服务，当前为纯前端 Demo，列入 P2 待办。

## 三、指挥模式 v2.0

### 3.1 应急合并（已确认方案）✅ 已完成

- 保留：特殊车辆追踪、物资需求、预案快捷入口
- 移除：5 阶段状态机、滞留预测、台风路径
- 场景感知：congestion / emergency 动态切换
- 弱化应急场景复杂度

**实现说明：**
- commandStore 新增场景感知字段（congestion / emergency）
- CommandSummaryBar 支持红色应急场景样式
- EmergencyPanels 组件（特殊车辆 / 物资需求 / 任务板）
- 应急模式从 App.tsx 条件渲染中移除（代码保留）

### 3.2 策略执行闭环监控（核心功能）✅ 已完成

**完整流程：**

```
策略推荐 → 预期效果 → 执行监控 → 偏差检测 → 主动询问 → 决策建议 → 更新预期 → 用户反馈 → AI 学习
```

**阶段 1：策略推荐 + 预期效果**
- AI 推荐策略（已有）
- 新增：明确的预期效果时间线
  - 1h 后拥堵指数 6.8 → 5.2
  - 2h 后拥堵指数 6.8 → 4.3
  - 预计分流 500 辆/小时

**v2.0 增强（开发阶段新增）：**
- 策略推荐卡片增加推荐理由 + 历史依据
  - N 次执行记录 / 场景匹配度 / 历史成功率 / AI 置信度
- 预期效果 = 资源到位时间 + 策略生效时间（指挥员可手动修正）

**阶段 2：执行监控**
- 实时/定时收集关键数据
- 预期曲线（虚线）vs 实际曲线（实线）对比图
- 偏差区域红色填充

**阶段 3：偏差检测**
- 偏差超过 15% 触发预警
- 11 个因素归因引擎（v2.0 增强）：
  - 拥堵指数 / 道路容量 / 天气条件 / 时段特征 / 港口状态
  - 历史模式 / 资源到位 / 路况变化 / 车型分布 / 汇入车流 / 信号灯方案
- 偏差三分类：
  - 策略问题：策略本身不适用当前场景
  - 执行问题：资源未到位、执行延迟
  - 环境变化：天气突变、事故等外部因素

**阶段 4：主动询问**
- 两个通道：
  - 大屏弹窗：问指挥员
  - 通信面板推送：问现场人员
- 提供快捷回答选项（S376 事故 / 路段畅通 / 新增车辆 / 其他）

**阶段 5：决策建议 + 更新预期**
- 根据反馈生成新建议
- 更新预期效果曲线
- 策略调整两种处理（v2.0 增强）：
  - 补充性质（参数微调）→ 版本迭代，同一策略内更新
  - 追加性质（新增策略）→ 独立监控，各自跟踪效果
- 保留调整版本记录：
  ```
  v1 原始预期：1h → 5.2
  v2 调整：1h → 5.8（原因：S376 事故，分流效率降低）
  v3 调整：1h → 5.5（原因：启动 S207 备用分流）
  ```

**阶段 6：用户反馈**
- 策略执行完毕后弹出评价
- 满意度评分（1-5 星）+ 改进建议输入
- 反馈历史记录
- 反馈用于调整策略推荐权重

**需要覆盖的场景：**

| 场景 | AI 行为 |
|------|---------|
| 效果达预期 | 提示"策略生效，建议继续执行" |
| 效果超预期 | 提示"效果优于预期，建议提前结束，减少对分流路段影响" |
| 效果不达预期 | 偏差检测 → 主动询问 → 决策建议 |
| 多策略叠加 | 归因拆分，区分各策略贡献度 |
| 外部因素干扰 | 区分"策略没效果"和"外部因素干扰" |

**实现说明：**
- strategyMonitorEngine.ts 监控引擎（偏差检测 + 主动询问 + 决策建议）
- DeviationMonitorPanel 偏差监控面板（地图左下角独立浮层 550x320px）
- InquiryModal 主动询问弹窗（30 秒倒计时 + 快捷回复）
- StrategyFeedbackPanel 用户反馈面板（1-5 星评分 + 改进建议）
- commandStore 扩展 monitorState 字段（deviations / inquiries / feedbacks）

### 3.3 AI 策略自我进化 ✅ 已完成

**进化触发条件：**
- 策略执行偏差 > 15%
- 用户反馈"建议无效"
- 发现新的影响因子

**进化内容：**
- 补充之前未考虑的因子（天气、车型、路况、汇入车流等）
- 调整因子权重
- 更新预测模型参数

**进化记录：**
```
v1.0 基础模型（6 个因子：港口积压/流量/时段/道路容量/信号灯/历史数据）
v1.1 +天气因子（大雾/雨天分流效率降低 30%）
     ← 来源：4-22 S376 分流偏差，大雾天实际分流 350 辆 vs 预期 500 辆
v1.2 +车型因子（大货车占比 >40% 时分流速度降低 25%）
     ← 来源：4-25 进港大道分流偏差，大货车占比 55%
v1.3 +路况因子（分流道路拥堵时自动降低预期）
     ← 来源：4-26 S376 事故反馈，分流道路本身拥堵
v1.4 +汇入车流因子（其他路口汇入量 >200 辆/h 时降低分流效果预期）
     ← 来源：4-27 用户反馈，X699 县道汇入车流未纳入计算
```

**进化展示位置：** AI 分析模式（策略进化记录面板）

**实现说明：**
- evolutionStore 存储进化历史
- 自动进化记录（偏差触发 + 反馈触发）
- 反馈→学习数据写入流程已打通

### 3.4 策略-资源绑定 ✅ 已完成（第 1 批新增）

**功能：** 每个策略关联具体的执行资源，提供资源决策依据

**资源决策面板：**
- 策略所需资源清单（人员/车辆/设备）
- 资源当前状态（可用/占用/调配中）
- 到位时间模型（基于距离和当前路况计算）
- 指挥员可手动修正到位时间

**实现说明：**
- ResourceDecisionPanel 组件
- 资源到位时间 + 策略生效时间 = 预期效果时间

### 3.5 现场反馈 AI 提取 ✅ 已完成（第 1 批新增）

**功能：** 从现场人员的聊天消息中自动提取关键信息

**流程：**
```
现场人员发消息 → AI 自动分析 → 提取关键信息 → 生成确认卡片 → 指挥员确认 → 等待指令
```

**AI 提取内容：**
- 事故信息（位置、类型、严重程度）
- 路况变化（拥堵加剧/缓解、道路封闭）
- 资源状态（人员到位、设备故障）
- 异常情况（天气变化、群众聚集）

**关键原则：** AI 分析→确认→等指令，不越权自动执行

### 3.6 结构化复盘报告 ✅ 已完成（第 1 批新增）

**功能：** 事件处置完毕后自动生成结构化复盘报告

**报告 5 部分：**
1. 事件概况：时间/地点/类型/严重程度/持续时长
2. 策略版本迭代：每次调整的原因和效果
3. 偏差事件记录：所有偏差的触发/原因/处理
4. AI 学习成果：本次事件 AI 模型的进化内容
5. 综合评价：策略有效性/响应速度/资源利用率

## 十一、业务梳理新增决策（2026-04-28~29）

> 以下决策在业务逻辑重构设计阶段（2026-04-28）确认，指导了第 1~3 批开发。

### 用户角色模型

**双角色协同：**
- 领导（县长/分管领导）：看总览模式，关注一句话结论和风险等级
- 指挥员（值班长/交警队长）：操作指挥模式，执行策略和资源调度

**核心原则：** 大屏是决策现场，不是审批系统。领导看结果，指挥员做操作。

### AI 行为模式

**三步原则：分析 → 确认 → 等指令**
- AI 负责分析和建议，不自动执行任何操作
- 所有策略执行需指挥员确认
- 偏差检测后主动询问，但不自动调整策略
- 现场反馈 AI 提取后生成确认卡片，等指挥员确认

### 时间模型

**预期效果 = 资源到位时间 + 策略生效时间**
- 资源到位时间：基于距离和当前路况计算（指挥员可手动修正）
- 策略生效时间：基于历史数据和当前条件预测
- 两个时间分开展示，让指挥员理解效果延迟的原因

### 跨模式数据流

```
总览模式 ──分级触发──→ 指挥模式 ──复盘数据──→ AI 分析模式
    ^                                              │
    │                    反哺推荐                    │
    └──────────────────────────────────────────────┘
```

- 总览 → 指挥：预测性分级触发（低/高/紧急三级，30min 前瞻）
- 指挥 → AI 分析：策略执行数据、偏差记录、用户反馈写入进化历史
- AI 分析 → 指挥：进化后的模型反哺策略推荐准确率

### AI 形象呈现

- 总览模式：虚拟人"小语"形象出现（右下角浮动 + 播报气泡）
- 指挥模式：AI 以面板形式呈现（策略推荐卡片 + 偏差监控面板 + 询问弹窗）
- AI 分析模式：AI 以面板形式呈现（模拟器 + 决策树 + 知识图谱）
