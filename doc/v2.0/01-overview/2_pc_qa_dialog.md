# 虚拟人问答交互

> **所属模块**：总览模式
> **使用端**：PC 大屏
> **前置阅读**：`1_common.md`

## 目标

让用户可以直接向系统提问，降低使用门槛，增强交互性。

## 问答范围

### 1. 态势类
- 当前整体情况怎么样？
- 哪里最堵？
- 港口现在情况怎么样？

### 2. 数据类
- 现在进港车辆多少？
- 当前待渡车辆多少？
- 现在风力多少级？

### 3. 风险类
- 今天有停航风险吗？
- 未来两小时压力会不会升高？

### 4. 功能引导类
- 怎么切换到指挥模式？
- 为什么建议切换到指挥模式？

## 第一版范围控制

- 仅支持文字输入
- 规则匹配回答
- 支持 20-30 条常见问题
- 不支持开放式复杂追问

## 页面变化

点击虚拟人展开问答框：
- 历史消息区
- 输入框
- 快捷问题推荐
- 关闭按钮

## 交互流程

1. 用户点击虚拟人
2. 展开问答框
3. 用户输入问题或点击快捷问题
4. 系统匹配规则并生成回答
5. 回答引用当前实时数据
6. 如无法回答，给出兜底提示

## 数据需求

- 当前总览关键指标
- 风险判断结果
- 模式切换规则
- 问答规则库

## 技术建议

- 第一版采用规则匹配
- 问答规则配置独立成 `assistantQaRules.ts`
- 回答模板中引用 store 实时数据

## 优先级

- P1
