**1. 文档说明**

**1.1 文档目的**

本文档为徐闻县智慧交通大屏系统的完整产品需求规格说明，面向研发团队（前端、后端、测试）。文档涵盖系统功能模块说明、数据指标定义、交互逻辑、数据来源及接口说明等内容，作为开发实现与测试验收的核心依据。

**1.2 文档范围**

本文档描述徐闻县智慧交通大屏系统当前最新版本的完整功能，包含以下核心模块：

- 交通态势分析面板（左侧 + 右侧）

- 中央交通态势地图

**2. 产品概述**

**2.1 产品背景**

徐闻县是中国大陆最南端的县级行政区，毗邻琼州海峡，徐闻港与海安新港是粤琼两地重要的滚装船渡运枢纽，日均车辆吞吐量大、高峰期拥堵问题突出。徐闻县智慧交通大屏系统依托政务数字化建设，整合交通感知设备、高德地图、港口运营及气象数据，为交通管理部门提供统一的可视化决策支撑平台。

**2.2 产品目标**

- 实时掌握全县交通态势，重点关注进出港主要路段拥堵情况

- 通过数据可视化降低管理人员的信息获取门槛，提升决策效率

- 基于拥堵预测与堵点预警提前干预，减少港口积压

- 整合多来源数据（感知设备、高德、海康、气象）统一展示

**2.3 用户角色**

| **用户角色**   | **使用场景**           | **核心需求**           |
|----------------|------------------------|------------------------|
| 交通管理指挥员 | 值班室大屏实时监控     | 快速感知拥堵、及时调度 |
| 港口运营管理员 | 港口运力与车辆进出监控 | 掌握船舶运力、待舶队伍 |
| 交通执法人员   | 违规事件查看与处置     | 关注交通违法事件统计   |
| 领导决策层     | 周期性数据汇报与决策   | 宏观数据趋势与预测     |

**2.4 系统整体架构**

大屏系统整体采用"三栏式 + 全屏地图"布局：

- 左侧面板：交通态势分析（感知体系 + 徐闻出行 + 交通违法事件）

- 中央区域：GIS交通态势地图（全域路网 + 设备点位 + 拥堵热力 + 重点路段）

- 右侧面板：港口交通拥堵分析（拥堵情况 + 港口车辆 + 港口运力 + 拥堵预测 + 道路承载）

- 顶部导航：系统标题、港口切换（徐闻港 / 海安新港）、天气信息、拥堵指数

- 弹窗层：卡口详情弹窗、重点路段弹窗、优化调度策略弹窗、预警响应弹窗

**3. 功能需求详述**

**3.1 顶部导航栏**

**3.1.1 功能概述**

顶部导航栏常驻显示，包含系统标题、实时时间、港口切换、实时天气、全局拥堵指数及调度策略入口。

**3.1.2 系统标题与时间**

- 标题文字：「徐闻县智慧交通」，居中展示

- 左上角展示当前系统实时时间，格式：YYYY.MM.DD HH:mm:ss，每秒刷新

**3.1.3 港口切换**

- 提供「徐闻港」和「海安新港」两个切换入口，位于右上角

- 两个港口功能完全相同，切换后右侧面板及地图数据对应切换至所选港口数据

- 当前激活港口高亮显示，另一港口为未激活状态

**3.1.4 实时天气**

天气信息展示于顶部中央区域，包含以下字段：

| **字段** | **展示规则** | **数据来源** |
|----|----|----|
| 天气图标 | 根据天气现象映射对应图标（晴/多云/阴/小雨/中雨/大雨/暴雨/雷阵雨/雾/大风/小雪/中雪/大雪/台风） | 天气接口 |
| 降雨量 | 晴天/多云只显示图标不显示数值；雨天显示"Xmm"（X取整） | 天气接口 |
| 风速 | 格式："Ym/s"，Y保留一位小数 | 天气接口 |
| 天气预警 | 有预警时显示预警标签（如"暴雨预警"），无预警时不显示 | 天气接口 |

**3.1.5 全局拥堵指数与调度策略**

- 当进港或出港方向拥堵指数 \> 2 时，顶部显示拥堵指数数值与拥堵状态标签

- 同时显示「调度策略」图标按钮，点击后弹出交通拥堵调度策略弹窗（详见 3.7）

- 拥堵指数 ≤ 2 时，不显示拥堵指数及调度策略入口

**3.2 左侧面板 — 交通态势分析**

**3.2.1 感知体系**

展示全县交通感知设备的在线状态统计，设备类型包含：

| **设备类型** | **展示内容**           | **说明** |
|--------------|------------------------|----------|
| 电子警察     | 在线数 / 总数，在线率% | —        |
| 违停抓拍     | 在线数 / 总数，在线率% | —        |
| 治安卡口     | 在线数 / 总数，在线率% | —        |
| 超速抓拍     | 在线数 / 总数，在线率% | —        |
| 信号灯控制   | 在线数 / 总数，在线率% | —        |
| 信息发布屏   | 在线数 / 总数，在线率% | —        |

> *📌 感知设备总数在感知体系标题右侧汇总显示，格式："感知设备总数：XXX"*

**3.2.2 徐闻出行 — 车流分析**

展示当日城市进出城车流统计数据：

- 入城车辆数：今日累计入城车辆数量，单位：辆

- 出城车辆数：今日累计出城车辆数量，单位：辆

- 6小时流量趋势图：折线/柱状图展示近6小时入城与出城的每小时流量；鼠标悬停显示tooltip，格式：HH:mm，入城 X 辆，出城 Y 辆

**3.2.3 徐闻出行 — 今日在途车**

- 折线图展示今日与昨日12小时内在途车数量对比趋势

- 提供「今日在途车」和「近7日流动车」两个tab切换

- 近7日流动车：展示近7天每日流动车数量趋势折线图，横坐标为日期（MM/DD格式）

**3.2.4 交通违法事件**

- 今日交通违法事件总数，格式：X 件

- 与上周同期对比，标注增减百分比及方向（↑红色增加，↓绿色减少）

- 近24小时违法事件趋势折线图，精确到小时粒度

**3.3 右侧面板 — 港口交通拥堵分析**

**3.3.1 拥堵情况**

区分「进港方向」和「出港方向」展示以下指标：

| **指标** | **展示规则** | **数据来源** |
|----|----|----|
| 拥堵状态 | 严重拥堵（指数\>8）/ 重度拥堵（6–8）/ 中度拥堵（4–6）/ 轻度拥堵（2–4）/ 道路通畅（0–2），各状态对应不同颜色标签 | 高德API |
| 拥堵指数 | 数值显示，保留一位小数 | 高德API |
| 拥堵时间 | 整条路段拥堵时长，单位：分钟，格式"X分钟"（向上取整） | 高德API |
| 拥堵距离 | \<1000米时格式"Y米"（取整）；≥1000米时格式"Z公里"（保留1位小数） | 高德API |
| 车流量 | 该方向单位时间车辆通过数，格式"X辆/小时" | 海康接口 |

拥堵状态颜色对照：

| **状态** | **拥堵指数范围** | **显示颜色** | **说明** |
|----------|------------------|--------------|----------|
| 道路通畅 | \[0, 2\]         | 绿色         | 正常行驶 |
| 轻度拥堵 | (2, 4\]          | 黄色         | 轻微减速 |
| 中度拥堵 | (4, 6\]          | 橙色         | 明显减速 |
| 重度拥堵 | (6, 8\]          | 红色         | 严重减速 |
| 严重拥堵 | \>8              | 深红色       | 接近停滞 |

**3.3.2 港口车辆**

- 进港车辆数：当日累计进港车辆总数，单位：辆

- 出港车辆数：当日累计出港车辆总数，单位：辆

- 待舶车辆数：当前正在等待上船的车辆数量，单位：辆

- 进出港车辆流量趋势图：折线图展示全天（0:00–24:00）进港与出港车流量曲线，以小时为粒度

**3.3.3 港口运力**

- 运力状态综述：文字标签展示当前整体运力状态（如"运力充足"/"运力紧张"等）

- 可运船舶数量：当前可投入运营的船舶数，单位：艘

- 可运车辆数量：当前可承载的车辆容量，单位：辆

- 待舶车辆分类：按小型车、大型车、电动车、油车分类统计当前等待数量

- 母舰车辆数：当前在渡船上已装载的车辆数，单位：辆

**3.3.4 拥堵预测**

基于当前数据对未来30分钟拥堵趋势进行预测展示：

- 横坐标：按具体时间显示，格式 HH:mm，每5分钟一个刻度，共7个刻度，总计30分钟

- 纵坐标：拥堵指数

- 数据刷新：每5分钟更新一次，更新后第一个刻度为当前最新时间

- 图形：折线图展示预测拥堵指数曲线，区分预测值与实测值（颜色或线型区分）

- 右上角下拉支持切换路段（如：进港大道 / 其他路段）

**3.3.5 堵点预警**

列表展示当前各拥堵路段信息，支持轮播展示：

| **路段名称**           | **拥堵类型** | **拥堵距离** | **拥堵时长** | **说明** |
|------------------------|--------------|--------------|--------------|----------|
| 环半岛公路（进港方向） | 轻度拥堵     | 200米        | 3分钟        | 示例数据 |
| 进港大道（进港方向）   | 轻度拥堵     | 1000米       | 10分钟       | 示例数据 |

**3.3.6 道路承载（轮播列表）**

- 列表展示各路段实时承载情况，字段：路段名称、当前车辆数、更新时间、预测车辆数

- 支持多行轮播，自动滚动

**3.4 中央地图 — 交通态势地图**

**3.4.1 地图底图**

- 基于GIS地图展示徐闻县全域路网，使用深色科技主题底图

- 支持图层控制面板，可按类型开关显示各类感知设备图层

**3.4.2 图层控制**

图层控制面板位于地图左下角，可独立开关以下图层：

| **图层名称** | **说明**                           |
|--------------|------------------------------------|
| 智慧交通     | 整体智慧交通叠加层（总开关）       |
| 电子警察     | 电子警察设备点位图标               |
| 违停抓拍     | 违停抓拍设备点位图标               |
| 治安卡口     | 治安卡口设备点位图标及卡口数字聚合 |
| 超速抓拍     | 超速抓拍设备点位图标               |
| 信号灯控制   | 信号灯控制点位图标                 |
| 信息发布屏   | 信息发布屏设备点位图标             |
| 港口视频     | 港口摄像头视频流点位               |
| 雪亮平台     | 雪亮工程接入摄像头点位             |
| 重点路段     | 进/出港重点路段承载热力展示        |

**3.4.3 路况叠加**

- 通过高德API在地图道路上叠加实时路况颜色（绿/黄/橙/红），直观展示拥堵程度

- 支持进港方向 / 出港方向切换展示，使用不同颜色箭头或线型区分方向

- 拥堵路段使用流动箭头动效标注车辆行进方向

**3.4.4 设备检索**

- 顶部搜索框支持按设备名称或地址关键字搜索

- 搜索结果在地图中定位高亮展示，并可点击查看设备详情

**3.5 卡口设备弹窗**

**3.5.1 触发方式**

点击地图上任意卡口设备图标，右侧滑出卡口详情弹窗。

**3.5.2 弹窗内容**

- 基本信息：设备名称、设备类型、设备地址

- 方向切换：「进港方向」/「出港方向」tab，默认展示进港方向

- 实时监控：调用该卡口对应摄像头实时视频画面

- 点位分析：展示所选方向的拥堵状态、拥堵指数、拥堵时长、平均车速、车流量（辆/小时）

- 出入城流量增长趋势图：折线图展示今日与昨日流量对比，横坐标 0:00–24:00，支持tooltip

**3.5.3 关闭方式**

- 点击弹窗右上角「×」关闭

- 点击地图空白区域关闭

**3.6 重点路段弹窗**

**3.6.1 触发方式**

在图层控制中开启「重点路段」图层后，点击地图上的路段进入重点路段详情弹窗。

**3.6.2 弹窗内容**

- 基本信息：更新时间（格式：HH:mm–HH:mm）

- 行驶车辆数：当前路段行驶车辆数量，并对比近10天历史峰值，用进度条/渐变条展示当前值在峰值中的位置

- 当日车辆数量趋势图：折线图展示今日实际值与预测值对比，横坐标 0:00–24:00，以小时为粒度，实际值用实线，预测值用虚线

- 近10天车辆高峰时间分布：气泡/散点矩阵图，纵轴为天数，横轴为时间（0:00–24:00），气泡大小反映车辆数量

- 近10天车辆峰值趋势（可选）：折线图展示每日峰值趋势

**3.7 交通拥堵调度策略弹窗**

**3.7.1 触发条件**

- 进港或出港方向拥堵指数 \> 2 时，顶部出现「调度策略」入口

- 点击该入口弹出调度策略弹窗

**3.7.2 弹窗内容**

- 弹窗标题：「交通拥堵调度策略」

- 系统概述文字：说明当前检测到的拥堵区域及触发原因

- 策略卡片（2×2 网格布局）：针对当前拥堵级别展示4条调度策略，每张卡片包含：策略名称、策略描述、执行状态（已实施 / 实施中 / 待实施）

**3.7.3 策略内容（按拥堵级别区分静态配置）**

| **策略名称** | **适用拥堵级别** | **策略内容描述** |
|----|----|----|
| 交通信号优化 | 中度拥堵及以上 | 调整进港路段交叉口信号灯配时，延长东西向绿灯时间15%，以缓解主干道压力 |
| 冷链车优先通行 | 中度拥堵及以上 | 在港口周边道路设置冷链运输专用通道，保障生鲜食品运输时效 |
| 交通信息诱导 | 中度拥堵及以上 | 通过电子路牌引导车辆绕行至外围环线，减少市中心区域车流 |
| 港口运力调度 | 中度拥堵及以上 | 优化港口集装箱卸装作业流程，增加夜间作业时段，提升港口吞吐效率 |

> *📌 策略内容当前为静态图文配置，后续可根据业务需要扩展为动态AI生成策略*

**3.7.4 关闭方式**

- 点击右上角「×」关闭弹窗

**3.8 港口交通拥堵预警响应方案弹窗**

**3.8.1 触发条件**

- 当港口拥堵指数达到三级预警阈值（具体数值待运营方确认）时，系统自动弹出预警响应方案弹窗

- 顶部显示当前拥堵指数数值及"重度拥堵"等状态标签，右侧显示「优化策略」按钮

**3.8.2 弹窗内容**

- 标题：「徐闻港口交通拥堵X级预警响应方案」，X根据实际预警级别动态填入

- 弹窗内容：对应级别的响应预案详情（静态配置文本，内容由运营方提供）

- 点击右上角「×」可手动关闭弹窗

> *📌 预警级别分级标准及各级响应预案内容由交通管理部门提供，开发侧按配置内容静态展示*

**4. 数据规格与接口说明**

**4.1 数据来源总览**

| **数据来源** | **提供数据内容** | **更新频率（建议）** | **备注** |
|----|----|----|----|
| 高德地图API | 路况拥堵指数、拥堵时间、拥堵距离、拥堵道路叠加图层 | 实时（约30秒） | 需申请高德交通态势接口权限 |
| 海康威视接口 | 卡口过车数量、车流量（辆/小时） | 每分钟 | 按设备ID查询 |
| 天气接口 | 天气现象、降雨量、风速、预警信息 | 每10分钟 | 推荐使用和风天气或高德天气API |
| 港口业务系统 | 进/出港车辆数、待舶车辆、可运船舶、可运车辆容量 | 每5分钟 | 由港口运营方提供数据接口 |
| 感知设备平台 | 设备在线数、总数、设备状态 | 每分钟 | 对接各设备厂商平台管理API |
| 拥堵预测模型 | 未来30分钟各路段拥堵指数预测值 | 每5分钟 | 由后端算法模型计算输出 |
| 交管业务系统 | 交通违法事件数量及时间序列 | 每分钟 | 对接交管系统API |

**4.2 关键指标计算口径**

**4.2.1 拥堵指数**

- 拥堵指数由高德API直接返回，前端按区间映射状态颜色和文字，无需自行计算

- 分级阈值：\[0,2\] 道路通畅；(2,4\] 轻度；(4,6\] 中度；(6,8\] 重度；\>8 严重

**4.2.2 拥堵时间**

- 数据来源：高德API返回值，单位为分钟

- 展示规则：如不是整数则向上取整（Math.ceil），格式：「X分钟」

**4.2.3 拥堵距离**

- 数据来源：高德API返回值，单位为米

- 展示规则：\<1000米 → 取整显示「Y米」；≥1000米 → 转换为公里保留1位小数「Z公里」

**4.2.4 车流量**

- 数据来源：海康威视接口，统计单位时间内通过卡口的车辆数

- 展示格式：「X辆/小时」，X为1小时内通过车辆总数（整数）

**4.2.5 在途车统计**

- 在途车 = 已入城车辆数 − 已出城车辆数（当日累计值差）

- 按小时粒度聚合展示12小时趋势

**5. 非功能需求**

**5.1 性能要求**

| **指标**         | **要求**                       |
|------------------|--------------------------------|
| 大屏首屏加载时间 | ≤ 3 秒（正常网络环境）         |
| 数据刷新延迟     | 实时数据展示延迟 ≤ 5 秒        |
| 地图渲染帧率     | 地图交互操作帧率 ≥ 30fps       |
| 并发支持         | 支持 ≥ 10 个大屏客户端同时在线 |
| 系统可用性       | ≥ 99.5%（年度）                |

**5.2 兼容性要求**

- 分辨率：主要适配 1920×1080（主屏），兼容 2560×1440（大尺寸会议屏）

- 浏览器：Chrome 90+（推荐），Edge 90+

- 网络：局域网内网部署，要求带宽 ≥ 10Mbps

**5.3 安全要求**

- 系统部署于政务内网，不直接对外网开放

- 前端访问需通过统一身份认证（SSO）

- 视频流调用需鉴权，防止未授权访问

**5.4 显示要求**

- 大屏系统专为展示设计，不需要支持移动端适配

- 主题采用深色科技风（深蓝色底色 + 青色高亮）

- 字体大小需保证在至少2米观看距离下清晰可见

**6. 异常状态处理**
