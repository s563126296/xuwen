#### 一、风格总纲

- 基调：深色沉浸式（#0A0F19 主底），青蓝（#00D0E9）为主色，琥珀橙（#F5A623）为强调色
- 语言：无衬线字体，信息层级清晰，模块圆角 + 描边发光，强化科技质感

#### 二、基础视觉规范

1. **色彩体系**

   | 类型   | 色值             | 用途                       | 占比   |
   | ------ | ---------------- | -------------------------- | ------ |
   | 背景色 | #0A0F19、#121A26 | 大屏底色、模块底板         | 70%    |
   | 主色   | #00D0E9          | 边框、标题、图表主色、高亮 | 15-20% |
   | 强调色 | #F5A623          | 关键数据、排名、重点标注   | ≤10%   |
   | 中性色 | #86909C、#C9CDD4 | 辅助文本、刻度、分割线     | 适配   |

2. **字体规范（1920×1080 基准）**

   | 层级     | 字号    | 字体                   | 应用               |
   | -------- | ------- | ---------------------- | ------------------ |
   | 主标题   | 36-40px | 思源黑体（加粗）       | 大屏顶部标题       |
   | 模块标题 | 24-28px | 思源黑体（加粗）       | 各模块标题         |
   | 核心数据 | 20-24px | DIN / 思源黑体（加粗） | 数据卡片、地图标注 |
   | 辅助文本 | 14-16px | 思源黑体（常规）       | 图表标签、说明     |
   | 角标信息 | 12-14px | 思源黑体（常规）       | 时间、备注         |

#### 三、布局与模块规范

1. **整体布局**

- 结构：中心地图（视觉核心）+ 四周模块（统计、占比、分类等），「1+N」布局
- 安全边距：四周预留 ≥20px，避免内容裁切
- 模块间距：统一 20px，保持视觉通透

2. **模块样式**

- 通用样式：圆角矩形（圆角 8-12px），1px 主色描边 + 5-8px 外发光
- 模块占比：中心地图 ≥40% 空间，其余模块均匀分布
- 栅格系统：12 列栅格，保证模块对齐、缩放统一

#### 四、组件规范

1. **数据卡片**

- 样式：深灰底板 + 主色 / 强调色边框，内部分区清晰
- 展示：数字放大加粗，标签文字弱化，主次分明

2. **图表组件**

| 图表类型        | 适用场景           | 样式规则                                        |
| --------------- | ------------------ | ----------------------------------------------- |
| 环形图 / 进度环 | 数据占比           | 主色填充，图例右上角放置                        |
| 柱状图          | 分类统计、数据分布 | 主色 / 强调色区分系列，网格线浅灰（透明度≤30%） |
| 地图            | 地理分布           | 矢量渐变，标注用六边形悬浮标签，hover 高亮      |

3. **交互组件**

- 按钮：主色填充，48×48px 最小尺寸，hover 亮度提升
- 状态：默认 /hover/ 点击 / 禁用四态清晰，禁用态透明度 50%

#### 五、动效规范

- 加载：模块淡入 + 从下往上入场（0.5s）
- 数据刷新：数字滚动动画（0.3s），无闪烁
- 呼吸效：边框 / 进度环轻微透明度波动，增强科技感
- 限制：单屏动效 ≤3 种，不干扰信息读取

#### 六、数据展示规范
- 格式：大数字用千分位分隔 / 简化（如 2000+），单位统一
- 对齐：数字右对齐，文本左对齐，保证视觉规整
- 异常：用强调色标注异常数据，搭配图标强化识别

#### 七、落地适配
- 分辨率：适配 1920×1080、3840×2160，等比例缩放
- 技术：Web 端（HTML/CSS/JS）、LED 拼接屏，保证 60fps 流畅
- 验收：远距离清晰可读，色彩 / 字体 / 布局无偏差，交互逻辑正常
