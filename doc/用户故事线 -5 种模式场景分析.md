# 徐闻智慧交通大屏 — 5 种模式用户故事线

> 版本：V1.0 | 日期：2026-04-13
> 目的：从真实用户场景出发，检验产品设计是否贴合实际业务需求

---

## 故事线设计方法论

每个故事线按以下结构展开：

```
【角色定位】谁在使用？
【时间场景】什么场景下打开大屏？
【核心目标】他想解决什么问题？
【操作路径】他会如何操作？
【关键信息】他需要看到什么？
【决策动作】最后他会做什么？
【断点检查】当前产品是否支持？缺口在哪？
```

## 一、总览模式 — 日常值班场景

### 故事线 01：早班值班员晨检

**【角色定位】**
- 县交通指挥中心值班员（张师傅，45 岁，值班 8 小时）
- 每天晨间第一次上屏查看全县交通态势

**【时间场景】**
- 时间：工作日早上 7:30
- 背景：早高峰即将来临，需要确认全县交通是否正常

**【操作路径】**
```
1. 打开大屏 → 默认进入总览模式
2. 扫一眼顶部导航 → 确认无红色/橙色预警横幅
3. 扫一眼地图 → 确认整体路况以绿色为主
4. 看右侧拥堵面板 → 确认进港/出港方向畅通
5. 看左栏徐闻出行 → 确认今日车流量正常
6. 10 秒后 → 心态："今天没啥大事，正常值班"
```

**【关键信息】**

| 模块 | 张师傅需要的信息 | 当前产品是否满足 |
|------|----------------|----------------|
| 顶部 | "今天有没有什么需要特别关注的？" — 无预警横幅 | ✅ 有预警栏 |
| 地图 | "全县路况怎么样？有没有大面积拥堵？" — 一眼看红/黄路段 | ✅ 有路况叠加 |
| 右栏拥堵 | "进港/出港通不通？有没有要切换指挥模式的迹象？" | ✅ 有拥堵数据 |
| 左栏车流 | "今天车多不多？有没有异常？" — 入城 1 万 + 出城 1 万正常 | ✅ 有车流统计 |
| 海峡通行 | "琼州海峡通行是否正常？会不会影响今天通行？" | ❌ 缺"海峡通行指数"核心指标 |
| 顶部状态 | "系统正常吗？数据是最新的吗？" — 右上角小标签 | ❌ 缺系统运行状态指示 |

**【决策动作】**
- 若无异常 → 继续值班待命
- 若拥堵指数>4 → 点击 [指挥] 按钮了解详情
- 若收到停航预警 → 点击 [应急] 进入应急模式

**【断点检查】**
> **当前最大缺口：** 缺少一句话结论！
> 
> 张师傅扫一眼大屏，他最想要的是系统告诉他：
> "✅ 今日交通态势正常，无需关注"
> 
> 或者：
> "⚠️ 进港大道轻度拥堵 (指数 2.8)，预计未来 30 分钟缓解"
> 
> **建议：** 在每个面板顶部增加 AI 摘要卡片，用一句话总结当前态势和建议

### 故事线 02：台风季值班员预警查看

**【角色定位】**
- 同张师傅
- 台风季节（7-10 月）

**【时间场景】**
- 时间：台风登陆前 12 小时
- 背景：气象台发布台风黄色预警，港口已收到停航通知

**【操作路径】**
```
1. 打开大屏 → 系统自动切换到应急模式
2. 看到红色横幅："🔴 台风'摩羯'红色预警 | 徐闻港已停航（14:30 起） | 预计停航 48 小时"
3. 扫一眼左栏滞留预测 → "预计最大滞留 3,200 辆"
4. 扫一眼右栏跨部门任务板 → 各部门已收到任务通知
5. 开始打电话协调物资：盒饭需要多少？饮用水多少？
```

**【关键信息】**

| 模块 | 张师傅需要的信息 | 当前产品是否满足 |
|------|----------------|----------------|
| 顶部红色横幅 | "台风预警等级？停航时间？预计滞留多少？" | ✅ 有（应急模式已设计） |
| 滞留预测 | "今天有多少车会被堵在路上？冷链车多少？" | ✅ 有（A/B 模块） |
| 物资需求 | "我明天需要协调多少盒饭和饮用水？" | ✅ 有（C 模块自动估算） |
| 特殊车辆追踪 | "冷链车都去哪了？有没有滞留超过 12 小时的？" | ✅ 有（B 模块） |
| 跨部门任务板 | "各部门收到通知了吗？执行进度怎么样？" | ✅ 有（F 模块） |
| 应急预案 | "我启动了什么预案？还需要做什么？" | ✅ 有（G 模块） |
| 通信记录 | "我要复盘整个事件，需要导出完整记录" | ✅ 有（H 模块） |

**【决策动作】**
- 确认滞留预测数值 → 调动物资
- 查看冷链车分布 → 安排重点值守
- 确认各部门任务进度 → 催促滞后部门
- 等待台风过境 → 进入复航准备期

**【断点检查】**
> **当前缺口：** 通信通知依赖人工！
> 
> 张师傅实际工作中会：
> 1. 打开大屏 → 看到任务板自动生成了任务
> 2. **打电话给公安交警** → "请部署 6 组交警到进港大道"
> 3. **打电话给民政局** → "请准备 3 万份盒饭"
> 4. 打完电话 → 在大屏上"确认"任务已下发
> 
> **理想情况：** 系统自动生成任务 → 智能外呼通知部门 → 部门按 1 确认接收
> 
> **问题：** 智能外呼功能是否已对接？当前仅能在任务板上手动标记

### 故事线 03：县委书记突发巡展

**【角色定位】**
- 县委书记（李书记，50 岁，不常看大屏）
- 某天下午偶然走进指挥中心

**【时间场景】**
- 时间：工作日下午 3 点
- 背景：进港大道轻度拥堵（指数 3.2），正在施工期间

**【操作路径】**
```
1. 走进指挥中心 → 大屏显示总览模式
2. 扫一眼 → "今天进港大道怎么有点堵？"
3. 看右栏拥堵数据 → "指数 3.2，轻度拥堵，拥堵距离 850 米"
4. 看地图 → "施工路段正在拓宽"
5. 看右栏 → "哦，顶部有'调度策略'按钮"
6. 点击 → "有哪些疏导方案？"
7. 看完策略 → "我该怎么处理？"
8. 点击"切换到指挥模式" → 进入详细决策支持
```

**【关键信息】**

| 模块 | 李书记需要的信息 | 当前产品是否满足 |
|------|----------------|----------------|
| 一句话结论 | "今天交通状态怎么样？有没有要管的事？" | ❌ 缺！全是原始数据 |
| 拥堵概况 | "哪里堵？堵多久了？影响多少车？" | ✅ 右栏有 |
| 拥堵原因 | "为什么堵？是港口积压还是施工占道？" | ❌ 缺拥堵归因分析 |
| 调度建议 | "我该怎么做？有方案吗？" | ✅ 调度策略弹窗（简化版） |
| 决策支持 | "执行哪个方案？效果怎么样？" | ❌ 需要切换到指挥模式 |
| 海峡通行 | "琼州海峡通行有问题吗？" | ❌ 缺海峡通行指数 |

**【决策动作】**
- 看完拥堵数据 → 询问"为什么堵？"
- 看完调度策略 → 选择"实施"
- 进入指挥模式 → 查看归因分析（港口积压型）
- 确认策略效果预测 → "那就执行 S-02 S376 分流"
- 一键执行 → 等待效果反馈

**【断点检查】**
> **最大痛点：** 领导不想看数据，想看**结论**！
> 
> 李书记看到的当前界面：
> ```
> 今日入城 12,847 辆 | 今日出城 11,235 辆 | 交通违法 356 件
> ```
> 
> 他真正想知道的是：
> ```
> 🟢 今日交通态势正常
> 
> ⚠️ 进港大道轻度拥堵 (指数 3.2)
> ├─ 原因：港口积压型 (置信度 85%)
> ├─ 待舶车辆溢出到进港大道
> └─ 建议：S376 分流 + 信号灯优化
> ```
> 
> **建议：** 在总览模式顶部增加"AI 总摘要"卡片，一句话总结态势和是否需要干预

### 故事线 03-B：设备异常 — 在线率骤降

**【角色定位】**
- 值班员张师傅
- 发现设备在线率异常下降

**【时间场景】**
- 时间：工作日上午 10 点
- 背景：暴雨导致部分路段设备断电

**【操作路径】**
```
1. 总览模式 → 统计栏自动展开（在线率 <90% 触发）
2. 看到黄色圆点 → "设备 🟡 98/119 在线率 82%"
3. 展开设备详情 → 电子警察 3 台离线、治安卡口 2 台离线
4. 看地图 → 离线设备位置标记为灰色
5. 判断 → "集中在南部路段，可能是暴雨导致断电"
6. 打电话给运维 → "南部 5 台设备离线，请排查"
7. 持续关注 → 等待恢复
```

**【关键信息】**

| 模块 | 张师傅需要的信息 | 当前产品是否满足 |
|------|----------------|----------------|
| 统计栏 | "哪些设备掉线了？掉了多少？" | ✅ 自动展开 + 设备列表 |
| 地图 | "掉线设备在哪？是不是集中在某个区域？" | ⚠️ 有标记但缺离线设备聚合高亮 |
| 原因推测 | "为什么掉线？是断电还是网络故障？" | ❌ 缺故障原因关联分析 |
| 影响评估 | "这些设备掉线影响哪些路段的监控？有没有盲区？" | ❌ 缺监控盲区提示 |
| 运维工单 | "我能直接派工单给运维吗？" | ❌ 缺运维工单对接 |

**【决策动作】**
- 在线率 <90% → 通知运维排查
- 在线率 <80% → 上报值班领导 + 安排人工巡查补位
- 关键路段设备全部离线 → 紧急调度移动设备

**【断点检查】**
> **缺口：设备离线只有「数量」，缺少「影响」**
>
> 张师傅知道 5 台设备掉线了，但不知道：
> - 这 5 台设备覆盖哪些路段？
> - 掉线后这些路段是否变成监控盲区？
> - 对交通管理有什么实际影响？
>
> **建议：** 设备离线时自动标注受影响路段，提示「XX 路段当前无监控覆盖」

### 故事线 03-D：节假日高峰 — 春运/黄金周值守

**【角色定位】**
- 值班指挥员（小王，28 岁）
- 春运期间 24 小时轮班值守

**【时间场景】**
- 时间：春节前 3 天，上午 9 点
- 背景：返乡高峰，日均车流量是平时的 3-5 倍

**【操作路径】**
```
1. 打开大屏 → 总览模式
2. AI 摘要 → "🟠 春运高峰第 3 天，当前车流量为日常 3.2 倍"
3. 看左栏统计 → 入城 38,000 辆（平时 12,000）
4. 看港口消化 → 徐闻港等待 1,200 辆，消化时间 5 小时 40 分
5. 看通道压力 → 南向通道 92%（红色），东向 78%（橙色）
6. 看右栏弹性 → 南向通道剩余容量仅 8%
7. 看右栏韧性 → 系统韧性评分 45（橙色），最弱维度：通道冗余
8. 判断 → "南向通道快饱和了，需要提前启动分流"
9. 切换指挥模式 → 查看分流策略
```

**【关键信息】**

| 模块 | 小王需要的信息 | 当前产品是否满足 |
|------|----------------|----------------|
| AI 摘要 | "今天是春运第几天？车流量是平时几倍？" | ⚠️ 有摘要但缺节假日上下文 |
| 统计栏 | "今天车流量多少？和平时比怎么样？" | ⚠️ 有绝对值但缺基线对比 |
| 通道压力 | "哪个方向快饱和了？" | ✅ 四方向压力罗盘 |
| 通道弹性 | "还有多少余量？能撑多久？" | ✅ 剩余容量百分比 |
| 系统韧性 | "整体还扛得住吗？最薄弱的环节是什么？" | ✅ 韧性评分 + 最弱维度 |
| 预测 | "接下来 2 小时会更堵还是缓解？" | ✅ 压力预测曲线 |
| 历史参考 | "去年春运同期是什么情况？" | ❌ 缺历史同期对比 |

**【决策动作】**
- 通道压力 >85% → 提前启动分流策略
- 港口消化 >4h → 建议旅客选择海安新港
- 系统韧性 <50 → 上报值班领导，申请增援
- 预测 2h 后恶化 → 提前部署交警到关键路口

**【断点检查】**
> **缺口：缺少「节假日模式」上下文**
>
> 春运/黄金周期间，所有指标的「正常基线」都不同。
> 平时入城 12,000 辆是正常的，春运期间 38,000 辆也是正常的。
> 但当前系统用同一套阈值判断，会导致：
> - 春运期间全屏红色/橙色，值班员「预警疲劳」
> - 真正的异常被淹没在大量「正常高峰」告警中
>
> **建议：**
> 1. 增加「节假日模式」，自动切换阈值基线
> 2. AI 摘要增加节假日上下文（"春运第 X 天，车流量为日常 X 倍，属正常高峰范围"）
> 3. 统计栏增加「vs 日常基线」和「vs 去年同期」对比

## 二、指挥模式 — 拥堵处置场景

### 故事线 04：拥堵指数飙升至重度

**【角色定位】**
- 交通指挥员（小王，28 岁，值班 8 小时）
- 日常负责监控拥堵态势

**【时间场景】**
- 时间：周五下午 5 点（下班高峰 + 港口大船靠港）
- 背景：拥堵指数从 2.1 飙升至 6.8（重度）

**【操作路径】**
```
1. 大屏顶部自动切换为指挥模式
2. 左栏 A 模块显示"🔴 重度拥堵"
3. 看 B 模块归因分析 → "港口积压型 (85%) + 流量高峰型 (60%)"
4. 看 C 模块影响评估 → "影响 1,200 辆车，预计持续 45 分钟"
5. 看右栏 F 模块策略卡片 → 推荐"开放应急车道"
6. 看 G 模块效果预测 → "执行后预计 30 分钟后缓解"
7. 思考 → "这个方案可以执行"
8. 点击"提交审批" → 等待值班领导确认
```

**【关键信息】**

| 模块 | 小王需要的信息 | 当前产品是否满足 |
|------|----------------|----------------|
| 拥堵态势 | "现在堵成什么样子了？" — 等级 + 指数 + 距离 + 时间 | ✅ 左栏 A 模块完整 |
| 归因分析 | "为什么堵？主因是什么？" — 5 种归因类型 + 置信度 | ✅ 左栏 B 模块 |
| 影响评估 | "影响多大？要多长时间缓解？" | ✅ 左栏 C 模块 |
| 策略建议 | "我该怎么做？" — 15 个策略 + 推荐标记 | ✅ 右栏 F 模块 |
| 效果预测 | "执行这个方案能降低多少？" — 预测值 + 置信度 | ✅ 右栏 G 模块 |
| 执行跟踪 | "策略执行后效果怎么样？" — 预测 vs 实际对比 | ✅ 右栏 H 模块 |
| 地图联动 | "我看到地图上拥堵路段在哪" — 高亮 + 排队长度 | ✅ 中央 D 模块 |
| 视频联动 | "我想看现场视频确认" — 自动调取最近摄像头 | ✅ 中央 E 模块画中画 |

**【决策动作】**
- 确认归因分析 → 港口积压型为主因
- 查看推荐策略 → 开放应急车道
- 查看效果预测 → 拥堵指数从 6.8 降到 4.2
- 提交审批 → 等待领导确认
- 领导确认后执行 → 等待效果反馈

**【断点检查】**
> **当前产品状态：** 指挥模式功能设计完整，但代码未实现
> 
> 目前 `CommandMode.tsx` 仅有基础框架：
> ```tsx
> <div className="main-content" style={{position:'absolute'}}>
>   <CommandLeftPanel />
>   <CommandMap />
>   <CommandRightPanel />
> </div>
> ```
> 
> **缺口：** A-J 共 10 个模块的完整功能（归因分析、15 策略库、效果预测、执行跟踪）均未实现
> 
> **优先级建议：** P0 级功能，应优先开发

### 故事线 05：策略执行后效果不佳

**【角色定位】**
- 同小王
- 策略执行后 15 分钟

**【时间场景】**
- 背景：执行"开放应急车道"策略后，拥堵指数从 6.8 仅降到 6.2（效果不佳）

**【操作路径】**
```
1. 看右栏 H 模块执行跟踪 → "策略执行中，但效果未达标"
2. 看 G 模块提示 → "建议追加方案 B：S376 分流"
3. 对比两个方案效果 → 策略 A 剩余 2 分钟缓解，策略 B 剩余 10 分钟
4. 点击"追加方案 B" → 提交审批
5. 领导确认后执行 → 等待效果反馈
```

**【关键信息】**

## 八、核心改进建议

### 1. 增加"AI 总摘要"卡片（所有模式）

**位置：** 每个模式顶部导航栏下方

**内容示例：**
```
🟢 今日交通态势正常
- 进港/出港方向均畅通
- 海峡通行指数：35（通畅）

⚠️ 进港大道轻度拥堵 (指数 3.2)
- 原因：港口积压型 (置信度 85%)
- 建议：S376 分流 + 信号灯优化
- [查看归因] [切换到指挥模式]
```

### 2. 实现智能模式切换（P0 优先级）

**触发条件：**
- 拥堵指数>4 → 顶部橙色提示 + [指挥] 闪烁
- 拥堵指数>6 → 自动切换指挥模式
- 台风预警/停航通知 → 自动切换应急模式
- 满足退出条件 → 提示切换回总览模式

### 3. 跨模式数据共享（架构级改造）

**统一数据层：**
- 所有模式共享同一套数据源（港口 API、气象 API、高德 API）
- 策略库全局可调用（应急模式可直接调用 15 个策略）
- 策略状态跨模式同步（执行中/已完成）

### 4. 优先实现指挥模式和应急模式（P0）

**理由：**
- 这是系统的核心差异化价值
- 领导最关心的是"告诉我为什么堵、该怎么办"
- 数据接入后，这两个模式能立即产生价值

### 5. 逐步补齐分析模式（P2）
