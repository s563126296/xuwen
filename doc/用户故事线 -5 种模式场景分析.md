# 徐闻智慧交通大屏 — 5 种模式用户故事线

> 版本：V2.0 | 日期：2026-04-23（基于 V1.0 校对更新，同步已实现功能状态）
> 目的：从真实用户场景出发，检验产品设计是否贴合实际业务需求

---

## 故事线设计方法论

每个故事线按以下结构展开：

```
【角色定位】谁在使用？
【时间场景】什么场景下打开大屏？
【核心目标】他想解决什么问题？
【操作路径】他会如何操作？
【关键信息】他需要看到什么？
【决策动作】最后他会做什么？
【断点检查】当前产品是否支持？缺口在哪？
```

## 一、总览模式 — 日常值班场景

### 故事线 01：早班值班员晨检

**【角色定位】**
- 县交通指挥中心值班员（张师傅，45 岁，值班 8 小时）
- 每天晨间第一次上屏查看全县交通态势

**【时间场景】**
- 时间：工作日早上 7:30
- 背景：早高峰即将来临，需要确认全县交通是否正常

**【操作路径】**
```
1. 打开大屏 → 默认进入总览模式
2. 扫一眼顶部导航 → 确认无红色/橙色预警横幅
3. 扫一眼地图 → 确认整体路况以绿色为主
4. 看右侧拥堵面板 → 确认进港/出港方向畅通
5. 看左栏徐闻出行 → 确认今日车流量正常
6. 10 秒后 → 心态："今天没啥大事，正常值班"
```

**【关键信息】**

| 模块 | 张师傅需要的信息 | 当前产品是否满足 |
|------|----------------|----------------|
| 顶部 | "今天有没有什么需要特别关注的？" — 无预警横幅 | ✅ 有预警栏 |
| 地图 | "全县路况怎么样？有没有大面积拥堵？" — 一眼看红/黄路段 | ✅ 有路况叠加 |
| 右栏拥堵 | "进港/出港通不通？有没有要切换指挥模式的迹象？" | ✅ 有拥堵数据 |
| 左栏车流 | "今天车多不多？有没有异常？" — 入城 1 万 + 出城 1 万正常 | ✅ 有车流统计 |
| 海峡通行 | "琼州海峡通行是否正常？会不会影响今天通行？" | ✅ 有海峡通行指数（StraitTransitIndex，风力/能见度/通航状态） |
| 顶部状态 | "系统正常吗？数据是最新的吗？" — 右上角小标签 | ✅ 顶部显示实时天气（温度/天气/风力/能见度/海况） |

**【决策动作】**
- 若无异常 → 继续值班待命
- 若拥堵指数>4 → 点击 [指挥] 按钮了解详情
- 若收到停航预警 → 点击 [应急] 进入应急模式

**【断点检查】**
> **V1.0 缺口已修复 ✅**
> 
> AI 态势研判面板（AiSummaryBar）已实现，折叠态一句话结论：
> "✅ 当前交通态势正常，无需特别关注"
> 或：
> "🟠 进港大道轻度拥堵（指数 3.2）| 建议 14:45 启动 S376 分流"
> 
> 展开态三栏布局：当前态势（4 指标）+ 趋势预判（2h 时间轴）+ 建议操作（3 条可执行）
> 
> **剩余缺口：** 骨架屏加载状态（P2 级）

### 故事线 03：县委书记突发巡展

**【角色定位】**
- 县委书记（罗书记，50 岁，不常看大屏）
- 某天下午偶然走进指挥中心

**【时间场景】**
- 时间：工作日下午 3 点
- 背景：进港大道轻度拥堵（指数 3.2），正在施工期间

**【操作路径】**
```
1. 走进指挥中心 → 大屏显示总览模式
2. 扫一眼 → "今天进港大道怎么有点堵？"
3. 看右栏拥堵数据 → "指数 3.2，轻度拥堵，拥堵距离 850 米"
4. 看地图 → "施工路段正在拓宽"
5. 看右栏 → "哦，顶部有'调度策略'按钮"
6. 点击 → "有哪些疏导方案？"
7. 看完策略 → "我该怎么处理？"
8. 点击"切换到指挥模式" → 进入详细决策支持
```

**【关键信息】**

| 模块 | 罗书记需要的信息 | 当前产品是否满足 |

| 决策支持 | "执行哪个方案？效果怎么样？" | ✅ 指挥模式效果预测曲线 + 执行跟踪 |

- 看完调度策略 → 选择"实施"
- 进入指挥模式 → 查看归因分析（港口积压型）
- 确认策略效果预测 → "那就执行 S-02 S376 分流"

### 故事线 03-B：设备异常 — 在线率骤降

**【角色定位】**
- 值班员张师傅
- 发现设备在线率异常下降

**【时间场景】**
- 时间：工作日上午 10 点
- 背景：暴雨导致部分路段设备断电

**【操作路径】**
```
1. 总览模式 → 统计栏自动展开（在线率 <90% 触发）
2. 看到黄色圆点 → "设备 🟡 98/119 在线率 82%"
3. 展开设备详情 → 电子警察 3 台离线、治安卡口 2 台离线
4. 看地图 → 离线设备位置标记为灰色
5. 判断 → "集中在南部路段，可能是暴雨导致断电"
6. 打电话给运维 → "南部 5 台设备离线，请排查"
7. 持续关注 → 等待恢复
```

**【关键信息】**

| 模块 | 张师傅需要的信息 | 当前产品是否满足 |
|------|----------------|----------------|
| 统计栏 | "哪些设备掉线了？掉了多少？" | ✅ 自动展开 + 设备列表 + 离线设备闪烁高亮 |
| 地图 | "掉线设备在哪？是不是集中在某个区域？" | ✅ 离线设备灰色标记 + 闪烁动画 |
| 原因推测 | "为什么掉线？是断电还是网络故障？" | ❌ 缺故障原因关联分析（P2 级） |
| 影响评估 | "这些设备掉线影响哪些路段的监控？有没有盲区？" | ❌ 缺监控盲区提示（P2 级） |
| 运维工单 | "我能直接派工单给运维吗？" | ❌ 缺运维工单对接（P3 级） |

**【决策动作】**
- 在线率 <90% → 通知运维排查
- 在线率 <80% → 上报值班领导 + 安排人工巡查补位
- 关键路段设备全部离线 → 紧急调度移动设备

**【断点检查】**
> **缺口：设备离线只有「数量」，缺少「影响」**
>
> 张师傅知道 5 台设备掉线了，但不知道：
> - 这 5 台设备覆盖哪些路段？
> - 掉线后这些路段是否变成监控盲区？
> - 对交通管理有什么实际影响？
>
> **建议：** 设备离线时自动标注受影响路段，提示「XX 路段当前无监控覆盖」

```

5. 看通道压力 → 南向通道 92%（红色），东向 78%（橙色）
6. 看右栏弹性 → 南向通道剩余容量仅 8%
7. 看右栏韧性 → 系统韧性评分 45（橙色），最弱维度：通道冗余

**【关键信息】**

| 模块 | 小王需要的信息 | 当前产品是否满足 |
|------|----------------|----------------|
| AI 摘要 | "今天是春运第几天？车流量是平时几倍？" | ✅ 节假日 AI 摘要上下文（"五一假期第 X 天 · 车流量为日常 X 倍"） |

| 系统韧性 | "整体还扛得住吗？最薄弱的环节是什么？" | ✅ 韧性评分 + 最弱维度高亮（SystemResilienceCard） |
| 预测 | "接下来 2 小时会更堵还是缓解？" | ✅ 压力预测曲线（PressurePredictionChart） |
| 历史参考 | "去年春运同期是什么情况？" | ⚠️ AI 摘要有同比参考（vs 去年同期），但缺详细历史对比视图 |

**【决策动作】**
- 通道压力 >85% → 提前启动分流策略
- 港口消化 >4h → 建议旅客选择海安新港
- 系统韧性 <50 → 上报值班领导，申请增援
- 预测 2h 后恶化 → 提前部署交警到关键路口

**【断点检查】**
> **V1.0 缺口已修复 ✅**
> 
> 节假日模式已实现：
> - ✅ AI 摘要自动识别节假日上下文（"五一假期第 X 天 · 车流量为日常 X 倍"）
> - ✅ 统计栏显示节假日倍率标签（vs 日常基线）
> - ✅ AI 摘要展开态显示同比数据（vs 去年同期）
> - ✅ 节假日场景下阈值自动调整，避免"预警疲劳"

## 二、指挥模式 — 拥堵处置场景

### 故事线 04：拥堵指数飙升至重度

**【角色定位】**
- 交通指挥员（小王，28 岁，值班 8 小时）
- 日常负责监控拥堵态势

**【时间场景】**
- 时间：周五下午 5 点（下班高峰 + 港口大船靠港）
- 背景：拥堵指数从 2.1 飙升至 6.8（重度）

**【操作路径】**
```
1. 大屏顶部自动切换为指挥模式
2. 左栏 A 模块显示"🔴 重度拥堵"
3. 看 B 模块归因分析 → "港口积压型 (85%) + 流量高峰型 (60%)"
4. 看 C 模块影响评估 → "影响 1,200 辆车，预计持续 45 分钟"
5. 看右栏 F 模块策略卡片 → 推荐"开放应急车道"
6. 看 G 模块效果预测 → "执行后预计 30 分钟后缓解"
7. 思考 → "这个方案可以执行"
8. 点击"提交审批" → 等待值班领导确认
```

**【关键信息】**

| 模块 | 小王需要的信息 | 当前产品是否满足 |
|------|----------------|----------------|
| 拥堵态势 | "现在堵成什么样子了？" — 等级 + 指数 + 距离 + 时间 | ✅ 左栏 A 模块完整 |
| 归因分析 | "为什么堵？主因是什么？" — 5 种归因类型 + 置信度 | ✅ 左栏 B 模块 |
| 影响评估 | "影响多大？要多长时间缓解？" | ✅ 左栏 C 模块 |
| 策略建议 | "我该怎么做？" — 15 个策略 + 推荐标记 | ✅ 右栏 F 模块 |
| 效果预测 | "执行这个方案能降低多少？" — 预测值 + 置信度 | ✅ 右栏 G 模块 |
| 执行跟踪 | "策略执行后效果怎么样？" — 预测 vs 实际对比 | ✅ 右栏 H 模块 |
| 地图联动 | "我看到地图上拥堵路段在哪" — 高亮 + 排队长度 | ✅ 中央 D 模块 |
| 视频联动 | "我想看现场视频确认" — 自动调取最近摄像头 | ✅ 中央 E 模块画中画 |

**【决策动作】**
- 确认归因分析 → 港口积压型为主因
- 查看推荐策略 → 开放应急车道
- 查看效果预测 → 拥堵指数从 6.8 降到 4.2
- 提交审批 → 等待领导确认
- 领导确认后执行 → 等待效果反馈
