核心系统流程图

模块交互

一次"拥堵事件"从触发到处置完成，系统各模块之间的调用关系和数据流向

触发源
高德路况API
拥堵指数突增 → 推送事件
视频AI检测
车辆排队超阈值 → 触发告警
人工上报
一线人员通过通讯系统上报
事件进入系统
AI引擎
风险预测引擎
读取9项指标 → 计算风险等级(绿/黄/橙/红)
因果分析引擎
5维归因(天气/事故/港口/潮汐/节假日)
策略匹配引擎
规则库匹配 → 冲突检测 → 输出3条方案
写入 commandStore + 推送UI
前端UI
AI摘要栏更新
显示预警等级 + 态势结论 + 建议
策略面板渲染
展示3条方案 + 冲突标记 + 效果预测
地图热力图
拥堵路段高亮 + 影响范围标注
指挥员点击"确认执行"
执行层
指令下发模块
拆解策略 → 生成子任务 → 分配到人/设备
通讯系统
消息推送 + 视频连线 + 快捷指令
外部系统调用
信号灯API / 诱导屏API / 广播系统
执行进度回传(30秒轮询)
监控层
偏差监控引擎
实际指标 vs 预期 → 超±15%触发预警
执行看板
进度条 + 子任务状态 + 视频画面
反馈分析引擎
执行完成 → 效果评分 → 归档到analysisStore
↺ 偏差超阈值时：监控引擎 → 生成调整建议 → 推送UI → 指挥员二次决策 → 重新进入执行层
