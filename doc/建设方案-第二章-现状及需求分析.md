# 第二章 现状及需求分析

## 2.1 现状分析

### 2.1.1 徐闻县交通区位概况

徐闻县位于中国大陆最南端，是琼州海峡北岸的重要交通枢纽，承担着粤琼两省陆路交通的咽喉功能。徐闻港（新港）作为琼州海峡主要客货滚装运输港口，是进出海南岛的核心通道之一。2023年第一季度，徐闻港车辆吞吐量约157万辆次，同比增长52%，旅客吞吐量约529万人次，同比增长68%，充分体现了琼州海峡通道的交通压力持续增长态势。

**交通基本数据：**

- 日均进港车辆：约8,000辆（平日），春运高峰期进港方向日均车流量可达50,000辆次
- 港口日均发船班次：约100+班次（徐闻港约86班/日 + 海安新港约21班/日）
- 单船载车量：35-120辆（主力船型60-80辆，大型船可达120辆以上）
- 港口日消化能力：约8,000-10,000辆（受泊位、航次、单船载车量限制）
- 主要进港通道：进港大道（双车道至三车道，约14.8公里）、徐海路（三车道，约4.2公里）、环半岛公路（三车道，约7.3公里），三条道路合计约20公里以上
- 拥堵高峰时段：中午至晚上八九点
- 历史最严重拥堵：2023年春运期间，进港大道、徐海大道、环半岛公路三条主要道路全线拥堵，排队长度约20公里，路上动态滞留车辆约1万辆，持续约5天
- 琼州海峡年停航次数：每年因台风、大雾等恶劣天气停航约10-20次（台风季7-10月尤为频繁），每次停航后恢复通航极易引发集中出行和大规模拥堵

由于港口运力有限（日消化能力约8,000-10,000辆），当车辆到达速率超过港口消化能力时，极易在进港通道形成大规模拥堵。春运期间日均车流量达平日的6倍以上，远超港口消化能力，拥堵问题尤为突出。此外，琼州海峡每年因台风、大雾等恶劣天气停航约10-20次，停航期间车辆积压，恢复通航后集中出行形成"堰塞湖效应"，是除春运外最主要的拥堵诱因。

### 2.1.2 现有交通管理手段

目前徐闻县针对进港交通拥堵已建立了一套以"预约管控"为核心的管理体系：

**（1）预约制管控**

所有车辆进入徐闻港必须提前通过预约系统预约，未预约车辆不允许进入港口区域。这是控制进港车辆总量的核心手段。

**（2）道路分流与临时停车**

在拥堵发生或逐步加剧时，徐闻县政府动态调整道路功能：
- 将次要道路临时划为停车区域，降低主要道路车辆通行压力；
- 保障非进港车辆的正常通行；
- 保障进港出租车（旅客坐船）的优先通行。

**（3）人工设卡检查**

在即将拥堵或拥堵逐步增加时，在关键路口设置检查点，安排人员核验车辆预约状态，未预约车辆不允许继续前行。

**（4）信息发布**

通过道路诱导屏、广播等方式发布分流指引和停车区指引信息。

**（5）视频监控**

主要路段和港口区域已部署视频监控摄像头（海康/大华），用于日常巡查和事件发现。

|-----------|------|------|

## 2.2 存在问题

尽管徐闻县已建立了以预约制为核心的交通管控体系，但在实际运行中仍存在以下突出问题：

### 2.2.1 拥堵预判能力不足

当前交通管理以"被动响应"为主——等到拥堵已经发生或明显加剧后才启动应对措施。缺乏基于预约数据、实时路况、气象信息等多源数据的综合预判能力，无法在拥堵形成之前提前介入。

**具体表现：**
- 无法根据未来时段的预约量预判即将到来的车流高峰；
- 无法结合气象预报（如台风恢复通航后的集中出行）提前预警；
- 从发现拥堵到启动分流措施存在明显时间差。

### 2.2.2 多源数据未打通

预约数据、实时路况数据、气象数据、视频监控数据分散在不同系统中，缺乏统一的数据汇聚和关联分析平台。管理人员需要在多个系统间切换查看，无法形成对交通态势的全局认知。

**具体表现：**
- 预约系统只能看到预约量，看不到实时路况；
- 路况数据只反映当前状态，无法结合预约数据预判趋势；
- 气象预警与交通管理决策之间缺乏自动化联动。

### 2.2.3 决策依赖人工经验

何时启动道路分流、何时设置检查点、向哪些预约用户发送通知、建议调整到什么时间到达——这些关键决策目前完全依赖管理人员的个人经验判断，缺乏数据支撑和标准化流程。

**具体表现：**
- 不同值班人员对同一态势的判断和响应可能不同；
- 缺乏量化的分级响应标准（如拥堵指数达到多少启动分流）；
- 预约用户通知的时机和内容缺乏精准性。

### 2.2.4 缺少空中视角能力

当完全拥堵发生时，地面视角受限，难以快速获取拥堵的全貌信息（拥堵长度、滞留车辆数量、拥堵蔓延方向等）。目前尚未配置无人机，无法提供空中巡查、喊话疏导等辅助手段。

### 2.2.5 缺乏复盘与持续优化能力

每次拥堵处置结束后，缺乏系统化的数据记录和效果评估机制。历史处置经验未能沉淀为可量化、可对比的数据资产，无法支撑策略的持续优化。

**具体表现：**
- 无法回答"上次春运分流措施效果如何"；
- 无法对比不同处置策略的缓解时间和效果；
- 领导汇报材料需要手动整理，缺乏自动化报告能力。

### 2.3.2 业务层面

徐闻港作为琼州海峡北岸核心港口，其交通保障能力直接关系到：
- **粤琼两省互联互通**：徐闻港是海南岛与大陆连接的生命线通道；
- **春运保障**：每年春运期间大量车辆集中过海，交通压力极大；
- **应急安全**：台风停航后恢复通航的集中出行、重大事故等突发事件需要快速响应；
- **民生服务**：减少旅客和司机的等待时间，提升出行体验。

建设智慧交通指挥系统，将现有的"被动响应"模式升级为"主动预判+精准调度"模式，是保障琼州海峡通道安全畅通的必要举措。

### 2.3.3 技术层面

当前人工智能、大数据分析、物联网等技术已经成熟，具备支撑智慧交通系统建设的条件：
- AI预测技术可实现基于多源数据的拥堵预判；
- 大屏可视化技术可实现交通态势的直观呈现；
- 无人机技术可提供低成本的空中巡查和应急服务能力；
- 数据融合技术可打通多个信息孤岛，形成统一决策视图。

## 2.4 需求分析

### 2.4.1 总体需求

建设一套AI驱动的智慧交通指挥决策系统，实现：
- **10秒判断态势**：一屏总览交通全局，AI实时评估风险等级；
- **1分钟拿到方案**：AI根据当前阶段自动推荐处置策略；
- **全程看到AI在干活**：从预判到执行到复盘，AI贯穿全流程。

### 2.4.2 功能需求

基于徐闻县交通管理的实际业务场景，系统需要支撑以下核心能力：

**（1）态势感知与一屏总览**

| 需求 | 说明 |
|------|------|
| 实时路况展示 | 在地图上展示主要道路的拥堵状态、车速、拥堵长度 |
| 预约数据可视化 | 展示当前及未来时段的预约量分布，识别即将到来的高峰 |
| 气象信息融合 | 展示气象预警信息，评估对交通的影响 |
| AI态势摘要 | AI自动生成当前交通态势的文字结论和风险等级 |
| 关键指标看板 | 拥堵指数、预约饱和度、港口消化率等核心KPI |

**（2）智能预判与预警**

| 需求 | 说明 |
|------|------|
| 拥堵预测 | 基于预约数据+路况趋势+气象信息，提前30分钟预判拥堵 |
| 分级预警 | 绿色（平稳）→ 黄色（关注）→ 橙色（严重）→ 红色（紧急）四级预警 |
| 主动提醒 | 预警触发时自动弹窗提醒值班人员，附带AI分析结论 |

**（3）分级响应与策略执行**

系统需支撑徐闻县现有的分阶段递进响应机制：

| 阶段 | 触发条件 | 系统支撑能力 |
|------|---------|-------------|
| 常态管控 | 拥堵指数正常 | 日常监控，预约数据展示 |
| 预警通知 | 预判即将拥堵 | AI建议向预约用户发送通知，调整到达时间 |
| 道路分流 | 拥堵逐步增加 | AI推荐分流方案，展示分流路线，指令下达 |
| 全面管控 | 完全拥堵 | 无人机任务派发，设卡点位建议，资源调度 |

**（4）无人机协同**

| 需求 | 说明 |
|------|------|
| 任务派发 | 拥堵达到一定程度时，系统建议无人机起飞，指定巡查路线 |
| 视频回传 | 无人机实时画面接入系统大屏，提供空中视角 |
| 拥堵统计 | 基于无人机视角统计拥堵长度、车辆数量，辅助决策 |
| 喊话疏导 | 通过无人机喊话引导车辆进入停车区或配合分流 |
| 效果监控 | 分流执行后通过无人机确认执行效果 |

**（5）复盘分析与报告**

| 需求 | 说明 |
|------|------|
| 历史回溯 | 时间轴回放历史事件处置全过程 |
| 策略评估 | 对比不同处置策略的效果（缓解时间、缓解率） |
| 趋势分析 | 按日/周/月/季度分析交通趋势 |
| 报告导出 | 一键生成Excel/PDF/PPT格式的分析报告 |
