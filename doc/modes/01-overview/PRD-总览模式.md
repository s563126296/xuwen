> 版本：V2.1 | 日期：2026-04-15
> 优先级：P0 | 状态：前端开发完成，V2.1 迭代优化完成

## 一、模式定位

日常值班的默认模式，7×24小时常驻显示。目标是让值班人员**10秒内掌握全县交通全局态势**，不需要点击、不需要切换，扫一眼大屏就知道"现在正不正常"。

### 1.1 目标用户

| 角色 | 场景 | 核心诉求 | 使用频率 |
|------|------|---------|---------|
| 值班员 | 7x24h 值班室监控 | 第一时间发现异常并上报，不遗漏 | 持续使用，每日 8h+ |
| 县领导 | 巡查/会议/汇报 | 30 秒了解全局，快速判断是否需要干预 | 每日 2-3 次，每次 1-5 分钟 |
| 交通指挥员 | 拥堵/事故时切入 | 从总览快速切换到指挥模式执行调度 | 事件驱动，每周 3-5 次 |

### 1.2 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|-------|---------|
| 异常发现时间 | 值班员发现异常到上报 ≤3 分钟 | 上报记录时间戳 vs 系统告警时间戳 |
| 信息自解释率 | 领导巡查时主动提问次数 ≤2 次/次 | 人工记录 |
| AI 建议采纳率 | 态势研判面板建议被执行 ≥60% | 建议点击率统计 |
| 模式切换响应 | 从总览切换到指挥/应急 ≤2 秒 | 前端埋点 |
| 大屏可用性 | 系统无故障运行 ≥99.5% | 运维监控 |

### 1.3 与其他模式的关系

| 关系 | 说明 |
| ---- | ---- |
| 总览 → 指挥 | AI 态势研判面板建议操作可一键切换到指挥模式 |
| 总览 → 应急 | 收到停航通知/台风橙色预警时自动切换 |
| 总览 → 港口 | AI 态势研判面板建议操作可一键切换到港口模式 |
| 总览 → 分析 | 手动切换 |
| 指挥/应急 → 总览 | 拥堵缓解/应急解除后提示切换回总览 |

**核心设计原则：**
1. **零操作感知** — 不需要点击就能获取关键信息
2. **异常突出** — 正常状态安静，异常状态醒目（颜色/动效）
3. **信息分层** — 一级信息一眼可见，二级信息点击可达
4. **决策导向** — AI 态势研判面板直接给出结论和可执行建议

## 三、核心指标体系（双核 15 指标）

> 详细设计见：[总览模式-指标体系升级设计.md](./总览模式-指标体系升级设计.md)

从"统计报表"升级为"双核决策体征"系统，围绕**港口交通**和**县城交通**两个核心，辅以共享底座。

### 3.1 港口交通核心（5 个指标）

| 指标 | 组件 | 状态 | 验收标准 |
|------|------|------|---------|
| 拥堵扩散情况 | PressureTransmissionCard | ✅ | 展示港口→通道→城区→全域四级传导链，数据源：高德路况 API，刷新 ≤30s |
| 停航风险预测 | ShutdownProbabilityCard | ✅ | 展示 6h/24h/48h/72h 四时间窗概率，数据源：气象+通航通知，刷新 ≤10min |
| 港口排队情况 | PortDigestionCard | ✅ | 展示双港等待车辆数+消化时间，数据源：港口闸机 API，误差 ≤5%，刷新 ≤5min |
| 进出港流量对比 | TidalEffectCard | ✅ | 展示进出港流量比+潮汐状态，数据源：卡口统计，刷新 ≤5min |
| 未来 2h 压力预测 | PressurePredictionChart | ✅ | 展示预测曲线+事件标注+Tooltip，实测点实线/预测点虚线区分，刷新 ≤5min |

### 3.2 县城交通核心（5 个指标）

| 指标 | 组件 | 状态 | 验收标准 |

| 进出城通道压力 | CorridorPressureCard | ✅ | 四向压力百分比，数据源：卡口统计，刷新 ≤5min |
| 城区拥堵热点 | UrbanHealthCard（集成） | ✅ | 常发性/偶发性分类标签，数据源：历史拥堵统计 |

## 四、AI 态势研判面板（核心创新）

替代原有的简单预警滚动条，升级为结构化决策面板。

### 4.1 折叠态（默认，48px）

**左侧区域：**
- 等级图标（绿色✓/黄色⚠/橙色⚠/红色✕）
- 一句话结论（如"正常高峰"）
- 建议提示（如"建议14:45启动S376分流"）

**右侧区域：**
- 三个关键数字徽章：
  - 车流（34,847 辆）
  - 港口（1,200 辆等待）
  - 韧性（45/100）

### 4.2 展开态（点击展开，~280px）

**三栏布局：**

| 栏位 | 内容 | 说明 |
|------|------|------|
| 左栏「当前态势」 | 4 个核心指标卡片 | 今日车流/港口等待/南向通道压力/应急韧性 |
| 中栏「趋势预判(2h)」 | 时间轴式预判 | 15:00达峰/15:30韧性降/16:30回落 |
| 右栏「建议操作」 | 3 条可执行建议 | 每条有优先级圆点+操作按钮（切换模式） |

**底部「同比参考」：**
- vs 去年同期数据对比（车流/港口/拥堵指数）

### 4.3 数据结构（与代码一致）

```typescript
// src/store/dashboardStore.ts
interface AiSummary {
  level: 'green' | 'yellow' | 'orange' | 'red';
  conclusion: string;           // 折叠态一句话结论
  suggestionHint: string;       // 折叠态建议提示
  badges: Array<{ label: string; type: 'flow' | 'port' | 'resilience' }>;
  headerTitle: string;          // 展开态标题
  metrics: AiSummaryMetric[];   // 当前态势 4 个指标卡片
  forecasts: AiSummaryForecast[]; // 趋势预判时间轴
  actions: AiSummaryAction[];   // 建议操作（可执行）
  compares: AiSummaryCompare[]; // 同比参考
  compareConclusion: string;    // 同比结论
  expanded: boolean;
}
```

### 4.4 实现状态

- ✅ 折叠/展开交互
- ✅ 三栏布局
- ✅ 操作按钮点击切换模式（指挥/港口/分析）
- ⏳ 规则引擎动态计算（当前静态 mock）

## 十、异常状态处理

| 异常场景 | 处理方式 |
|---------|---------|
| 接口超时 | 保留上次数据，显示"数据更新中..." |
| 空数据 | 显示"—"，图表显示空状态占位 |
| 设备离线 | 灰色图标，在线率分母仍计入 |
| 视频断开 | "信号丢失"占位图 |
| 高德加载失败 | 降级显示 SVG 模拟路网 |
| 多异常并发 | AI 摘要按严重度排序：红>橙>黄>绿，同级按时间倒序，折叠态只显示最高级别 |
| 无人机信号丢失 | 地图上该无人机图标变灰+闪烁，标签显示"信号丢失"，弹窗画面冻结并提示 |
| 无人机电量不足(<20%) | 标签变橙色显示"电量低"，AI 摘要自动提示"无人机-XX 电量不足，建议召回" |
| 无人机超出巡航范围 | 自动触发返航，地图上显示返航轨迹 |
| 节假日模式切换 | 系统根据特殊事件日历自动切换，值班员可手动覆盖；切换时 AI 摘要提示"已切换为节假日基线" |
| 设备在线率 <90% | AI 摘要自动增加设备告警条目，等级提升为黄色 |
| 设备在线率 <80% | AI 摘要等级提升为橙色，建议操作增加"排查设备故障" |
