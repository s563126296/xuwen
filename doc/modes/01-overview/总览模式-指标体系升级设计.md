# 总览模式 — 指标体系升级设计

> 版本：V2.0 | 日期：2026-04-13
> 基于：PRD-总览模式 V1.0 + 用户故事线头脑风暴
> 核心变化：从"统计报表"升级为"决策体征"

---

## 一、设计理念

### 1.1 核心认知

徐闻不是普通县城。它是中国大陆通往海南的唯一陆路口岸，琼州海峡年运输 552 万辆车。
徐闻的交通本质是**跨海通道的陆侧枢纽**，核心矛盾是：

> **有限的陆地通道容量** vs **波动剧烈的跨海需求**

这意味着徐闻的拥堵是**事件驱动型、脉冲式**的（大船靠港、台风停航），
而非北京式的"每天差不多"的通勤拥堵。指标体系必须围绕这个特性设计。

### 1.2 指标分层架构

```
第一层：AI 态势摘要（一句话结论）
  ↑ 综合
第二层：决策体征指标（10 个核心指标）
  ↑ 提炼
第三层：基础统计指标（设备/车流/违法，折叠展示）
  ↑ 采集
第四层：原始数据源（高德/海康/港口/气象 API）
```

### 1.3 交互原则

- 默认展示第一层 + 第二层（零操作感知）
- 第三层折叠，点击展开查看详情
- 第四层不直接展示，仅作为计算输入

## 三、第二层：决策体征指标（10 个核心指标）

### 3.1 港城压力传导指数

**回答的问题**：港口的问题有没有"传染"到城区？

**计算模型**：
```
传导链路（4 个节点，每个节点 0-100 分）：

N1. 港口压力源 = f(待舶车辆数 / 港口容量, 发船延误率)
    - 待舶/容量 < 50%：0-30 分
    - 待舶/容量 50%-80%：30-60 分
    - 待舶/容量 > 80%：60-100 分

N2. 进港通道压力 = f(进港大道拥堵指数, 排队长度)
    - 拥堵指数 < 2：0-20 分
    - 拥堵指数 2-4：20-50 分
    - 拥堵指数 4-6：50-80 分
    - 拥堵指数 > 6：80-100 分

N3. 城区交叉口压力 = f(关键路口饱和度均值)
    - 饱和度 < 0.6：0-30 分
    - 饱和度 0.6-0.8：30-60 分
    - 饱和度 > 0.8：60-100 分

N4. 全域扩散度 = f(拥堵路段数 / 总监测路段数)
    - 拥堵占比 < 10%：0-20 分
    - 拥堵占比 10%-30%：20-50 分
    - 拥堵占比 > 30%：50-100 分

传导状态 = 最高已激活节点（N 值 ≥ 40 视为激活）
```

**状态判定**：
| 状态 | 条件 | 颜色 | 含义 |
|------|------|------|------|
| 🟢 解耦 | N1 < 40 | 绿色 | 港口压力低，未传导 |
| 🟡 传导中 | N1 ≥ 40 且 N2 ≥ 40 | 黄色 | 压力到达进港通道 |
| 🟠 蔓延 | N3 ≥ 40 | 橙色 | 压力扩散到城区路口 |
| 🔴 全域 | N4 ≥ 40 | 红色 | 全城受影响 |

**数据依赖**：
| 数据 | 来源 | 当前状态 | 降级方案 |
|------|------|---------|---------|
| 待舶车辆数 | 港口闸机 API | P0 待接入 | 用进港大道流量推算 |
| 进港大道拥堵指数 | 高德路况 API | 已有 Key | — |
| 城区路口饱和度 | 海康卡口 5min | P1 升级 | 用高德路况替代 |
| 拥堵路段数 | 高德路况 API | 已有 Key | — |

**展示设计**：
```
┌─────────────────────────────────────────────┐
│ 港城压力传导  🟡 传导中                       │
│                                             │
│ [港口]──→[进港通道]──→[城区]──→[全域]        │
│  ●62      ●45         ○28      ○12          │
│  ▓▓▓▓▓▓  ▓▓▓▓░░      ░░░░░    ░░░░░        │
│                                             │
│ 港口待舶 450 辆 → 进港大道排队 850m          │
│ 预计 30min 后传导至城区路口                   │
└─────────────────────────────────────────────┘
```

交互：
- 点击任意节点 → 展开该节点的详细数据
- 点击"港口"节点 → 显示待舶车辆、发船间隔、装载效率
- 点击"进港通道"节点 → 显示拥堵指数、排队长度、平均车速

### 3.2 通道弹性余量

**回答的问题**：哪条路快撑不住了？（拥堵前预警）

**计算模型**：
```
弹性余量 = (设计通行能力 - 当前流量) / 设计通行能力 × 100%
额外可承受 = 设计通行能力 - 当前流量（辆/h）
```

**各通道参数**（需交通局确认实际值）：
| 通道 | 设计容量（辆/h） | 车道数 | 备注 |
|------|----------------|--------|------|
| 进港大道 | ~1,200 | 双向四车道 | 按单向估算 |
| G207 国道 | ~800 | 双向两车道 | 国道标准 |
| S376 省道 | ~600 | 双向两车道 | 省道标准 |
| 环半岛公路 | ~400 | 双向两车道 | 县道标准 |

**阈值**：
| 余量 | 状态 | 颜色 | 含义 |
|------|------|------|------|
| ≥ 50% | 充裕 | 绿色 | 可应对突发 |
| 30%-50% | 一般 | 黄色 | 正常运行 |
| 15%-30% | 紧张 | 橙色 | 接近饱和 |
| < 15% | 临界 | 红色 | 即将拥堵 |

**数据依赖**：

### 3.3 系统韧性评分

**回答的问题**：如果现在来一个突发事件，系统能不能扛住？

**计算模型**：
```
韧性评分 = w1×通道冗余分 + w2×备选路线分 + w3×调控空间分 + w4×港口缓冲分

通道冗余分（权重 30%）：
  = 各通道弹性余量的加权平均（按流量权重）
  
备选路线分（权重 20%）：
  = 可用替代路线数 / 总替代路线数 × 100
  - 施工封闭的路线不计入可用
  - 拥堵指数 > 6 的路线不计入可用

调控空间分（权重 20%）：
  = (信号灯可调整数 / 总信号灯数 × 50) + (诱导屏可用数 / 总诱导屏数 × 50)
  - 已处于最大绿信比的信号灯不计入可调整

港口缓冲分（权重 30%）：
  = (港口容量 - 当前待舶) / 港口容量 × 100
```

**等级判定**：
| 评分 | 等级 | 含义 |
|------|------|------|
| 80-100 | 💪 健壮 | 能应对大多数突发 |
| 60-80 | 🟢 正常 | 能应对一般突发 |
| 40-60 | 🟡 脆弱 | 突发可能导致拥堵 |
| 20-40 | 🟠 危险 | 任何突发都会导致严重拥堵 |
| 0-20 | 🔴 崩溃边缘 | 已无调控空间 |

**展示设计**：
```
┌──────────────────────────┐
│ 系统韧性  82/100  💪     │
│                          │
│      ╭────────╮          │
│     ╱   82    ╲         │
│    ╱    💪     ╲        │
│    ╰──────────╯         │
│                          │
│ 通道冗余 ████░ 78        │
│ 备选路线 █████ 90        │
│ 调控空间 ███░░ 65        │
│ 港口缓冲 ████░ 85        │
│                          │
│ 薄弱环节：调控空间        │
└──────────────────────────┘
```

交互：
- 点击任意子维度 → 展开详细数据
- 评分 < 40 时自动在 AI 摘要中提示

### 3.5 停航概率预测

```
- 风力 ≥ 8 级：停航概率 95%

- 能见度 < 500m（持续 2h+）：停航概率 60%
- 台风中心距离 < 300km：停航概率 80%
- 台风中心距离 300-500km：停航概率 40%

时间窗口置信度：
- 未来 6h：高置信度（气象短临预报准确率 > 85%）
- 未来 24h：中置信度（气象预报准确率 ~70%）
- 未来 48h：低置信度（趋势参考）
- 未来 72h：仅供参考
```

### 3.6 港口消化时间预估

**回答的问题**：港口排队的车多久能走完？

**计算模型**：
```
消化时间 = 当前待舶车辆数 / 有效发运速率

有效发运速率 = 船舶装载量 × (60 / 发船间隔) × 装载效率

参数参考值（需港口方确认）：
  徐闻港：每船 ~80 辆，发船间隔 ~25min，装载效率 ~85%
  海安新港：每船 ~60 辆，发船间隔 ~30min，装载效率 ~80%

健康阈值：
- < 2h：🟢 正常消化
- 2-4h：🟡 消化偏慢（关注）
- 4-6h：🟠 消化不良（需干预）
- > 6h：🔴 严重积压（建议启动分流）
```

### 3.7 潮汐效应指数

**回答的问题**：现在是进港潮还是出港潮？强度如何？

**计算模型**：
```
潮汐比 = 进港方向总流量 / 出港方向总流量

状态判定：
- 潮汐比 > 1.5：进港潮（车辆涌向港口）
- 潮汐比 0.67-1.5：均衡态
- 潮汐比 < 0.67：出港潮（车辆涌出港口）

强度 = |ln(潮汐比)| / ln(历史最大潮汐比) × 100%
- 强度 < 30%：轻度
- 强度 30%-60%：中等
- 强度 > 60%：强烈

翻转时间预测：基于历史同期模式 + 船舶到港时刻表
```

**展示设计**：
```
┌─────────────────────────────────────────┐
│ 潮汐效应                                 │
│                                         │
│      ← 出港潮  │  进港潮 →              │
│  ░░░░░░░░░░░░░█▓▓▓▓▓▓░░░░░░░░░░       │
│                    ▲ 当前 2.3:1          │
│                                         │
│ 状态：进港潮（中等强度）                  │
│ 进港 847 辆/h  ←→  出港 368 辆/h        │
│ 预计翻转时间：16:30                      │
│                                         │
│ 💡 进港方向资源应倾斜配置                 │
```

### 3.8 气象-交通耦合度

**回答的问题**：天气对交通的影响有多大？

**计算模型**：
```
耦合度 = max(海峡影响分, 陆地影响分)

海峡影响分：
- 风力 ≥ 6 级：+40
- 浪高 > 2m：+30
- 能见度 < 1km：+30
- 通航限制通知：+50

陆地影响分：
- 暴雨（>50mm/h）：+50
- 大雾（能见度 < 1km）：+40
- 路面积水报告：+30
- 大风（≥ 7 级）：+20

等级：
- 0-20：🟢 无影响
- 20-40：🟡 轻微影响
- 40-60：🟠 显著影响
- 60-80：🔴 严重影响
- 80-100：⛔ 极端影响
```

**展示设计**：
```
┌─────────────────────────────────────────┐
│ 气象-交通耦合  🟡 轻微影响                │
│                                         │
│ 🌊 海峡：风力 5 级(阵风 6 级) → 船舶减速  │
│ 🛣️ 陆地：小雨 → 路面湿滑                 │
│                                         │
│ 趋势：4h 后风力增至 6 级 → 🟠 显著影响   │
└─────────────────────────────────────────┘
```

交互：
- 点击 → 展开完整气象数据面板（风力/浪高/能见度/降雨/温度）
- 耦合度 ≥ 40 时自动在 AI 摘要中提示

### 3.9 异常流量检测

**回答的问题**：今天的车流量正不正常？

**计算模型**：
```
异常度 = (当前流量 - 历史同期均值) / 历史同期标准差

历史同期定义：
- 同星期几（周一 vs 周一）
- 同时段（上午 vs 上午）
- 排除节假日和特殊事件日

判定：
- |异常度| < 1.0：正常波动（不标注）
- 1.0 ≤ |异常度| < 2.0：轻微异常（黄色标注）
- |异常度| ≥ 2.0：显著异常（红色标注 + 推测原因）

原因推测规则：
- 进城异常高 + 周末/节假日 → "旅游/探亲高峰"
- 进城异常高 + 工作日 → "周边活动/施工绕行"
- 出城异常低 + 港口正常 → "高速封闭/天气影响"
- 双向异常高 + 节假日前一天 → "节前出行高峰"
- 双向异常低 + 恶劣天气 → "天气抑制出行"
```

**展示方式**：
集成在"徐闻出行"折叠条的默认态中：
```
出行  进城 12,847(+23%⚠️)  出城 11,235(-2%)
                  ↑ 异常检测结果直接标注
```

点击展开后显示详细分析：
```
异常分析：
├─ 进城流量 12,847 辆（历史同期均值 10,450 ± 1,100）
├─ 异常度 2.18 → 显著偏高 ⚠️
├─ 推测原因：周边县市有大型活动（需人工确认）
└─ 历史相似日：2025-10-15（国庆后补班日，流量 12,300）
```

### 3.13 进出城通道压力

**回答的问题**：进出徐闻县城的几个方向压力怎么样？

**计算模型**：
```text
四个方向：
- 北向（G207 → 湛江/广州方向）
- 南向（进港大道 → 港口方向）← 与港口指标共享数据

- 西向（S376 → 海安新港方向）← 与港口指标共享数据

每个方向计算：
- 压力 = 当前流量 / 设计容量 × 100%
```

### 3.14 城区拥堵热点

## 七、实施优先级

### 第一批：可立即实现（无新数据依赖）

| 指标 | 依赖数据 | 说明 |
|------|---------|------|
| 基础体征折叠条 | 现有数据 | 纯前端交互改造 |
| AI 态势摘要 | 现有拥堵数据 | 规则引擎，先用简化版 |
| 通道弹性余量 | 海康卡口 + 设计容量 | 需交通局确认容量参数 |
| 异常流量检测 | 海康卡口历史数据 | 需积累 2-4 周历史数据 |
| 潮汐效应 | 海康卡口（进/出方向） | 现有数据可算 |
| 特殊事件日历 | 人工录入 | 纯前端，季节性事件预置 |

### 第二批：需高德 API 接入

| 指标 | 依赖数据 | 说明 |
|------|---------|------|
| 港城压力传导 | 高德路况 + 港口闸机 | 港口数据可先用 mock |
| 未来 2h 压力预测 | 高德路况 + 历史模式 | 先做历史模式版，后加 AIS |
| 系统韧性评分 | 综合多指标 | 依赖通道弹性等指标先上线 |
| 城区路网健康度 | 高德路况（城区路段） | 与港城传导同批接入 |
| 城区拥堵热点 | 高德路况 + 历史模式 | 常发/偶发分类需历史数据 |
| 进出城通道压力 | 高德路况 + 海康卡口 | 四向数据，南/西向与港口共享 |

### 第三批：需外部数据接入

| 指标 | 依赖数据 | 说明 |
|------|---------|------|
| 海峡通行指数 | 气象 + AIS + 通航通知 | 可先用气象降级版 |
| 停航概率预测 | 气象预报 + 历史停航数据 | 需气象局数据 |
| 气象-交通耦合度 | 海洋气象 API | 需接入和风天气/国家海洋局 |
| 港口消化时间 | 港口闸机 + 船舶排班 | 需港口方配合 |
| 关键路口饱和度 | 海康卡口(路口级) + 信号灯配时 | 需信号灯系统对接 |
