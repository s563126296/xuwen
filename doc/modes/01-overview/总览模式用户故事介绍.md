# 徐闻智慧交通大屏 — 总览模式用户故事介绍

> 作者：张明晨 | 公司：科大讯飞 | 日期：2026-04-14

---

## 一、总览模式定位

**核心理念**：10 秒态势感知

值班人员打开大屏，不用点任何东西，10 秒内就能判断「今天正不正常」。这是 24/7 日常监控的默认模式。

---

## 二、5 个典型场景

### 场景 1：早班晨检 — 一天的开始

**角色**：值班员张师傅，45 岁，早上 7:30 第一次上屏

**他想知道什么？**
- 今天有没有需要特别关注的事？
- 全县路况怎么样？
- 进港/出港通不通？
- 车流量正不正常？

**10 秒操作路径**：
```
打开大屏 → 扫顶部（无预警横幅）→ 扫地图（绿色为主）
→ 看右栏拥堵（畅通）→ 看左栏车流（正常）
→ 心态："今天没啥大事，正常值班"
```

**产品缺口**：
- ❌ 缺一句话结论（"今日交通态势正常，无需关注"）
- ❌ 缺海峡通行指数
- ❌ 缺系统运行状态指示

---

### 场景 2：台风预警 — 应急响应

**角色**：值班员张师傅，台风登陆前 12 小时

**他想知道什么？**
- 台风预警等级？停航时间？
- 预计滞留多少车？
- 需要协调多少物资？
- 各部门任务执行进度？

**操作路径**：
```
打开大屏 → 系统自动切换应急模式
→ 看红色横幅（台风预警 + 停航通知）
→ 看滞留预测（3,200 辆）
→ 看物资需求（自动估算盒饭/饮用水）
→ 看任务板 → 开始打电话协调
```

**产品缺口**：
- ❌ 通信通知依赖人工（理想：系统自动外呼通知部门）

---

### 场景 3：领导巡展 — 快速决策

**角色**：县委书记李书记，50 岁，偶然走进指挥中心

**他想知道什么？**
- 今天交通状态怎么样？有没有要管的事？
- 为什么堵？
- 我该怎么做？

**操作路径**：
```
走进指挥中心 → 扫一眼大屏
→ "今天进港大道怎么有点堵？"
→ 看拥堵数据（指数 3.2，轻度拥堵）
→ 看地图（施工路段）
→ 点击"调度策略" → 查看疏导方案
→ 切换指挥模式 → 详细决策
```

**产品缺口**：
- ❌ 领导不想看数据，想看结论（"进港大道轻度拥堵，原因：港口积压型，建议：S376 分流"）
- ❌ 缺拥堵归因分析

---

### 场景 4：设备异常 — 运维响应

**角色**：值班员张师傅，暴雨导致设备断电

**他想知道什么？**
- 哪些设备掉线了？
- 掉线设备在哪？是不是集中在某个区域？
- 为什么掉线？
- 影响哪些路段的监控？

**操作路径**：
```
总览模式 → 统计栏自动展开（在线率 82% < 90%）
→ 看到黄色圆点 + 设备列表
→ 看地图（离线设备标记为灰色）
→ 判断："集中在南部路段，可能是暴雨断电"
→ 打电话给运维 → 持续关注
```

**产品缺口**：
- ⚠️ 地图缺离线设备聚合高亮
- ❌ 缺故障原因关联分析（断电 vs 网络故障）
- ❌ 缺监控盲区提示（"XX 路段当前无监控覆盖"）
- ❌ 缺运维工单对接

---

### 场景 5：节假日高峰 — 春运值守

**角色**：值班指挥员小王，28 岁，春运期间 24 小时轮班

**他想知道什么？**
- 今天是春运第几天？车流量是平时几倍？
- 哪个方向快饱和了？
- 还有多少余量？
- 去年春运同期是什么情况？

**操作路径**：
```
打开大屏 → AI 摘要（"春运第 3 天，车流量为日常 3.2 倍"）
→ 看统计（入城 38,000 辆 vs 平时 12,000）
→ 看通道压力（南向 92% 红色）
→ 看弹性（剩余容量 8%）
→ 看韧性（评分 45，最弱：通道冗余）
→ 判断："南向通道快饱和了"
→ 切换指挥模式 → 启动分流策略
```

**产品缺口**：
- ⚠️ AI 摘要缺节假日上下文
- ⚠️ 统计栏缺基线对比（"vs 日常基线" "vs 去年同期"）
- ❌ 缺节假日模式（阈值基线自动切换，避免预警疲劳）
- ❌ 缺历史同期数据参考

## 三、核心发现

### 1. 设计亮点

✅ **10 秒感知理念贯穿始终** — 所有卡片都有颜色编码，异常自动展开

✅ **压力传导链设计** — 港口→通道→城区→全城，因果关系可视化

✅ **多角色适配** — 值班员看数据，领导看结论，运维看设备

### 2. 共性缺口

所有 5 个场景都指向同一个问题：

> **缺少「一句话结论」**

用户不想自己分析数据，他们想要系统直接告诉他们：
- 今天正不正常？
- 有没有要管的事？
- 我该怎么做？

### 3. 优先级建议

**P0 — 必须补的：**
1. AI 摘要卡片（一句话总结态势 + 行动建议）
2. 节假日模式（阈值基线切换）
3. 监控盲区提示（设备离线影响评估）

**P1 — 应该做的：**
4. 拥堵归因分析
5. 基线对比（vs 日常 / vs 去年同期）
6. 离线设备聚合高亮

**P2 — 可以做的：**
7. 运维工单对接
8. 智能外呼通知
9. 历史同期数据参考
