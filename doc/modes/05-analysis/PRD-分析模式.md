# 05 分析模式（Analysis Mode）— 产品需求文档

> 版本：V1.0 | 日期：2026-04-19
> 优先级：P1 | 状态：需求定义中

---

## 一、模式定位

历史回溯、数据分析、决策支持的**智能分析中枢**。为管理者提供**历史查询 + 策略评估 + 事件复盘 + 报告生成**的完整分析能力，让大屏从"实时工具"升级为"分析工具"。

核心价值：从"看当前"到"看趋势"，从"凭经验"到"用数据"，从"手工整理"到"自动报告"。

**与其他模式的关系：**

| 模式 | 定位 | 数据流向 |
| ---- | ---- | -------- |
| 总览模式 | 实时监控 | 当前态势 → 分析模式历史库 |
| 指挥模式 | 策略执行 | 策略记录 → 分析模式统计 |
| 应急模式 | 应急指挥 | 事件记录 → 分析模式归档 |
| 港口模式 | 通行分析 | 船舶数据 → 分析模式趋势 |
| **分析模式** | 历史分析 | 读取所有模式历史数据 |

---

## 二、触发与退出规则

### 触发条件

- 手动切换：任何时候均可从导航栏进入
- 无自动触发条件（分析模式是主动查询工具）

### 退出条件

- 手动切换到其他模式
- 无自动退出条件

## 四、核心功能模块

### A. 查询条件面板（左上）

**功能：** 多维度筛选历史数据

**字段：**

| 编号 | 字段 | 类型 | 说明 |
| ---- | ---- | ---- | ---- |
| A-01 | 时间范围 | 日期选择器 | 开始日期 ~ 结束日期，快捷按钮：今日/本周/本月/近3月 |
| A-02 | 事件类型 | 多选下拉 | 拥堵事件/台风停航/大雾停航/春运高峰/重大事故/日常运行 |
| A-03 | 策略筛选 | 多选下拉 | 15种策略（S-01~S-15），支持按类型分组 |
| A-04 | 区域筛选 | 单选 | 全域/进港大道/S376省道/G207国道/县城区域/港口区域 |
| A-05 | 响应等级 | 多选 | I级/II级/III级/IV级 |
| A-06 | 查询按钮 | 按钮 | 点击执行查询，更新中央视图和右栏统计 |

### B. 历史事件列表（左中）

**功能：** 展示筛选后的历史事件，支持滚动和点击查看详情

| 编号 | 功能 | 说明 |
| ---- | ---- | ---- |
| B-01 | 事件卡片 | 事件名称 + 时间 + 类型标签 + 严重程度色条 |
| B-02 | 排序 | 按时间倒序（最新在前） |
| B-03 | 点击详情 | 点击事件卡片 → 中央区显示事件详情 |
| B-04 | 事件状态 | 已归档（灰色）/ 进行中（青色）/ 待复盘（橙色） |
| B-05 | 搜索 | 支持按事件名称关键词搜索 |

### C. 报告管理（左下）

**功能：** 管理自动生成和手动创建的分析报告

| 编号 | 功能 | 说明 |
| ---- | ---- | ---- |
| C-01 | 自动报告 | 每日运行报告（自动生成）、每周汇总报告 |
| C-02 | 手动报告 | 用户选择时间范围 + 事件 → 生成自定义报告 |
| C-03 | 报告模板 | 日报模板/周报模板/事件复盘模板/领导汇报模板 |
| C-04 | 报告列表 | 按时间倒序展示已生成报告，点击预览 |

### D. 时间轴控制器（底部）

**功能：** 拖动时间轴回放历史态势

| 编号 | 功能 | 说明 |
| ---- | ---- | ---- |
| D-01 | 时间轴 | 可拖动的时间轴，标注关键事件节点 |
| D-02 | 播放控制 | 播放/暂停/快进/快退按钮 |
| D-03 | 速度控制 | 1x/2x/4x/8x 回放速度 |
| D-04 | 事件标记 | 时间轴上标注拥堵/停航/策略执行等关键节点 |
| D-05 | 当前时间 | 显示当前回放时间点 |

### E. 主视图区（中央）

**功能：** 根据查询条件和选中事件展示不同视图

| 编号 | 功能 | 说明 |
| ---- | ---- | ---- |
| E-01 | 趋势对比图 | 车流量/拥堵指数/港口消化率的时间趋势线 |
| E-02 | 场景对比 | 今年vs去年、工作日vs节假日、春运vs平日 双轴对比 |
| E-03 | 策略效果分析 | 策略执行前后指标变化、平均缓解时间、采纳率 |
| E-04 | 事件时间轴 | 选中事件的完整时间线（触发→响应→执行→恢复） |
| E-05 | 热力图 | 按小时×星期的车流量/拥堵热力矩阵 |
| E-06 | 视图切换 | Tab 切换不同视图（趋势/对比/策略/事件/热力） |

### F. 详情面板（中央叠加）

**功能：** 点击事件或策略后展示详细信息

| 编号 | 功能 | 说明 |
| ---- | ---- | ---- |
| F-01 | 事件详情 | 事件概况 + 影响范围 + 响应过程 + 处置结果 |
| F-02 | 策略详情 | 策略名称 + 执行时间 + 效果指标 + 前后对比 |

### G. 统计摘要（右上）

**功能：** 查询结果的关键指标概览

| 编号 | 功能 | 说明 |
| ---- | ---- | ---- |
| G-01 | 事件总数 | 查询范围内的事件总数 |
| G-02 | 策略执行次数 | 查询范围内的策略执行总次数 |
| G-03 | 平均缓解时间 | 策略执行后平均缓解时间 |
| G-04 | 策略采纳率 | AI建议策略被采纳的比例 |
| G-05 | 最高拥堵指数 | 查询范围内的峰值拥堵指数 |
| G-06 | 总滞留车辆 | 查询范围内的累计滞留车辆数 |

### H. 快速筛选（右中）

**功能：** 常用查询场景的快捷入口

| 编号 | 功能 | 说明 |
| ---- | ---- | ---- |
| H-01 | 今日概览 | 一键查看今日运行数据 |
| H-02 | 本周汇总 | 一键查看本周数据汇总 |

## 五、数据模型

### 历史事件（HistoryEvent）

```typescript
interface HistoryEvent {
  id: string;
  name: string;                    // 事件名称
  type: 'congestion' | 'typhoon' | 'fog' | 'spring_rush' | 'accident' | 'normal';
  severity: 'critical' | 'major' | 'minor' | 'info';
  startTime: string;               // ISO 时间
  endTime: string | null;          // 进行中为 null
  status: 'archived' | 'active' | 'pending_review';
  location: string;                // 发生区域
  peakCongestionIndex: number;     // 峰值拥堵指数
  maxStrandedVehicles: number;     // 最大滞留车辆数
  strategiesUsed: string[];        // 使用的策略ID列表
  responseLevel: 'I' | 'II' | 'III' | 'IV' | null;
  summary: string;                 // 事件摘要
  timeline: EventTimelineItem[];   // 事件时间线
}

interface EventTimelineItem {
  time: string;
  action: string;
  actor: string;                   // 执行人/部门
  result: string;
}
```

### 策略执行记录（StrategyRecord）

```typescript
interface StrategyRecord {
  id: string;
  strategyId: string;              // S-01 ~ S-15
  strategyName: string;
  eventId: string;                 // 关联事件
  executedAt: string;
  completedAt: string | null;
  preIndex: number;                // 执行前拥堵指数
  postIndex: number;               // 执行后拥堵指数
  reliefMinutes: number;           // 缓解时间（分钟）
  adopted: boolean;                // 是否被采纳
  executor: string;                // 执行人
}
```

### 分析报告（AnalysisReport）

```typescript
interface AnalysisReport {
  id: string;
  title: string;
  type: 'daily' | 'weekly' | 'event_review' | 'custom';
  generatedAt: string;
  dateRange: { start: string; end: string };
  content: ReportSection[];
  status: 'draft' | 'published';
}

interface ReportSection {
  title: string;
  type: 'text' | 'chart' | 'table' | 'metrics';
  data: any;
}
```

## 七、用户故事

### 故事线 08：领导要数据

> 县委书记要在下周的常委会上汇报交通治理成效，需要一份近3个月的数据报告。

**操作流程：**
1. 切换到分析模式
2. 左栏设置时间范围：近3个月
3. 右栏点击"生成报告" → 选择"领导汇报模板"
4. 系统自动生成报告（含趋势图、策略统计、事件列表）
5. 预览报告 → 导出PPT

### 故事线 09：复盘台风事件

> 台风"摩羯"过后，应急办要复盘整个应急响应过程。

**操作流程：**
1. 切换到分析模式
2. 左栏事件列表找到"台风摩羯停航事件"
3. 点击 → 中央区显示完整事件时间线
4. 查看各阶段响应时间、策略效果、资源调度情况
5. 生成事件复盘报告

### 故事线 10：对比分析

> 交通局想知道今年春运和去年春运的差异。

**操作流程：**
1. 切换到分析模式
2. 右栏点击"春运专题"快捷筛选
3. 中央区切换到"场景对比"视图
4. 选择对比维度：今年春运 vs 去年春运
5. 查看车流量、拥堵指数、策略执行次数的对比
