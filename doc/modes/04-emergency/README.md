# 04 应急模式（Emergency Mode）

## 定位

台风停航、极端天气等应急场景下的指挥调度模式。徐闻独有的高价值场景。

## 触发方式

- 台风预警发布 → 自动切换 + 红色横幅
- 港口停航通知 → 自动切换
- 手动切换（退出需二次确认）

## 核心功能

1. 停航滞留预测模型（当前流量 × 停航时长 → 滞留峰值）
2. 冷链车专项追踪（不能熄火，最紧急的特殊车辆）
3. 跨部门协同任务板（公安交警/民政/交通局/港口管理方）
4. 无人机调度集成（大疆无人机实时画面）
5. 应急物资自动估算（盒饭、饮用水需求量）

## 数据依赖

- 台风预警：中国气象局API（可接入）
- **停航提前通知：港口管理方会在线上公告前提前通知县政府相关部门**
  - 这是关键的"提前量"数据，可能比公众早数小时甚至半天
  - 利用这个时间差，系统可提前计算滞留预测、部署应急资源
- 冷链车识别：卡口摄像头 + 车牌识别（部分可用）
- 危化品车辆：报备数据库 + 车辆标识识别
- 无人机画面：大疆SDK（待采购飞手服务）
- 港口售票数据：用于精确预测停航后的滞留车辆规模

## 优先级

P0 — 徐闻独有场景，差异化价值最高
