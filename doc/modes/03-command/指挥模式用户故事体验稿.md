# 指挥模式用户故事体验稿

> 作者：张明晨 | 公司：科大讯飞 | 日期：2026-04-14
> 基于：指挥模式-指标体系设计 V1.0 + 用户故事线 04/05

---

## 设计理念

指挥模式的大屏体验围绕一条决策链展开：

```
看到拥堵 → 知道为什么 → 知道影响多大 → 知道怎么办 → 知道效果如何 → 验证结果
```

值班指挥员从进入指挥模式到完成一次处置，经历完整的 6 步。每一步在大屏上都有对应的区域和交互。

---

## 故事一：周五晚高峰 — 港口积压型拥堵处置全流程

### 角色
交通指挥员小王，28 岁，值班 8 小时。已在总览模式下监控了一天。

### 背景
周五下午 5 点，下班高峰 + 港口 5000 吨大船靠港，进港大道拥堵指数从 2.1 开始持续攀升。

---

**大屏变化**：
```

│                                                        │
```

**小王看到的**：顶部出现橙色横幅，[指挥] 按钮在闪。他扫了一眼左栏港口排队 — 徐闻港等待 580 辆，消化时间 3 小时 20 分。

**小王的判断**："港口积压了，指数还在往上走，我先观察一下。"

## 故事四：事故 + 拥堵 — 策略自动联动

### 背景

**小王看到什么**：不需要自己判断能不能往 G207 分流，系统已经替他想好了。

## 核心体验总结

| 步骤 | 用户问题 | 大屏位置 | 体验目标 |
|------|---------|---------|---------|
| 指挥摘要 | 什么情况？该怎么办？ | 顶部摘要条 | 一句话搞定，3 秒理解全局 |
| 态势感知 | 有多堵？ | 左栏上 | 大字数字，一眼识别严重程度 |
| 归因分析 | 为什么堵？ | 左栏中 | 原因 + 置信度，不用猜 |
| 影响评估 | 影响多大？ | 左栏下 | 车辆数 + 冷链车标记 + 波及预测 |
| 策略推荐 | 该怎么办？ | 右栏上 | 自动推荐 + 冲突检查 + 一键执行 |
| 效果预测 | 会怎么样？ | 右栏中 | 预测曲线 + 置信区间 + 风险提示 |
| 执行跟踪 | 做到哪了？ | 右栏下 | 预测 vs 实际 + 达标判定 + 追加建议 |

**设计原则**：指挥员的视线路径是 **左上 → 左中 → 左下 → 右上 → 右中 → 右下**，正好对应决策链的 6 步。不需要翻页、不需要切换 Tab、不需要弹窗。一屏解决。
