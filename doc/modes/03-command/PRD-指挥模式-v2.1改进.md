# 指挥模式 v2.1 改进文档

> 版本：v2.1
> 日期：2026-04-29
> 基于：指挥模式 v2.0
> 改进类型：交互优化 + 智能化提升

---

## 一、改进概述

v2.1 版本在 v2.0 的基础上，针对用户测试反馈进行了 11 项核心改进，重点提升了：
1. **决策支持完整性**：策略卡片信息从"建议"升级为"完整方案"
2. **执行可视化**：地图从"态势图"升级为"策略执行图"
3. **交互流畅性**：面板按决策阶段自适应，减少信息过载

---

## 二、11 项核心改进

### 1. 右侧面板按决策阶段重构

**改进前：**
- 右侧面板内容固定，不随执行状态变化
- 所有信息堆砌在一起，决策阶段不清晰

**改进后：**
- 面板内容根据执行阶段动态切换（选择策略/执行中/执行完毕）
- 每个阶段只显示当前需要的信息

**三个阶段：**

#### 阶段 1：选择策略（未执行）
```
┌─────────────────────────┐
│ AI 推荐策略              │
│ [完整增强卡片]           │
├─────────────────────────┤
│ 备选策略（可折叠）        │
├─────────────────────────┤
│ [自定义策略]             │
├─────────────────────────┤
│ 当前可用资源              │
└─────────────────────────┘
```

#### 阶段 2：执行中
```
┌─────────────────────────┐
│ 当前执行策略              │
│ 执行进度 + 预期时间线     │
├─────────────────────────┤
│ 资源到位状态              │
├─────────────────────────┤
│ 偏差监控摘要              │
├─────────────────────────┤
│ 备选/追加策略             │
└─────────────────────────┘
```

#### 阶段 3：执行完毕
```
┌─────────────────────────┐
│ 执行结果摘要              │
│ 效果评价（达标/超预期）    │
├─────────────────────────┤
│ AI 学习摘要               │
│ 本次学到了什么             │
├─────────────────────────┤
│ [查看复盘报告]            │
│ [返回总览模式]            │
└─────────────────────────┘
```

**业务价值：**
- 减少信息过载，每个阶段只看需要的内容
- 决策流程更清晰，用户不会迷失在功能堆砌中

---

### 2. 增强策略卡片（推荐理由 + 历史依据 + 预期效果 + 资源需求）

**改进前：**
- 策略卡片只显示：策略名称、效果值（6.5→4.8）、权限标签、难度星级
- 缺少"为什么推荐"、"历史战绩如何"、"需要什么资源"

**改进后：**
每个策略卡片包含 4 部分完整信息：

#### 1. 推荐理由（1-2 句话）
```
当前港口积压 70%，南向通道压力 96%，
S376 可拦截 40% 车流，历史数据显示类似场景下成功率 87%
```

#### 2. 历史依据
```
30 次执行 | 成功率 87% | 匹配度 85% | 置信度 86%
```

#### 3. 预期效果时间线
```
资源 10 分钟到位
1 小时后拥堵指数从 6.8 降至 5.2
检查点：30min→6.2 | 60min→5.2 | 90min→4.5
```

#### 4. 资源需求摘要
```
交警 2 人 ✓ | 诱导屏 1 块 ✓ | 锥桶 30 个 ✓
预计到位时间：10 分钟
```

**技术实现：**
- 新建 `EnhancedStrategyCard.tsx` 组件
- 新建 `strategyReasonGenerator.ts` 工具函数
  - `generateRecommendationReason()` - 推荐理由生成
  - `calculateMatchRate()` - 匹配度计算
  - `calculateConfidence()` - 置信度计算
  - `generateEffectCheckpoints()` - 效果时间线生成

**业务价值：**
- 从"给建议"升级为"给方案"
- 指挥员拿到的是可执行的完整方案，不是一句空话
- 决策依据透明，建立信任

---

### 3. 移除顶部栏返回按钮

**改进前：**
- 顶部栏常驻"返回总览"按钮
- 容易误操作，策略执行中途退出

**改进后：**
- 顶部栏移除返回按钮
- 只在"执行完毕"阶段的右侧面板显示"返回总览模式"按钮

**业务价值：**
- 防止误操作，策略执行中不会意外退出
- 明确的退出时机：执行完毕后才能返回

---

### 4. 底部时间轴重构（流程节点 + 关键信息）

**改进前：**
- 底部是横向滚动的消息流（CommandCommPanel）
- 显示所有类型消息（系统、AI、指挥、现场、审批、告警）
- 信息密度低，需要翻找关键信息

**改进后：**
- 底部改为流程时间轴（ProcessTimeline）
- 显示 6 个关键里程碑，每个节点带关键信息摘要

**6 个里程碑类型：**

| 里程碑 | 关键信息 |
|--------|---------|
| 拥堵触发 | 拥堵指数 6.8 + 港口积压 70% |
| AI 推荐 | S-02 分流 + 预期 1h 降至 5.2 + 置信度 85% |
| 确认执行 | 王警官、李警官已派出 |
| 资源到位 | 诱导屏已更新 + 分流开始 |
| 当前状态 | 偏差 4% + 实际 5.8 vs 预期 5.5 |
| 预计达标 | 目标 5.2 + 预计 15:30 |

**技术实现：**
- 新建 `ProcessTimeline.tsx` 组件
- 替代原有的 `CommandCommPanel.tsx`

**业务价值：**
- 一眼看到执行进度和关键信息
- 不用翻聊天记录找信息
- 领导站在大屏前能快速判断"走到哪一步了"

---

### 5. 地图图层筛选（6 个图层开关）

**改进前：**
- 地图同时显示所有元素（拥堵路段、人员、设备、路线、车辆）
- 信息过载，视觉杂乱

**改进后：**
- 地图左上角增加图层筛选按钮组
- 6 个图层：全部 / 拥堵路段 / 执行人员 / 设备点位 / 分流路线 / 特殊车辆
- 默认只显示"拥堵路段 + 执行人员"
- 支持多选

**技术实现：**
- 新建 `LayerFilterButtons.tsx` 组件
- commandStore 新增 `mapLayers` 状态
- useCommandMap hook 根据图层状态控制地图元素显示/隐藏

**业务价值：**
- 减少视觉杂乱，聚焦当前关注点
- 指挥员按任务需要切换视图

---

### 6. 视频通话延迟到策略执行后 5 秒

**改进前：**
- 进入指挥模式立即弹出视频/来电
- 打断领导先看态势和选策略

**改进后：**
- 进入指挥模式时不自动弹出
- 策略执行确认后，延迟 5 秒触发视频通话
- 形成顺序：进入指挥模式 → 看态势 → 选策略 → 点执行 → 5 秒后视频接入

**技术实现：**
- 修改 `useIncomingCallHandler.ts`
- 监听策略状态变化（idle → executing）
- 使用 `setTimeout` 延迟 5 秒触发

**业务价值：**
- 不打断决策流程
- 视频接入时机更合理（策略已确认，需要现场协同）

---

### 7. 地图元素与策略执行联动（4 阶段渐进）

**改进前：**
- 地图元素静态显示，不随策略执行变化
- 用户看不出"策略是否在执行、执行到哪一步了"

**改进后：**
- 地图从"态势图"升级为"策略执行图"
- 元素随执行进度分 4 阶段渐进展示

**4 个阶段：**

| 时间 | 展示内容 | 视觉表现 |
|------|---------|---------|
| 立即（0s） | 路线激活 | S376 分流路线从淡虚线 → 亮色动态流线 |
| 3 秒后 | 人员移动 | 王警官、李警官图标开始脉冲发光 |
| 6 秒后 | 设备激活 | 诱导屏图标闪烁，表示内容已更新 |
| 15 秒后 | 结果可视化 | 拥堵热力图颜色变化（红色 → 橙色），排队长度缩短 |

**执行状态视觉编码：**
- 待执行：淡色虚线
- 执行中：亮色动态流线（青色流动）
- 已生效：稳定高亮 + 指标变化标注
- 异常/偏差：红色闪烁 + 偏差标注

**技术实现：**
- 修改 `useCommandMap.ts`
- 监听策略状态变化
- 使用 `setTimeout` 分阶段触发视觉变化
- 添加 CSS 动画类（脉冲、闪烁、流动）

**业务价值：**
- 领导一眼看到"策略在执行、执行到哪一步了、效果怎么样"
- 地图不是静态背景，而是策略执行的实时画面
- 增强现场感和掌控感

---

### 8. 视频源与策略关联（自动切换到相关摄像头）

**改进前：**
- 视频源固定，不随策略变化
- 执行分流策略时，可能显示的是港口摄像头

**改进后：**
- 根据执行的策略类型，自动切换到最相关的视频源
- 显示视觉指示器，说明当前视频源与哪个策略关联

**策略-视频源映射：**

| 策略类型 | 关联视频源 |
|---------|-----------|
| S-02/S-03 分流 | 分流路口摄像头 + 无人机俯拍 |
| S-01/S-13 车道借用 | 进港大道摄像头 |
| S-07 事故处置 | 事故点摄像头 + 现场对讲 |
| S-04 信号灯优化 | 关键路口摄像头 |
| S-08 临时停车区 | 停车区摄像头 |
| S-14 交警增援 | 部署点位摄像头 |

**技术实现：**
- 新建 `useStrategyVideoSync.ts` hook
- 定义策略-摄像头映射表
- 策略执行后 5.5 秒自动切换视频源
- 支持手动覆盖（用户可手动切换）

**业务价值：**
- 视频源和策略匹配，提供相关的现场画面
- 增强情境感知，辅助决策

---

### 9. 自定义策略入口（策略库选择 + 手动新增）

**改进前：**
- 只有 AI 推荐和备选策略
- 如果都不合适，指挥员没有出路

**改进后：**
- 增加"自定义策略"入口
- 支持两种模式：从策略库选择 + 手动新增

**两种模式：**

#### Tab 1：策略库选择
- 显示 15 个标准策略列表
- 每个策略显示：ID、名称、描述、权限等级
- 支持多选组合
- 底部"确认执行"按钮

#### Tab 2：手动新增
- 策略名称（必填）
- 执行动作描述（必填）
- 预期效果说明（选填）
- 底部"添加并执行"按钮

**技术实现：**
- 新建 `CustomStrategyModal.tsx` 组件
- 集成到 SelectingPhasePanel 和 ExecutingPhasePanel

**业务价值：**
- 给指挥员留了应急发挥空间
- 强调"AI 是参谋长，不是指挥官"
- 决策权在人手里

---

### 10. 资源面板状态感知内容切换

**改进前：**
- 资源面板内容固定，不随执行状态变化
- 策略未执行时显示"执行中 6、在途 1、待命 3"没有意义

**改进后：**
- 资源面板内容根据执行阶段动态切换

**三个阶段：**

| 执行状态 | 面板标题 | 显示内容 |
|---------|---------|---------|
| 未执行 | 当前可用资源 | 在岗交警 3 人、可用拖车 2 辆、诱导屏 2 块 |
| 已选未执行 | 策略所需资源 | 需要 2 人 / 可用 3 人 ✓、预计到位时间 10 分钟 |
| 执行中 | 执行资源状态 | 王警官已到位、李警官在途、诱导屏已更新 |
| 执行完毕 | 隐藏或显示释放状态 | 资源释放情况 |

**技术实现：**
- SelectingPhasePanel 显示可用资源概览
- ExecutingPhasePanel 显示执行资源状态
- CompletedPhasePanel 隐藏资源面板

**业务价值：**
- 每个阶段显示的信息都有决策价值
- 不显示无关信息

---

### 11. 流程时间轴增加关键里程碑和信息摘要

**改进前：**
- 底部消息流只显示消息文本
- 缺少结构化的流程节点

**改进后：**
- 时间轴显示 6 个关键里程碑
- 每个里程碑包含：时间、图标、标题、1-2 行关键信息

**视觉设计：**
- 横向布局，圆点连线
- 当前里程碑：大圆点 + 发光效果
- 过去里程碑：实心圆点 + 实线
- 未来里程碑：虚线圆点 + 虚线 + 半透明
- 自动滚动到当前里程碑

**技术实现：**
- 新建 `ProcessTimeline.tsx` 组件
- 替代 `CommandCommPanel.tsx`

**业务价值：**
- 清晰展示执行进度
- 关键信息一目了然

## 四、用户体验提升

### 决策阶段清晰化
- 从"功能堆砌"升级为"阶段引导"
- 每个阶段只看需要的内容，不会迷失

### 信息完整性
- 策略卡片从"建议"升级为"完整方案"
- 包含推荐理由、历史依据、预期效果、资源需求

### 执行可视化
- 地图从"态势图"升级为"策略执行图"
- 4 阶段渐进展示，一眼看到执行进度

### 交互流畅性
- 视频不再打断决策流程
- 地图图层筛选减少信息过载
- 底部时间轴清晰展示关键信息

## 五、业务价值总结

### 对指挥员
1. **决策更有底气**：完整的策略信息，不是拍脑袋
2. **执行更有掌控感**：地图实时反映执行进度
3. **应急更有灵活性**：自定义策略留了发挥空间

### 对领导
1. **一眼看懂进度**：底部时间轴 + 地图联动
2. **信任 AI 判断**：推荐理由透明，历史战绩可查
3. **看到持续进化**：AI 学习摘要，准确率可追溯

### 对系统
1. **从"建议助手"升级为"参谋长"**
2. **从"态势监控"升级为"执行指挥"**
3. **从"功能堆砌"升级为"阶段引导"**

---
