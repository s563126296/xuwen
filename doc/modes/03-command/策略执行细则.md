# 指挥模式 — 策略执行细则

> 版本：V1.0 | 日期：2026-04-10
> 本文档为 PRD-指挥模式.md 的附件，详细定义每个策略的执行规则和多策略并行逻辑

---

## 一、策略作用分层

15个策略按作用环节分为4层：

```text
车辆从哪来          车辆怎么走           车辆到了港口          车辆上船
（源头控制层）      （路径优化层）        （现场管理层）        （运力提升层）

S-06 限流入港      S-02 S376分流        S-01 应急车道借用      S-05 港口增开班次
S-11 货客分时      S-03 G207分流        S-08 临时停车区        S-10 海安新港分流
S-15 预约通行      S-04 信号灯优化       S-13 潮汐车道借用      S-12 冷链车优先
                  S-07 事故快速处置      S-14 交警增援
                  S-09 诱导屏发布
```

核心原则：**不同层的策略天然可并行，同层策略需冲突检测。**

## 二、15个策略详细执行规则

---

### S-01 应急车道借用

**作用层：** 现场管理层
**审批权限：** 🔴 值班领导审批

#### 触发条件
- 进港大道拥堵指数 > 6（重度拥堵）
- 且拥堵持续时间 > 20分钟
- 且排队长度 > 2公里

#### 执行参数
- 借用路段：进港大道[起点]至[终点]（系统根据排队长度自动计算）
- 限速：40km/h
- 应急车辆优先：借用期间应急车辆仍享有绝对优先权

#### 开放时长测算
```text
开放时长 = f(排队长度, 当前流量, 港口消化速率)

公式：
  待消化车辆数 = 排队长度(m) ÷ 平均车间距(8m)
  消化速率 = 港口单位时间可运车辆 - 持续进港流量
  预计消化时间 = 待消化车辆数 ÷ 消化速率

示例：
  排队长度 3000m → 待消化车辆 ≈ 375辆
  港口可运 800辆/时，持续进港 520辆/时 → 消化速率 280辆/时
  预计消化时间 = 375 ÷ 280 ≈ 1.3小时
  建议开放时长：1.5小时（含30%冗余）
```

#### 退出条件（满足任一）
- 拥堵指数回落 ≤ 3 且持续10分钟
- 排队长度 < 500米
- 达到预设开放时长上限
- 出现需要应急车辆通行的紧急事件

#### 效果监测指标
- 进港大道通行速度变化（借用前 vs 借用后）
- 排队长度消减速率
- 应急车道内车辆平均速度（应 ≥ 35km/h）

### S-02 S376省道分流

**作用层：** 路径优化层
**审批权限：** 🟡 值班指挥员确认

#### 触发条件
- 进港大道拥堵指数 > 4
- 且 S376省道当前流量 < 其承载能力的60%（前置条件）

#### 执行参数
- 分流比例：默认30%（可调整范围 10%-50%）
- 诱导方式：诱导屏显示"前方拥堵，建议经S376省道绕行进港"

#### 分流比例测算规则
```text
S376最大承载力 = S376车道数 × 单车道通行能力(约500辆/时)

假设S376为双向4车道（进港方向2车道）：
  进港方向承载力 = 2 × 500 = 1000辆/时
  当前S376流量 = X辆/时（从卡口数据获取）
  可接纳增量 = 1000 × 0.8 - X（保留20%余量给本地交通）

分流比例 = min(可接纳增量 ÷ 进港大道当前流量, 50%)

安全约束：
  S376分流后流量 < S376承载力的80%（保护本地群众出行）
  分流比例上限50%（不能让进港大道流量骤降导致信号灯失衡）

示例：
  S376承载力 1000辆/时，当前流量 300辆/时
  可接纳增量 = 1000 × 0.8 - 300 = 500辆/时
  进港大道当前流量 520辆/时
  分流比例 = min(500 ÷ 520, 50%) = 50%，但受上限约束 → 实际50%
  分流后S376流量 = 300 + 260 = 560辆/时（承载力56%，安全）
```

#### 对本地交通的影响评估
```text
系统自动评估：
  S376本地车辆占比 = S376历史同时段流量 ÷ S376当前流量
  如果分流后S376流量 > 承载力70%：
    → 标记"⚠️ 可能影响S376本地交通"
    → 建议降低分流比例至20%
  如果分流后S376流量 > 承载力80%：
    → 标记"🔴 严重影响本地交通"
    → 不建议执行，或改为G207分流
```

#### 退出条件
- 进港大道拥堵指数回落 ≤ 2.5
- 或 S376流量达到承载力75%（保护性退出）

#### 效果监测指标
- 进港大道流量变化率
- S376流量变化率
- S376本地车辆平均通行时间是否显著增加（>20%则告警）

### S-03 G207国道分流

**作用层：** 路径优化层
**审批权限：** 🟡 值班指挥员确认

#### 触发条件
- 进港大道拥堵指数 > 4
- 且 G207当前流量 < 承载力的60%
- 且 G207无事故/施工（系统自动检测）

#### 执行参数
- 分流比例：默认20%（G207绕行距离较远，分流意愿低于S376）
- 诱导方式：诱导屏 + 导航平台推送

#### 分流比例测算（与S-02同逻辑）
```text
G207可接纳增量 = G207承载力 × 0.8 - G207当前流量
分流比例 = min(G207可接纳增量 ÷ 进港大道当前流量, 30%)

注意：G207分流上限30%（低于S376的50%），因为：
  1. G207绕行距离更远，实际分流效果打折
  2. G207途经更多居民区，对本地影响更大
```

#### S-02和S-03同时执行时的约束
```text
S-02分流比例 + S-03分流比例 ≤ 60%（总分流上限）
进港大道保留至少40%的流量，否则进港大道利用率过低，信号灯配时失效

推荐分配比例：
  S-02(S376) : S-03(G207) = 6 : 4
  例如总分流40% → S376承担24%，G207承担16%
```

#### 退出条件
- 进港大道拥堵指数回落 ≤ 2.5
- 或 G207流量达到承载力70%
- 或 G207出现事故（立即退出）

### S-04 信号灯配时优化

**作用层：** 路径优化层
**审批权限：** 🟡 值班指挥员确认

#### 触发条件
- 进港大道拥堵指数 > 3（轻度拥堵即可启用，低门槛策略）

#### 执行参数
```text
配时调整规则：
  正常配时：进港方向绿灯 40秒 / 交叉方向绿灯 35秒
  拥堵配时方案A（轻度）：进港 50秒 / 交叉 30秒（进港+25%）
  拥堵配时方案B（中度）：进港 60秒 / 交叉 25秒（进港+50%）
  拥堵配时方案C（重度）：进港 70秒 / 交叉 20秒（进港+75%）

选择规则：
  拥堵指数 3-4 → 方案A
  拥堵指数 4-6 → 方案B
  拥堵指数 > 6 → 方案C

约束条件：
  交叉方向绿灯 ≥ 15秒（最低安全通行时间）
  调整范围：仅限进港大道沿线的信号灯（截图中显示共8台）
```

#### 对交叉方向的影响评估
```text
交叉方向延误增加 = (原绿灯时长 - 新绿灯时长) × 周期数 / 小时
  方案A：延误增加约 5秒/周期 → 高峰时段累计延误约3分钟
  方案B：延误增加约 10秒/周期 → 高峰时段累计延误约6分钟
  方案C：延误增加约 15秒/周期 → 高峰时段累计延误约9分钟

系统提示：
  方案B/C执行时标注"交叉方向通行效率降低约X%，建议同步发布诱导信息"
```

#### 退出条件
- 拥堵指数回落 ≤ 2 → 恢复正常配时
- 渐进恢复：方案C→B→A→正常，每级间隔10分钟，避免流量反弹

### S-05 港口增开班次（建议通报类）

**作用层：** 运力提升层
**策略性质：** ⚪ 建议通报（非执行类）

> **重要说明：** 港口管理方不归徐闻县政府管辖，我们无权要求港口增开班次。
> 本策略定位为"数据通报"——系统自动生成运力缺口分析报告，
> 通过正式渠道发送给港口管理方，供其参考决策。港口方是否采纳由其自行判定。

#### 触发条件
- 港口运力饱和度 > 80%
- 且待舶车辆数 > 500辆
- 且预计等待时长 > 3小时

#### 系统生成内容
```text
运力缺口分析报告（自动生成，发送给港口管理方）：

  当前待舶车辆：{X}辆
  今日剩余班次：{N}个，可运载：{M}辆
  运力缺口：{X-M}辆
  预计等待时长：{Y}小时
  
  如增开{Z}个班次，可将等待时间缩短至{W}小时。
  
  附：近4小时到港流量预测（基于售票数据+卡口数据）
```

#### 大屏交互
- 策略卡片标记"建议通报"而非"一键执行"
- 点击"发送通报" → 生成报告 → 发送给港口联络人
- 显示通报状态：已发送 / 港口已阅 / 港口已采纳 / 港口未采纳

#### 注意事项
- 通报频率不宜过高，建议每2小时最多1次
- 通报中附带数据图表，增强说服力

### S-06 限流入港

**作用层：** 源头控制层
**审批权限：** 🔴 值班领导审批

#### 触发条件
- 进港大道拥堵指数 > 6
- 且排队长度 > 3公里（排队已溢出到城区）
- 且路径优化层策略（S-02/S-03/S-04）已执行但效果不足

#### 执行参数
```text
限流方式：在城区主要路口（截图中"城区路口"位置）设卡放行
放行规则：
  每次放行 N 辆车（批次放行）
  放行间隔 = 批次车辆数 × 平均通过时间 × 安全系数

N值测算：
  N = 进港大道剩余通行能力(辆/分钟) × 放行间隔(分钟)
  
示例：
  进港大道通行能力 520辆/时 = 8.7辆/分钟
  设定放行间隔 5分钟
  每批放行 = 8.7 × 5 ≈ 40辆
  即每5分钟放行40辆车进入进港大道

约束条件：
  城区路口排队不得超过3个路口（避免城区瘫痪）
  如城区排队过长 → 降低放行频率 or 启用S-08临时停车区
```

#### 退出条件
- 进港大道拥堵指数 ≤ 4
- 渐进退出：先加大放行批次，观察15分钟，再完全放开

### S-07 事故快速处置

**作用层：** 路径优化层
**审批权限：** 🟡 值班指挥员确认

#### 触发条件
- 系统检测到事故报警（摄像头AI识别/交警上报/报警电话）
- 拥堵归因分析判定为"事故阻断型"

#### 执行参数
```text
响应等级：
  轻微事故（无人伤）：调度1辆拖车 + 1组交警，目标20分钟清障
  一般事故（轻伤）：调度1辆拖车 + 1辆救护车 + 2组交警，目标30分钟
  严重事故（重伤）：调度2辆拖车 + 救护车 + 消防车 + 3组交警

调度逻辑：
  最近拖车位置 → 预计到达时间
  最近交警巡逻位置 → 预计到达时间
  系统自动推荐最快响应方案
```

#### 联动策略
- 事故发生在G207 → 自动禁用S-03（G207分流），标记"目标路段有事故"
- 事故发生在S376 → 自动禁用S-02（S376分流）
- 事故发生在进港大道 → 自动触发S-02+S-03分流

#### 退出条件
- 事故现场清障完毕，道路恢复通行

### S-08 临时停车区启用

**作用层：** 现场管理层
**审批权限：** 🔵 跨部门协调（交通局+城管）

#### 触发条件
- 排队长度 > 5公里
- 或待舶车辆 > 800辆
- 或台风预警导致预计长时间滞留

#### 执行参数
```text
停车区选址规则：
  1. 距离进港大道 < 3公里
  2. 面积 > 5000平方米
  3. 有基本的进出通道
  4. 不影响其他重要交通节点

停车容量估算：
  每辆小型车占地 12.5平方米（2.5m × 5m）
  每辆货车占地 37.5平方米（3m × 12.5m）
  停车区面积 ÷ 加权平均占地 = 可容纳车辆数

示例：
  可用空地 10000平方米
  假设货车占60%、小车占40%
  可容纳 ≈ 10000 ÷ (0.6×37.5 + 0.4×12.5) ≈ 345辆
```

#### 退出条件
- 排队长度 < 1公里 且持续30分钟
- 引导停车区车辆有序回到进港大道排队

### S-09 诱导屏信息发布

**作用层：** 路径优化层
**审批权限：** 🟢 系统可直接执行

#### 触发条件
- 任何策略启动时均可配套执行
- 拥堵指数 > 2 时自动发布拥堵提示

#### 执行参数
```text
信息内容模板库：

拥堵提示类：
  "前方进港大道拥堵{X}公里，预计等待{Y}分钟"
  "当前港口排队约{Z}辆，预计等待{W}小时"

分流引导类：
  "前方拥堵，建议经S376省道绕行进港，预计节省{X}分钟"
  "G207方向畅通，建议绕行"

事故提示类：
  "前方{X}公里处发生事故，占用{Y}车道，请减速慢行"

天气提示类：
  "台风预警：港口预计{X}点停航，建议提前安排"

诱导屏位置（2台）：
  1号屏：城区路口方向（截图中城区路口位置）→ 分流引导
  2号屏：进港大道中段 → 拥堵提示+等待时间
```

#### 退出条件
- 拥堵消除后恢复默认信息（"欢迎来到徐闻，安全驾驶"）

### S-10 海安新港分流（建议通报类）

**作用层：** 运力提升层
**策略性质：** ⚪ 建议通报（非执行类）

> **重要说明：** 海安新港管理方同样不归徐闻县政府管辖。
> 本策略分两部分：诱导屏引导（我方可执行）+ 港口协调（通报类）。

#### 触发条件
- 徐闻港待舶车辆 > 600辆
- 且海安新港运力饱和度 < 60%（有余力接纳）

#### 我方可执行部分
```text
诱导屏引导（🟡 值班指挥员确认）：
  在城区路口诱导屏发布："海安新港当前无排队，车程约20分钟"
  通过高德导航平台推送分流建议（如已接入）
  
  注意：这是信息引导，非强制，司机自主选择是否转港
```

#### 需港口方配合部分（建议通报）
```text
通报内容：
  "徐闻港当前待舶{X}辆，预计等待{Y}小时。
   海安新港当前余力约{Z}辆，建议贵方做好接纳准备。"

通报对象：海安新港管理方
通报方式：系统生成报告 → 发送给港口联络人
```

#### 退出条件
- 徐闻港待舶车辆降至正常水平
- 或海安新港反馈运力饱和度 > 70%

### S-11 货客分时通行

**作用层：** 源头控制层
**审批权限：** 🔴 值班领导审批

#### 触发条件
- 节假日+拥堵指数 > 5
- 且港口运力紧张

#### 执行参数
```text
分时规则：
  客车优先时段：08:00-10:00, 14:00-18:00（高峰时段）
  货车通行时段：06:00-08:00, 10:00-14:00, 18:00-22:00
  
  特殊车辆例外（不受分时限制，全时段通行）：
    - 冷链车：特殊标识放行，不能熄火
    - 危化品运输车：必须走指定通道和指定航班，不受分时限制
      但需提前完成资质报备，按《危险废物转移管理办法》执行
    - 锂电池/铅酸电池类货车：强制走海安新港新能源车辆专班
      （不在徐闻港通行范围内，系统自动识别并引导至海安新港）

效果估算：
  假设高峰时段货车占比 40%（不含冷链车和危化品车）
  限制普通货车后进港大道流量降低约 30%
  客车通过效率提升约 50%（客车上船速度快于货车）
```

#### 退出条件
- 节假日结束
- 或拥堵指数连续4小时 < 3

### S-12 特殊车辆优先通道（建议通报类）

**作用层：** 运力提升层
**策略性质：** ⚪ 建议通报（港口方执行）+ 🟡 我方引导（诱导屏+卡口识别）

> **重要说明：** 港口上船通道的管理权在港口方，我方无法直接开辟优先通道。
> 本策略分两部分：我方负责识别和引导特殊车辆，港口方决定是否开辟优先通道。

#### 触发条件
- 拥堵持续 > 4小时
- 且拥堵区域内冷链车 > 20辆 或 危化品车辆 > 5辆

#### 车辆分类与处置规则
```text
特殊车辆分3类，处置方式不同：

1. 冷链车（不能熄火，货损风险）
   识别方式：车牌识别 + 车辆类型数据库 + 制冷机组外观特征
   我方可做：卡口识别后标记，在大屏地图上高亮位置
   通报港口：建议预留优先上船通道，附冷链车数量和等待时长
   
2. 危化品运输车（安全风险，有资质要求）
   识别方式：危化品运输标识 + 车辆报备数据库
   我方可做：卡口识别后标记，确认资质是否完备
   规则：必须走指定航班和指定码头，不可与普通车辆混装
   通报港口：附车辆数量、品类、资质状态

3. 锂电池/铅酸电池类货车（强制走海安新港）
   识别方式：报备数据 + 车牌识别
   我方可做：在城区路口诱导屏主动引导 "锂电池类货车请走海安新港专班"
   规则：这类车辆不能在徐闻港通行，是硬性规定
   → 如果卡口识别到此类车辆驶向徐闻港，系统自动告警
```

#### 大屏交互
- 地图上用不同颜色标记3类特殊车辆位置
- 冷链车：蓝色图标 | 危化品车：红色图标 | 锂电池类：橙色图标
- 右侧面板显示各类特殊车辆实时数量和等待时长
- "发送通报"按钮 → 生成特殊车辆情况报告发送给港口方

#### 退出条件
- 拥堵区域内特殊车辆全部处置完毕
- 或拥堵完全缓解

### S-13 潮汐车道借用

**作用层：** 现场管理层
**审批权限：** 🔴 值班领导审批

#### 触发条件
- 进港方向拥堵指数 > 7（严重拥堵）
- 且出港方向拥堵指数 < 2（出港畅通，有借用条件）
- 且S-01应急车道借用已执行仍不足

#### 执行参数
```text
借用规则：
  进港大道假设双向6车道（进3出3）
  借用后：进4出2（借出港方向1车道给进港）

安全约束：
  出港方向至少保留2车道（1车道通行+1车道应急）
  出港方向当前流量必须 < 单车道承载力的70%（约350辆/时）
  设置物理隔离（锥桶/水马）

注意：进港大道正在拓宽施工（需求文档提到），
  施工期间可用车道数需要实时确认，
  如果施工导致已经减少到双向4车道（进2出2），
  则借用后变成进3出1，出港方向仅1车道，风险较高
  → 系统需根据施工信息动态调整借用可行性判断
```

#### 退出条件
- 进港方向拥堵指数 ≤ 4
- 或出港方向流量增大到接近承载力
- 渐进退出：先恢复1车道，观察15分钟

### S-14 交警增援部署

**作用层：** 现场管理层
**审批权限：** 🟡 值班指挥员确认

#### 触发条件
- 拥堵指数 > 4
- 且任一路口出现交通秩序混乱（摄像头AI识别或人工上报）

#### 执行参数
```text
增援力量测算：
  拥堵指数 4-6 → 增派2组交警（4人），部署到最堵的2个路口
  拥堵指数 6-8 → 增派4组交警（8人），覆盖进港大道全部关键路口
  拥堵指数 > 8 → 增派6组交警（12人）+ 启动备勤

部署位置优先级（基于截图路网）：
  1. 城区路口（城区进入进港大道的汇入点）
  2. 华四村路口（X699与进港大道交叉口）
  3. 高速入口路口
  4. 港口入口
```

#### 退出条件
- 拥堵指数 ≤ 3 且持续20分钟
- 路口秩序恢复正常

### S-15 预约通行引导

**作用层：** 源头控制层
**审批权限：** 🟢 系统可直接执行

#### 触发条件
- 港口运力饱和度 > 60%（常态化引导）
- 不依赖拥堵触发，是长期运行的策略

#### 执行参数
```text
预约规则：
  通过微信公众号/小程序/导航APP推送预约入口
  按小时时段预约，每时段放号 = 港口该时段运力 × 0.8
  预约车辆在S-06限流时优先放行

效果估算：
  如果预约率达到50%，进港大道流量波动降低约40%
  高峰时段流量从520辆/时降至约400辆/时
```

#### 退出条件
- 常态化运行，不退出
- 根据港口运营数据动态调整每时段放号量

## 三、多策略并行场景

### 场景一：常规港口积压（拥堵指数 4-6）

最常见的日常场景，港口排队车辆多，进港大道开始堵。

```text
并行策略组合（4个🟡，指挥员即可拍板）：
├── 🟡 S-04 信号灯配时优化（方案A/B） ← 5分钟生效
├── 🟡 S-09 诱导屏发布拥堵提示        ← 即时生效
├── 🟡 S-02 S376省道分流（30%）       ← 20分钟生效
└── 🟡 S-14 交警增援（2组）           ← 15分钟生效

冲突检查：无冲突
前置条件：S376当前流量 < 承载力60%
预计综合效果：拥堵指数降低1.0-1.5，约30分钟见效
```

### 场景二：严重港口积压（拥堵指数 6-8，场景一执行15分钟后效果不足）

场景一的升级版，需要领导介入。

```text
在场景一基础上追加：
├── 🟡 S-03 G207国道分流（20%）        ← 路径优化层追加
├── 🔴 S-01 应急车道借用               ← 需领导审批
├── 🔴 S-06 限流入港（每5分钟放行40辆） ← 需领导审批
└── 🔵 S-05 港口增开班次               ← 需港口方配合

冲突检查：
  S-02 + S-03 总分流量 ≤ 60%
  S-06 限流 + S-01 应急车道 = 前端减流 + 后端加速，效果叠加无冲突

预计综合效果：拥堵指数降低2.0-3.0，约45分钟见效
```

### 场景三：节假日高峰（春运/国庆 + 港口积压）

流量高峰叠加港口积压，最复杂的常态场景。

```text
全部4层策略并行：

源头控制层：
├── 🔴 S-11 货客分时通行（高峰限货）
├── 🔴 S-06 限流入港
└── 🟢 S-15 预约通行引导

路径优化层：
├── 🟡 S-04 信号灯配时优化（方案C）
├── 🟡 S-02 S376分流（30%）
├── 🟡 S-03 G207分流（20%）
└── 🟡 S-09 诱导屏全量发布

现场管理层：
├── 🔴 S-01 应急车道借用
├── 🟡 S-14 交警全面增援（6组）
└── 🔵 S-08 临时停车区启用

运力提升层：
├── 🔵 S-05 港口增开班次
├── 🔵 S-10 海安新港分流
└── 🔵 S-12 冷链车优先通道

冲突检查：
  ⚠️ S-11货客分时 + S-12冷链车优先 → 冷链车不受货车限制，标记"特殊放行"
  ⚠️ S-02+S-03 总分流量 ≤ 60%
  ⚠️ S-06限流 + S-15预约 → 预约车辆在限流时优先放行

预计综合效果：拥堵指数峰值控制在5.0以内（不干预可能达8.0+）
```

### 场景四：事故叠加拥堵

```text
事故处置最高优先级 + 路径规避：
├── 🟡 S-07 事故快速处置（最高优先级）
├── 🟡 S-09 诱导屏发布事故提示
├── 🟡 S-04 信号灯调整（减少事故路段车辆涌入）

冲突检查：
  事故在G207 → 自动禁用S-03，标记"不可用：目标路段有事故"
  事故在S376 → 自动禁用S-02
  事故在进港大道 → S-02和S-03同时启用
```

## 四、策略升级与退出路径

### 升级路径（拥堵持续恶化时）

```text
阶段1：拥堵指数 3-4（轻度）
  → S-04信号灯A方案 + S-09诱导屏
  → 观察15分钟

阶段2：拥堵指数 4-6 或 阶段1无效
  → 追加 S-02分流 + S-14交警增援
  → 观察15分钟

阶段3：拥堵指数 6-8 或 阶段2无效
  → 追加 S-01应急车道 + S-06限流 + S-05港口增开（需审批）
  → 观察30分钟

阶段4：拥堵指数 > 8 或 阶段3无效
  → 启动全部策略 + 考虑S-13潮汐车道
  → 持续监控直到缓解
```

### 退出路径（拥堵缓解时）

```text
原则：高风险策略先退出，低风险最后退出，逐级退出避免反弹

退出顺序：
  1. S-13 潮汐车道借用（最高风险，最先退出）
  2. S-01 应急车道借用
  3. S-06 限流入港（恢复自由通行）
  4. S-11 货客分时（恢复货车通行）
  5. S-02/S-03 分流（恢复诱导屏正常信息）
  6. S-04 信号灯（C→B→A→正常，每级间隔10分钟）
  7. S-14 交警增援（最后撤回）
  8. S-09 诱导屏恢复默认信息

每级退出后观察10分钟：
  拥堵指数无反弹 → 继续退出下一级
  拥堵指数反弹 > 0.5 → 暂停退出，重新评估
```


### 5.2 数据驱动的参数优化

```text
积累数据后可自动优化的参数：

1. 分流比例修正
   初期默认：S376分流30%
   积累50次数据后：
     工作日S376实际可分流25%（本地车多，接纳空间小）
     周末S376实际可分流38%（本地车少）
     节假日S376实际可分流42%（本地出行也减少）
   → 系统自动按时段调整默认分流比例

2. 效果预测修正
   初期默认：S-04信号灯调整后通过率提升15%
   积累数据后：
     路口A实际提升18%（信号灯位置好）
     路口B实际提升10%（路口几何结构限制）
   → 系统对不同路口使用不同的效果系数

3. 缓解时间修正
   初期默认：场景一策略组合30分钟见效
   积累数据后：
     港口积压型平均35分钟见效
     流量高峰型平均22分钟见效
     事故阻断型取决于清障时间（平均18分钟）
   → 系统按归因类型给出差异化的时间预测

4. 策略推荐排序修正
   初期：按固定优先级排序
   积累数据后：按历史成功率 × 当前匹配度排序
     成功率 = 该策略历史执行中效果达标的比例
     匹配度 = 当前场景与该策略最佳适用场景的相似度
```

### 5.3 模拟仿真与策略验证

```text
分析模式中的模拟推演功能，为策略优化提供"沙盘"：

1. 事前推演（策略预案验证）
   输入：春运第3天，预计日流量25000辆，港口正常运力
   输出：各时段拥堵预测 + 不同策略组合的效果对比
   价值：提前准备预案，而不是等拥堵发生再临时决策

2. 事后复盘（策略效果分析）
   输入：某次拥堵事件的完整数据（执行档案）
   分析：如果当时选了方案B而不是方案A，效果会如何？
   价值：验证决策质量，积累经验

3. 参数调优（模拟实验）
   输入：S-02分流比例从10%到50%，步长5%
   输出：每个比例下的拥堵指数变化 + S376流量变化
   价值：找到最优分流比例（效果最大化 + 对本地影响最小化）

4. 极端场景测试
   输入：台风停航3天 + 春运高峰 + 进港大道施工
   输出：最大滞留量 + 需要的应急资源 + 最佳策略组合
   价值：为最坏情况准备预案
```

### 5.5 策略优化的两期实现

**初期（规则引擎）：**
- 策略参数固定（分流30%、信号灯方案A/B/C）
- 效果预测基于经验值
- 策略推荐基于归因类型固定匹配
- 人工定期review执行档案，手动调整参数

**后期（ML模型，积累6个月数据后）：**
- 策略参数动态调整（按时段/天气/车型分布）
- 效果预测基于回归模型（输入：当前状态 → 输出：预测效果）
- 策略推荐基于历史成功率 × 场景匹配度排序
- 冲突检测从硬编码规则扩展到基于历史数据发现隐性冲突
