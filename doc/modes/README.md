# 徐闻智慧交通大屏 — 多模式架构

## 模式总览

| 模式 | 文件夹 | 定位 | 优先级 | 触发方式 |
|------|--------|------|--------|---------|
| 总览模式 | 01-overview | 日常值班，全局态势感知 | P0 | 默认模式 |
| 港口模式 | 02-port | 港口运力、海峡通行、船舶动态 | P1 | 手动切换 / 大船靠港提示 |
| 指挥模式 | 03-command | 拥堵疏导、AI决策建议、策略执行 | P0 | 拥堵指数>4自动提示，>6自动切换 |
| 应急模式 | 04-emergency | 台风停航、滞留预测、跨部门协同 | P0 | 台风预警/停航通知自动切换 |
| 分析模式 | 05-analysis | 历史回溯、模拟推演、领导汇报 | P2 | 手动切换 |

## 模式切换智能触发规则

```
拥堵指数 > 4        → 顶部橙色提示，建议切换指挥模式
拥堵指数 > 6        → 自动切换指挥模式 + 推送通知
台风预警发布         → 自动切换应急模式 + 红色横幅
港口停航通知         → 自动切换应急模式
大船靠港（AIS）      → 右下角提示，建议关注港口模式
```

## 迭代路径

- 第一期：总览模式升级 + 指挥模式
- 第二期：应急模式 + 港口模式
- 第三期：分析模式 + 全模式联动优化
