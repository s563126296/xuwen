## 二、封关专项 KPI

> 海南封关后，以下指标成为徐闻口岸的核心考核项，需在大屏显著位置展示。

| KPI | 定义 | 数据源 | 展示位置 | 备注 |
|-----|------|--------|---------|------|
| 入琼车辆指数 | 日均经徐闻入琼车辆数（7 日滑动平均） | 卡口数据 + 港口数据 | Layer 1 顶部 | 联动海南省发布 |
| 平均等待时长 | 车辆从进港到上船的平均耗时 | 港口数据（需真实标签） | 港口模块 | 依赖港口实时数据 |
| 滞港量 | 当前在港等待车辆数 | 港口停车场数据 | 港口模块 | 需新增数据源 |
| 保供车辆保障率 | 生鲜/医疗等保供车辆的优先通行比例 | 港口调度数据 | 港口模块 | 需定义保供车辆标识 |
| 双港协同分流率 | 拥堵时成功分流到另一港口的比例 | 双港调度记录 | 港口模块 | 依赖双港联动 |
| 停航冲击恢复时长 | 从停航结束到恢复正常通行的时间 | 港口 + 卡口数据 | 港口模块 | 事后统计指标 |

**数据依赖说明**：平均等待时长、滞港量、保供车辆保障率需要港口方提供**实时/准实时真实标签**（排队长度真值、实际装载率、发班/延误记录），当前数据源（每天轮播的预测数据）不足以支撑。建议在需求评审阶段明确数据获取方案。

## 三、需求清单（按叙事线分组）

### Layer 1 · 战略感知

#### R-04: 入境车辆分析

- **来源**: 驾驶舱交通主题业务指标.xlsx
- **模块**: 交通管理
- **优先级**: **P0**
- **叙事角色**: 展示徐闻作为陆路口岸的战略地位，入琼车辆是封关核心指标
- **原始描述**: 统计全县出入境车辆及各类型车辆数据情况，各地车辆情况
- **展示要求**:
  - 车辆来源地画像：各省/市车辆占比 TOP10、外地车高峰时段分布
  - 海南方向车辆专项分析：经徐闻入琼车辆趋势（封关核心指标）
  - 区分货运 vs 客运车辆比例，对港口运力分配提供决策支撑
  - **入琼车辆指数**：7 日滑动平均，作为封关后口岸繁忙度核心指标
- **联动**: 点击入琼车辆指数 → 展开港口模块（R-17/R-18/R-19）

#### R-06: 城区交通流量总览

- **来源**: 驾驶舱交通主题业务指标.xlsx
- **模块**: 交通流量分析
- **优先级**: **P0**
- **叙事角色**: 全域交通健康度的一眼总览
- **原始描述**: 全县整体交通流量统计信息
- **展示要求**:
  - 全域交通流量热力图（小时级），不只是数字
  - 同比/环比分析：今日 vs 昨日 vs 上周同期
  - 交通健康指数：综合流量、速度、拥堵多维指标
  - 港口车流关联：港口排队是否导致城区拥堵蔓延
- **联动**: 点击流量数字 → 下钻到 Layer 2 各子面板

#### R-24: 地图拥堵叠加

- **来源**: 智慧交通大屏原型优化功能说明.docx
- **模块**: 交通态势地图（优化）
- **优先级**: **P0**
- **叙事角色**: 地图是 Layer 1 的视觉核心，拥堵着色提供空间感知
- **原始描述**: 高德 API 红色拥堵道路叠加到 GIS 地图
- **展示要求**:
  - 全等级路况着色：通畅绿 / 缓行黄 / 拥堵橙 / 严重拥堵红
  - 拥堵路段点击交互：显示拥堵详情、预测恢复时间
  - 历史拥堵轨迹回放：回放过去 24h 的拥堵演变过程
  - 拥堵蔓延预测可视化：在地图上动态展示拥堵可能的蔓延方向
- **联动**: 点击拥堵路段 → 弹出 R-14 拥堵详情

#### R-25: GIS 态势地图交互

- **来源**: 驾驶舱交通主题业务指标.xlsx
- **模块**: 交通态势地图
- **优先级**: **P0**
- **叙事角色**: 地图基础功能，承载所有空间信息的底座
- **原始描述**: 感知设备标注、设备检索、视频调用、卡口流量、城区交通流量、城市交通态势
- **展示要求**:
  - 多图层切换：设备层 / 流量层 / 拥堵层 / 港口层
  - 框选统计：在地图上框选区域，实时统计该区域流量
  - 时间轴回放：在地图上回放历史交通态势演变
  - 地图智能聚焦：根据当前告警自动定位到关注区域
- **交互**: 见第五节交互体系设计

### Layer 2 · 运行监控

#### R-07: 城市车流量多维分析

- **来源**: 驾驶舱交通主题业务指标.xlsx
- **模块**: 交通流量分析
- **优先级**: **P0**
- **叙事角色**: 高信息密度展示城市车辆动态（入城/出城/在途/活跃/流动）
- **原始描述**: 今日入城/出城流量、在途车、活跃车、近 7 日流动车
- **展示要求**:
  - 入城-出城-在途三线联动分析，构建实时城市车辆存量模型
  - 在途车空间分布（在哪个区域逗留）
  - 流动车类型分解：货车/客车/小汽车比例
  - 在途车异常检测：偏离历史区间时自动告警
- **联动**: 点击在途车 → 展开在途车空间分布热力图

#### R-08: 出入城整体流量趋势

- **来源**: 驾驶舱交通主题业务指标.xlsx
- **模块**: 交通流量分析
- **优先级**: **P0**
- **叙事角色**: 时间维度的流量变化规律
- **原始描述**: 当日/近 7 日/月度出入城整体流量信息及流量增长趋势
- **展示要求**:
  - 三时间维度联动钻取：月度 → 周 → 日 → 小时
  - 趋势预测线：基于历史数据预测未来 7 日流量
  - 区分港口方向 vs 非港口方向的流量趋势
  - 流量季节性模式：节假日/周末/工作日的不同流量模式
- **联动**: 与 R-06 总览联动，下钻展示

#### R-12: 在途车分析

- **来源**: 驾驶舱交通主题业务指标.xlsx
- **模块**: 交通流量分析
- **优先级**: **P1**（DR-04: 差值法偏差大，待校准方案确认后可升级）
- **叙事角色**: 城市运行动态的关键实时指标
- **原始描述**: 当日/7×24h 在途车流量分析
- **展示要求**:
  - 时序曲线叠加历史均值线，异常区间自动高亮
  - 在途车峰值预测：预测今日峰值及到达时间
  - 区分港口在途车 vs 城区在途车
  - 在途车容量预警：接近承载上限时触发预警
- **歧义说明**: 在途车 = 入城卡口过车数 − 出城卡口过车数，存在系统性偏差（绕行/未过卡口车辆），需定义校准方案（AMB-17）

#### R-14: 拥堵路段分析

- **来源**: 驾驶舱交通主题业务指标.xlsx
- **模块**: 交通拥堵分析
- **优先级**: **P1**
- **叙事角色**: 交通治理的核心依据——哪里堵、有多堵
- **原始描述**: 区域内拥堵路段分析、高频拥堵路段分析
- **展示要求**:
  - 拥堵路段分级排名：按拥堵频次、持续时长、影响范围三维度排名
  - 拥堵传播链路：一个路段拥堵如何影响上下游路段
  - 拥堵路段与港口排队的关联分析
  - 拥堵根因分析：结构性（容量不足）vs 偶发性（事故/施工）
- **联动**: 拥堵路段点击 → 在 GIS 地图上定位并展示详情

#### R-15: 拥堵时间分析

- **来源**: 驾驶舱交通主题业务指标.xlsx
- **模块**: 交通拥堵分析
- **优先级**: **P1**
- **叙事角色**: 拥堵的时间规律——什么时候堵
- **原始描述**: 区域内拥堵路段分析、高频拥堵时间段分析
- **展示要求**:
  - 拥堵时间热力图（24h × 7 天）
  - 拥堵持续时间分布：短时拥堵（<30min）vs 长时拥堵（>2h）占比
  - 港口方向拥堵的潮汐规律分析
  - 拥堵预测时间窗口：预测未来 2 小时各路段拥堵概率

#### R-16: 重点路段道路承载

- **来源**: 驾驶舱交通主题业务指标.xlsx
- **模块**: 重点路段承载
- **优先级**: **P1**（DR-04: 重点路段定义待确认 AMB-04）
- **叙事角色**: 容量监控——路还能不能扛得住
- **原始描述**: 行驶车辆数、当日车辆数量趋势、近 10 天车辆高峰时间分布
- **展示要求**:
  - 路段承载率 = 当前车流量 / 路段设计容量
  - 承载趋势预警：承载率接近 80% 黄色预警，接近 100% 红色预警
  - 多路段承载力对比雷达图
  - What-if 分析：模拟不同流量场景下的路段表现
- **歧义说明**: "重点路段"定义标准待确认（AMB-04）——固定路段清单 vs 动态识别

#### R-17: 港口进出港车辆

- **来源**: 驾驶舱交通主题业务指标.xlsx
- **模块**: 港口交通
- **优先级**: **P0**
- **叙事角色**: 港口核心指标——多少车在进出
- **原始描述**: 港口车辆进出港统计，基于徐闻港一天车流数据预测到每小时
- **展示要求**:
  - 实际 vs 预测对比（如有实际数据）
  - 进出港车辆类型分解：大车/小车/新能源车
  - 进港 vs 出港流量差异分析，反映运力瓶颈方向
  - 港口车辆排队时长估算：基于进港流量和运力推算等待时间
- **数据说明**: 当前港口数据为每天轮播的预测数据（AMB-01），需推动港口方提供实时数据

#### R-18: 港口运力

- **来源**: 驾驶舱交通主题业务指标.xlsx
- **模块**: 港口交通
- **优先级**: **P0**
- **叙事角色**: 运力监控——船够不够用
- **原始描述**: 船舶运载能力，根据排班表和船舶基本信息计算
- **展示要求**:
  - 运力-需求匹配度分析：当前排队车辆 vs 可用运力
  - 运力利用率 = 实际装载 / 最大装载
  - 电车/油车/大车的分类型运力分配
  - 运力缺口预警：预测需求超过运力时提前 2 小时预警

#### R-19: 港口拥堵状态分级

- **来源**: 智慧交通大屏原型优化功能说明.docx

  - 港口拥堵与城区拥堵联动分析
  - 拥堵等级预测：基于当前趋势预测未来 30 分钟变化
- **歧义说明**: 5 级分级阈值定义待确认（AMB-03）——高德标准 vs 自定义

## 四、创新需求提案

### I-01: 跨海交通态势一张图

- **优先级**: P2
- **描述**: 在 GIS 地图上构建徐闻-海口跨海交通全景图，整合徐闻港口排队、海上船舶位置、海口港口接收三方数据
- **预期价值**: 全国唯一的陆海联运交通态势感知，海南封关标杆示范
- **依赖数据**: ✅ 徐闻港进出港车流/船舶排班 | ❌ 船舶 AIS 实时位置、海口新海港数据、海上气象数据
- **可行性**: 中 — 技术可行，主要障碍是跨省数据对接
- **风险**: 海南方数据共享的行政协调

### I-02: 港口智慧排队与预约通行

- **优先级**: P1
- **描述**: 基于历史数据和实时流量预测未来 2h 排队长度和等待时间，提供预约通行时段建议
- **预期价值**: 节假日排队时间缩短 30%+
- **依赖数据**: ✅ 卡口流量/港口进出港 | ❌ 停车场实时车位、车辆 GPS 轨迹、预约系统
- **可行性**: 高 — 核心算法可用现有数据训练
- **风险**: 需政策配合和用户习惯培养

### I-03: AI 交通态势推演引擎

- **优先级**: P3
- **描述**: 场景推演——「如果台风导致停航 6h」「如果春节车流翻倍」「如果某路段封闭」，自动推演全域影响
- **预期价值**: 为交通管理者提供决策沙盘
- **可行性**: 中 — 需交通仿真引擎集成，开发量大
- **风险**: 模型精度依赖大量校准数据

### I-04: 港口运力智能匹配引擎

- **优先级**: **P1**（从 P0 调整，需先获取港口真实标签）
- **描述**: 实时计算排队车辆→可用运力的匹配度，预测未来 4h 供需缺口，建议加开/调整班次
- **预期价值**: 运力利用率提升 15-20%
- **依赖数据**: ✅ 港口进出港车流/船舶排班 | ❌ 船舶实时装载率、车辆预约数据、实时排队长度
- **可行性**: 高 — 核心算法基于供需匹配优化
- **风险**: 需港口方排班调整权限或信息推送能力

### I-05: 交通碳足迹追踪

- **优先级**: P2
- **描述**: 基于车流量和车型结构估算全域交通碳排放，区分港口运输和城市交通碳排放
- **预期价值**: 响应双碳政策
- **可行性**: 高 — 碳排放计算公式成熟

### I-06: 多模态交通融合分析

- **优先级**: P1
- **描述**: 构建港口-城区-公路一体化交通分析模型，量化三者相互影响
- **预期价值**: 全局优化视角
- **可行性**: 中 — 需构建交通网络图模型

### I-07: 节假日交通压力智能预案

- **优先级**: P1
- **描述**: 基于历年节假日数据构建压力预测模型，提前 7 天预测高峰日/高峰时段/峰值流量，自动生成分级应急预案
- **预期价值**: 节假日管理从经验驱动升级为数据驱动
- **可行性**: 高 — 时间序列预测 + 规则引擎

### I-08: 智能告警与决策辅助中心

- **优先级**: P1
- **描述**: 统一告警中心，汇聚所有模块告警，按影响范围和紧急程度排序，附带 AI 生成的处置建议
- **预期价值**: 从"到处看数据"升级为"系统主动推送关键信息+建议"
- **可行性**: 高 — 规则引擎 + LLM 生成建议
- **风险**: AI 建议准确性需持续优化

### I-09: 交通经济指数

- **优先级**: P2
- **描述**: 综合`港口吞吐量`+`车流量`+`车型结构`生成反映经济活跃度的指数曲线
- **预期价值**: 交通数据赋能经济分析

### I-10: 数字孪生交通可视化

- **优先级**: P3
- **描述**: 在港口/城区核心路段构建三维数字孪生场景
- **可行性**: 低 — 需大量三维建模和高性能渲染引擎
- **备注**: 远期规划

## 七、非功能需求

### 7.1 数据质量与 SLA

| 指标 | 要求 | 监控方式 |
|------|------|---------|
| 接口延迟 | 实时数据 < 5s，聚合数据 < 30s | 请求耗时日志 |
| 数据缺失率 | 核心指标 < 1%，辅助指标 < 5% | 定时对账任务 |
| 外部 API 降级 | 高德/天气 API 中断时使用缓存数据，标注"数据延迟" | 心跳检测 + 降级开关 |
| 设备在线率 | 感知设备在线率 > 95% | 设备心跳监控 |

### 7.2 处置闭环

> 当前大屏定位为**展示型驾驶舱**（非指挥调度系统），处置闭环预留接口但不作为 MVP 要求。

- 告警确认：告警产生后需人工确认，记录确认人和时间
- 人工覆盖：AI 预测/建议支持人工覆盖，覆盖原因留痕
- 审计日志：所有关键操作（预警触发、调度建议、人工覆盖）留存 180 天

### 7.3 权限与合规

| 项目 | 要求 |
|------|------|
| 车牌脱敏 | 展示车牌时对非授权用户脱敏（粤A***89） |
| 视频访问 | 实时视频调用需授权，访问记录留存 |
| 第三方 API 授权 | 高德/天气等 API 使用政务版授权，确认商用合规 |
| 数据留存 | 原始数据留存 1 年，聚合数据留存 3 年 |

### 7.4 性能要求

| 场景 | 要求 |
|------|------|
| 大屏首屏加载 | < 3s（局域网环境） |
| 数据刷新间隔 | 实时数据 5min，聚合数据 1h |
| 地图渲染 | 1000+ 标注点不卡顿 |
| 并发支持 | 大屏 + 3 个管理终端同时访问 |
