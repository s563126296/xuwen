## 2. 产品概述

### 2.1 版本定位

v2.3 是 v2.2（统计分析补全）之后的**AI可解释性增强版本**，聚焦 AI 策略模式的交互能力，让用户能够理解 AI 的决策逻辑、追踪推理路径、检索策略关系，建立对 AI 系统的信任。

### 2.2 核心问题

| 问题 | 现状 | 目标 |
|------|------|------|
| 看不懂AI为什么推荐 | 决策树只能展开/折叠，无法追踪推理链条 | 点击策略节点高亮完整决策路径 |
| 不理解策略关系 | 知识图谱的边没有详细解释 | 点击边查看互斥/联动的具体原因 |
| 找不到目标节点 | 50+节点无搜索能力 | 搜索框+高亮+导航 |
| 节点信息不足 | 只有浏览器原生tooltip | 富详情面板展示完整元数据 |

### 2.3 产品目标

1. 让AI决策过程"可解释"——用户能追踪从问题到策略的完整推理路径
2. 让策略关系"可理解"——互斥/联动规则有清晰的解释和上下文
3. 让信息"可检索"——大量节点中快速定位目标

### 2.4 目标用户

| 用户角色 | 核心诉求 | 使用频率 |
|---------|---------|---------|
| 指挥员 | 理解AI推荐逻辑，建立信任 | 每次使用AI策略时 |
| 交通管理领导 | 审查AI决策是否合理 | 周/月 |
| 系统运维人员 | 调试和优化策略规则 | 按需 |

### 2.5 成功指标

- AI策略模式4项功能100%完成
- 整体功能完成率：93% → 95%
- 用户能在3次点击内找到任意策略的决策路径

### 5.3 交互规则

| 场景 | 规则 |
|------|------|
| 节点详情 vs 边详情 | 互斥：点击节点关闭边详情，点击边关闭节点详情 |

| 搜索时点击节点 | 保留搜索状态，同时显示路径高亮和节点详情 |
| 筛选后节点隐藏 | 不满足筛选条件的节点置灰（opacity: 0.2），不移除 |
| 点击空白区域 | 取消路径高亮，关闭详情面板，保留搜索状态 |

### 5.4 异常场景处理

| 异常 | 处理方式 |
|------|---------|
| 搜索无结果 | 搜索框下方显示"未找到匹配节点"，所有节点保持正常显示 |
| 节点无详情数据 | 详情面板显示已有字段，缺失字段显示"—" |
| 边无说明文本 | 边详情面板"原因"字段显示"暂无说明" |
| 决策树层级过深 | 路径高亮时自动滚动/平移视图，确保完整路径可见 |

### 6.2 路径计算算法

```typescript
// 从选中节点向上追溯到根节点
function computePath(tree: TreeNode, targetId: string): string[] {
  const path: string[] = [];
  function dfs(node: TreeNode, currentPath: string[]): boolean {
    currentPath.push(node.id);
    if (node.id === targetId) {
      path.push(...currentPath);
      return true;
    }
    if (node.children) {
      for (const child of node.children) {
        if (dfs(child, currentPath)) return true;
      }
    }
    currentPath.pop();
    return false;
  }
  dfs(tree, []);
  return path;
}
```

| 节点详情 | 点击节点后右侧面板展示完整信息（类型/名称/命中/成功率/版本/效果） |
| 边详情 | 点击知识图谱中的边，面板展示关系类型+原因+触发条件 |
| 搜索高亮 | 输入关键词后 200ms 内高亮匹配节点，显示结果计数 |
| 搜索导航 | 点击上/下按钮在结果间跳转，视图自动居中到当前结果 |
| 筛选 | 拖动成功率滑块，低于阈值的节点置灰 |
| 边悬停 | 鼠标悬停在边上 300ms 后显示Tooltip |
| 快捷键 | Esc关闭面板，Ctrl+F聚焦搜索 |
| 互斥规则 | 节点详情和边详情面板互斥，不会同时打开 |
