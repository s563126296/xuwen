# 徐闻县智慧交通大屏系统

## 核心目标

为徐闻县（中国大陆唯一通往海南的陆路口岸）开发智慧交通大屏，支持 5 种模式：总览、指挥、应急、港口、分析。

## 技术栈

React 18 + TypeScript + Vite + Zustand + Recharts + 高德地图 JS API，1920x1080 大屏，transform:scale 自适应。

## v2.0 迭代规划（新需求）

**规划文档：** `doc/v2.0-iteration-plan.md`

### 核心需求变更

1. **应急与指挥模式合并**
   - 业务上强调"指挥"，轻"应急"
   - 应急业务过于复杂，实际操作难以通过系统完成
   - 方案：指挥模式吸收应急核心功能（特殊车辆追踪、物资需求、预案快捷入口）
   - 工作量：10-15 天

2. **突出 AI 能力展示**
   - 总览模式：动态预测、主动预警、AI 学习能力展示
   - 指挥模式：多因素归因分析、策略推荐优化、效果预测可视化
   - 工作量：7-9 天

3. **总览模式增加虚拟人**
   - 主动播报：系统启动、异常检测、定时播报
   - 对话交互：支持 20-30 个常见问题（规则匹配）
   - 技术方案：Web Speech API (TTS) + 规则匹配引擎
   - 工作量：9-12 天

**总工作量：26-36 天（约 5-7 周）**

## 验收检查清单

### 面板滚动与遮挡检查
- 所有面板卡片必须使用 `overflow: hidden` + `display: flex` + `flexDirection: column` 外层结构
- 标题行必须加 `flexShrink: 0`，防止被压缩
- 内容区必须使用 `flex: 1` + `minHeight: 0` + `overflowY: auto`，确保内容可滚动
- 修改任何面板高度/flex 比例后，必须检查相邻面板是否被挤压遮挡
- 使用 absolute 定位时，必须精确计算 top/bottom 值，修改高度后同步调整相邻元素

### Mock 数据真实性
- 车牌号使用真实格式（粤G·K7823），不用连号/重复数字（12345、8888、1111）
- 手机号使用真实格式（13726581903），不用连号（13900139001）
- 人名、地名、时间等数据要符合实际场景，不能一眼看出是假数据
- 车牌省份字母要合理：粤（广东）、琼（海南）、桂（广西）为主
