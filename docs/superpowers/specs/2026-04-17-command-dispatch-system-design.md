# 指挥模式调度交互体系设计

> 日期：2026-04-17
> 状态：待评审

---

## 一、设计目标

将指挥模式从"看消息 + 点策略"升级为**完整的指挥调度系统**，覆盖四个维度：

| 维度 | 当前状态 | 目标 |
| ---- | -------- | ---- |
| 空间（地图） | 高德底图 + 静态路线标注 | 人员位置 + 来电弹窗 + 拥堵热力 + 车流粒子 |
| 通信（聊天） | 只读事件时间线 | 独立聊天浮窗 + 按人员分组 + 快捷回复 + 主动发起通话 |
| 流程（策略） | 流程条 + 消息关联 | 步骤详情留痕 + 关键动作记录 + 执行报告 |
| 资源（调度） | 无 | 当前策略关联的人员/物资实时展示 |

### 2.3 来电弹窗

当 `commandFeed` 中出现 `icon: 'phone'` 类型消息时：

1. 对应人员 Marker 状态变为 `calling`（红色脉冲）
2. 地图上弹出来电通知：

```
┌────────────────────────────────┐
│ 📞 来电                        │
│ 李四 · 交警一队                 │
```

4. 点击"接听语音" → 聊天窗口自动打开该人员对话，显示"语音通话中"状态
5. 通话结束后，在 commandFeed 追加一条 `type: 'command'` 消息："已接通李四视频通话，时长 2 分钟"

### 2.4 地图视觉增强

在高德底图（暗色主题）上叠加大屏视觉效果：

**拥堵热力渐变：**
- 替换当前的单色红线，改为路段宽度 + 颜色渐变
- 根据 `congestionIndex` 动态映射：
  - 0-2：绿色（#2ED573），宽度 4px
  - 2-4：黄色（#F5A623），宽度 6px
  - 4-6：橙色（#FF8C00），宽度 8px
  - 6+：红色（#FF4757），宽度 10px，带发光效果
- 使用 `AMap.Polyline` 的 `strokeWeight` 和 `strokeColor` 动态更新

**车流粒子动画：**
- 沿进港大道路径移动的小圆点（代表车辆）
- 使用 `AMap.Marker` + `setInterval` 更新位置
- 速度根据拥堵程度变化（畅通时快，拥堵时慢/停）
- 颜色：普通车辆青色，危化品橙色
- 数量：5-8 个粒子循环

**路段脉冲效果：**
- 当策略执行中，目标路段（如 S376）显示青色脉冲边框
- 使用 `AMap.Polyline` 的 `strokeOpacity` 动画（0.3 → 1.0 循环）
- 旁边显示文字标签："S376 分流执行中"

**人员轨迹线：**
- `status: 'moving'` 的人员，显示从当前位置到目标位置的虚线路径
- 绿色虚线，`strokeDasharray: [10, 5]`

**信号灯状态标注（Mock）：**

**诱导屏位置+内容（Mock）：**

## 三、通信聊天窗口

### 3.1 窗口形态

**独立浮动窗口**，位于地图区域右下角，可最小化/关闭。

**尺寸：** 320px 宽 × 400px 高
**位置：** 视频面板左侧（视频面板右下角 260px，聊天窗口在其左侧 320px）

**最小化状态：** 折叠为一个图标按钮（💬 + 未读数 badge）

### 3.2 窗口结构

```
┌──────────────────────────────┐
│ 💬 指挥通信    [—] [×]        │
├──────────────────────────────┤
│ 联系人列表                    │
│ ┌──────────────────────────┐ │
│ │ 👤 张三 · 执行中  ●未读   │ │
│ │ 👤 李四 · 待命           │ │
│ │ 👤 王五 · 移动中         │ │
│ │ 🤖 AI 分析              │ │
│ │ 📋 系统消息              │ │
│ └──────────────────────────┘ │
├──────────────────────────────┤
│ 对话区域（选中某人后显示）     │
│ ┌──────────────────────────┐ │
│ │ 张三 15:27               │ │
│ │ 已到达路口 A，开始引导分流  │ │
│ │                          │ │
│ │        我 15:28          │ │
│ │        收到，注意安全      │ │
│ │                          │ │
│ │ 张三 15:32               │ │
│ │ 📷 现场照片：分流正常通行  │ │

├──────────────────────────────┤

│ [收到] [已协调] [继续执行]     │
│ ┌──────────────────────┐     │
│ │ 输入消息...    [📞][📹]│     │
│ └──────────────────────┘     │
└──────────────────────────────┘
```

### 3.3 联系人列表

**数据来源：** `FIELD_PERSONS` 数组 + AI 分析 + 系统消息

**分组显示：**
- 交警一队（张三、李四）
- 拖车公司（王五）
- AI 分析
- 系统消息

**状态标识：**
- 绿色圆点：在线/空闲
- 青色圆点：执行中
- 黄色圆点：移动中
- 红色圆点 + 脉冲：请求通话
- 灰色圆点：离线

**未读消息：** 红色 badge 显示未读数量

### 3.4 对话区域

**消息气泡样式：**
- 对方消息：左对齐，暗色背景 `rgba(13,27,42,0.8)`
- 我方消息：右对齐，青色背景 `rgba(0,208,233,0.15)`
- 系统消息：居中，灰色小字
- 照片消息：缩略图 + [查看] 按钮
- 通话消息：通话图标 + 时长

**数据来源：** 从 `commandFeed` 中按 `source` 筛选，映射到对应联系人的对话。

### 3.5 快捷回复

**固定模板（4 个）：**

| 按钮 | 消息内容 | 场景 |
| ---- | -------- | ---- |
| 收到 | 收到，已知悉 | 确认收到现场反馈 |
| 已协调 | 已协调资源，请等待 | 回复支援请求 |
| 继续执行 | 继续按计划执行 | 确认策略继续 |
| 需支援 | 已记录，正在协调支援 | 回复紧急请求 |

**点击后：**
- 往 `commandFeed` 追加一条 `type: 'command', source: '指挥员'` 消息
- 对话区域实时显示新消息

### 3.6 主动发起通话

**输入栏右侧两个按钮：**
- 📞 语音通话 → 点击后追加一条 "发起语音通话" 消息，人员 Marker 显示通话状态
- 📹 视频通话 → 点击后 MapVideoDock 切换到通话模式

## 四、策略流程留痕

### 4.1 关键动作记录

策略执行过程中，只有**关键节点**记录到流程详情，非关键消息只在聊天窗口显示。

**关键节点定义：**

| 步骤 | 关键动作 | 记录内容 |
| ---- | -------- | -------- |
| 1. 策略确认 | 指挥员确认执行 | 确认人、确认时间、策略 ID |
| 1. 策略确认 | 领导审批（如需） | 审批人、审批时间、审批结果 |
| 2. 指令下发 | 指令单生成 | 指令编号、推送对象、推送时间 |
| 3. 现场执行 | 人员到达 | 到达人、到达时间、到达位置 |
| 3. 现场执行 | 开始执行 | 执行人、开始时间 |
| 3. 现场执行 | 请求支援（如有） | 请求人、请求内容、响应结果 |
| 4. 效果验证 | 数据达标 | 拥堵指数变化、达标时间 |
| 4. 效果验证 | 策略完成 | 完成时间、总用时、效果评估 |

**非关键消息（不记录到流程详情）：**
- "正在前往中"
- "路上有点堵"
- "收到"
- 日常沟通消息

### 4.2 流程步骤详情展开

点击 StrategyFlowBar 的某个步骤后，在流程条下方展开一个详情区域：

```
┌─────────────────────────────────────────────────────────┐
│ [1.策略确认✓] → [2.指令下发✓] → [3.现场执行●] → [4.效果验证○] │
├─────────────────────────────────────────────────────────┤
│ 步骤 3 详情：现场执行                                     │
│ • 15:26 张三收到指令                                      │
│ • 15:27 张三到达 S376 路口，开始引导分流                    │
│ • 15:32 张三发来现场照片                                   │
│ • 15:42 李四请求视频通话（应急车道违停）→ 已接听              │
└─────────────────────────────────────────────────────────┘
```

### 4.3 执行报告（本期不做，留接口）

策略执行完成后（`status: 'done'`），可生成一份执行报告：
- 策略基本信息
- 关键节点时间轴
- 效果数据（拥堵指数变化曲线）
- 资源使用情况
- 总用时

本期只预留数据结构，不做 UI。

## 十、风险和约束

1. **高德地图 Marker 性能**：人员标记 + 车流粒子 + 热力图同时渲染，可能影响帧率。建议粒子数量控制在 8 个以内，人员标记不超过 10 个。
2. **聊天窗口与底部面板的布局冲突**：聊天窗口浮在地图上方，需要确保不遮挡关键地图信息。最小化按钮提供快速折叠能力。
3. **来电弹窗的时机**：需要在 `commandFeed` 中检测 `icon: 'phone'` 消息，但 Mock 数据是预置的。实现时需要对 f14（李四请求通话）做特殊处理，或者在策略执行流程中触发。
4. **坐标精确度**：人员坐标使用 Mock 固定点，与实际道路可能有偏差。后期接入 GPS 后替换。
