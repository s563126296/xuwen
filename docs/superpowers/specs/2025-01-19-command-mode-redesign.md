# 指挥模式重新设计方案

**日期**：2025-01-19  
**版本**：V3.0  
**状态**：设计完成，待实施

---

## 背景

当前指挥模式（V2.1）存在两个核心问题：

1. **缺乏指挥感**：进入指挥模式后视觉上跟总览模式差不多，没有"紧急状态需要立刻决策"的氛围
2. **决策链断裂**：6 步决策链（态势→归因→影响→策略→预测→跟踪）在界面上是散落的独立卡片，看不出因果关系和流程

用户反馈："整体感觉，看不出这个指挥大屏的价值"

---

## 设计目标

1. **强化指挥氛围**：让用户一进来就感觉"这是紧急状态，需要立刻行动"
2. **可视化决策链**：6 步流程在视觉上连贯，能看出因果关系
3. **地图为核心**：地图不是背景板，而是每一步决策的视觉焦点
4. **调度中心风格**：地图 + 实时数据流 + 策略状态板 + 执行跟踪

## 模块详细设计

### 1. 指挥摘要条（48px）

**位置**：Header 下方，top: 80px

**布局**：
```
┌────────────────────────────────────────────────────────────────┐
│ [图标] 进港大道重度拥堵 6.5 · 排队3.2km · 持续42min │ 建议：S376分流+应急车道 │ 预计45分钟缓解 [返回] │
└────────────────────────────────────────────────────────────────┘
```

**左区（40%）**：
- Lucide AlertTriangle 图标（红/橙/黄/绿，随等级变化）
- 拥堵概况：`{路段名}{等级} {指数} · 排队{距离} · 持续{时间}`
- 字号：13px，加粗

**中区（40%）**：
- 推荐策略组合：`建议：{策略1} + {策略2}`
- 来自策略推荐引擎的 top 2 策略
- 字号：12px，青色

**右区（20%）**：
- 预计缓解时间：`预计 {时间} 缓解`
- 港口切换按钮（徐闻港/海安新港）
- 返回总览按钮
- 字号：11px

**背景色**：
- 重度拥堵（指数 > 6）：`rgba(220, 38, 38, 0.15)` 暗红渐变
- 中度拥堵（指数 4-6）：`rgba(249, 115, 22, 0.15)` 暗橙
- 轻度拥堵（指数 2-4）：`rgba(245, 158, 11, 0.15)` 暗黄
- 道路通畅（指数 < 2）：`rgba(16, 185, 129, 0.15)` 暗绿

**动画**：
- 重度/严重拥堵时有呼吸闪烁效果（opacity 0.15 ↔ 0.25，2s 循环）

### 2. 决策链步骤条（40px）

**位置**：指挥摘要条下方，top: 128px

**布局**：
```
┌────────────────────────────────────────────────────────────────┐
│ ① 态势感知 ──→ ② 归因分析 ──→ ③ 影响评估 ──→ ④ 策略推荐 ──→ ⑤ 效果预测 ──→ ⑥ 执行跟踪 │
└────────────────────────────────────────────────────────────────┘
```

**步骤状态**：
- **已完成**：绿色 ✓ + 绿色文字 + 绿色连接线
- **当前步骤**：青色高亮 + 脉冲动画 + 青色连接线
- **未到步骤**：灰色 + 灰色虚线

**交互**：
- 点击步骤 → 左侧面板切换内容 + 地图图层切换
- 策略执行后自动推进步骤（①→②→...→⑥）

**样式**：
- 步骤编号：圆形徽章，直径 24px
- 步骤文字：12px
- 连接线：2px 实线/虚线，长度自适应

### 5. 地图图层随步骤切换

## 视觉紧迫感设计

### 进入指挥模式时
- 0.3s 过渡动画：背景从 `#0A0F19` 渐变为 `#0D0812`（略带暗红）
- 指挥摘要条从顶部滑入
- 决策链步骤条从左到右依次显示（0.1s 间隔）

### 重度拥堵时
- 指挥摘要条呼吸闪烁（opacity 0.15 ↔ 0.25，2s 循环）
- 拥堵指数大字号（36px）+ 红色 + 加粗
- 等级标签有红色背景色块

### 策略执行时
- 执行按钮变为「执行中...」+ 禁用
- 进度条推进动画（0.5s ease）
- 拥堵指数逐步下降（2s 后降到中间值，4s 后降到预测值）
- 警告条背景色从红渐变为橙/黄/绿

## 设计原则

1. **YAGNI**：第一批只做框架，不做地图深度交互
2. **渐进增强**：先跑通决策链流程，再补充地图交互
3. **数据驱动**：所有内容从 Store 读取，不硬编码
4. **视觉一致性**：沿用总览模式的色彩体系和字号规范
