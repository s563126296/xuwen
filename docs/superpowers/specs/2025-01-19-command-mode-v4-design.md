# 指挥模式设计方案 V4.0

**日期**：2025-01-19
**版本**：V4.0（基于用户故事深度讨论重新设计）
**状态**：设计中

---

## 一、核心定位

**指挥模式 = 拥堵处置的通信指挥中枢**

不是"拥堵分析报告"，不是"6步决策Tab"，而是：
- **地图为主操作界面** — 点击拥堵路段聚焦，调出视频，派出无人机
- **AI给方案，人来拍板** — AI分析归因+推荐策略，指挥员选择执行
- **指令下达+双向通信** — 生成指令单推送责任人，接收现场反馈
- **实时监控+闭环调整** — 地图实时更新，效果不好可追加策略

## 二、用户故事 V2.0（完整版）

### 第1幕：发现拥堵，切换模式

小王在总览模式值班，AI摘要条变红：「进港大道重度拥堵 6.5，建议启动S376分流」。点击「执行」，自动切换到指挥模式。

### 第2幕：看清堵况

进入指挥模式后：

**地图主屏（70%面积）：**
- 进港大道从北到南红色渐变，排队3.2km可视化
- 点击拥堵路段 → 地图聚焦放大，显示详细信息：
  - 拥堵车辆：1200辆（冷链车47辆、危化品车3辆）
  - 持续时间：42分钟
  - 预测趋势：未来30分钟约300辆车驶入，拥堵可能加剧
- 天气信息叠加：当前天气状况对策略选择的影响提示

**视频面板（右下角）：**
- 自动调出拥堵路段沿线摄像头画面
- 可切换查看其他摄像头
- 可点击放大全屏查看

**无人机调度：**
- 点击「派出无人机」→ 地图显示飞行动画 → 到达后自动切换到俯拍画面
- 无人机画面显示在视频面板中

**历史参考：**
- 显示"上次类似拥堵"的信息：上周五同时段，用了S376分流+信号灯优化，45分钟缓解

**实时更新：**
- 拥堵指数、车辆数量每30秒刷新

### 第3幕：选择策略

**右侧策略栏，AI分析后给出方案：**

每个方案包含：
- 策略名称 + 执行权限（🟢直接执行 / 🟡需确认 / 🔴需审批）
- 预计效果 + 缓解时间
- 执行要求 + 责任人
- 紧急程度标记（普通/紧急/特急）

**AI智能提示：**
- 策略冲突检测：如S376正在施工，自动排除S376分流并说明原因
- 资源占用提示：执行方案需要的交警人数 vs 当前在岗人数
- 组合效果预测：方案1+方案3同时执行的预测效果

**小王决策：**
- 方案3（诱导屏）立即执行
- 方案1（S376分流）确认执行
- 方案2（应急车道）提交领导审批

### 第4幕：执行指令

**执行方案3（诱导屏发布，🟢直接执行）：**
1. 点击「执行」→ 弹出内容编辑面板
2. AI预填文案：「前方拥堵3.2km，建议绕行S376省道」
3. 小王确认 → 点击「发布」
4. 生成指令单（编号、任务、责任人、时间）
5. 推送给责任人
6. 地图上诱导屏图标变绿

**执行方案1（S376分流，🟡需确认）：**
1. 点击「确认执行」→ 弹出分流方案详情
2. 地图高亮分流路线 + 标注需调整的信号灯路口
3. 小王点击「下达指令」
4. 生成指令单（含具体信号灯配时参数、交警部署要求）
5. 推送给交警队长
6. 底部进度条显示执行流程

**指令管理：**
- 指令可撤回/修改（在责任人确认前）
- 紧急程度影响响应时间要求

### 第5幕：监控效果 + 双向通信

**实时监控：**
- 地图颜色实时变化（红→橙→黄→绿）
- 顶部摘要条指数实时下降
- 策略栏显示每个策略的执行状态和效果数据

**现场反馈（底部通信面板）：**
- 文字消息：「已到达路口A，开始引导分流」
- 图片反馈：现场照片
- 视频通话请求：点击接听，画面显示在视频面板
- 异常反馈：「S376省道K12路段有事故，建议调整分流路线」

**AI持续分析：**
- 效果符合预期：「S376分流已生效，预计16:15恢复通畅」
- 效果不佳：「缓解速度低于预期，建议追加应急车道借用」
- 新情况提示：「港口通知15:30有大船靠港，预计增加500辆车」

**事件升级：**
- 如果拥堵持续恶化且策略无效，提示升级到应急模式

### 第6幕：策略退出 + 指挥报告

**策略退出流程（拥堵缓解后）：**
- AI提示：「拥堵指数已降至3.0，建议按顺序退出策略」
- 退出顺序：先退应急车道 → 再退分流 → 最后恢复信号灯
- 每步退出生成指令单，通知责任人恢复

**指挥报告（退出指挥模式时自动生成）：**
- 事件概况：时间、地点、最高拥堵指数、持续时长
- 策略执行：使用了哪些策略、执行时间、效果数据
- 通信记录：指令下达和现场反馈的完整时间线
- 经验总结：AI生成的改进建议

**指挥日志：**
- 整个过程的操作记录自动保存，用于事后复盘

### 布局说明

**顶部：指挥摘要条（48px）**
- 拥堵概况 + 主因 + 推荐策略 + 返回总览
- 背景色随拥堵等级变化，重度拥堵有呼吸闪烁

**中部左侧：地图主屏（flex:1，约70%面积）**
- 高德地图 + 拥堵着色 + 设备标注
- 点击拥堵路段聚焦放大
- 视频画中画（右下角，可切换/放大）
- 无人机飞行动画 + 俯拍画面

**中部右侧：策略栏（300px）**
- AI推荐方案列表（含执行按钮）
- 当前可调度资源状态（交警人数、无人机、拖车）
- 历史参考（上次类似拥堵的策略和效果）
- AI持续分析提示

**底部：指挥通信面板（120px）**
- 左侧：策略执行进度条（每个策略一行，流程节点+时间戳）
- 右侧：现场反馈消息流（文字、图片、视频通话请求）
- 同时也是指挥日志（自动记录所有操作和反馈）

## 四、策略执行进度条设计

### 不同策略类型的流程节点

**🟢 直接执行类（诱导屏发布）：**
```
内容编辑 → 确认发布 → 已生效
```

**🟡 需确认类（S376分流、信号灯优化）：**
```
指令生成 → 指令下达 → 责任人确认 → 现场执行 → 效果验证
```

**🔴 需审批类（应急车道借用）：**
```
指令生成 → 提交审批 → 领导审批 → 指令下达 → 现场执行 → 效果验证
```

### 节点状态

- ● 绿色实心 = 已完成（显示完成时间）
- ◉ 青色脉冲 = 进行中
- ○ 灰色空心 = 待执行
- ● 红色 = 异常（如责任人超时未确认）

### 策略退出进度

拥堵缓解后，退出策略也有进度条：
```
退出指令 → 现场恢复 → 确认完成
```

## 五、AI 智能能力

### 归因分析
- 基于港口排队、车流量、天气、施工等数据自动判断拥堵原因
- 显示置信度（如：港口积压70%、节假日高峰20%）

### 策略推荐
- 根据归因结果匹配策略
- 冲突检测：自动排除不可用策略（如施工路段不能分流）
- 资源检查：提示资源不足的策略
- 组合预测：多策略同时执行的效果预测

### 持续监控
- 实时评估策略效果
- 效果不佳时主动建议追加策略
- 新情况出现时及时提示（如港口大船靠港）

### 指挥报告
- 退出时自动生成报告
- 包含：事件概况、策略执行、通信记录、经验总结

## 七、与 V3.0 设计的核心差异

| 维度 | V3.0（6步Tab） | V4.0（指挥中枢） |
|------|---------------|-----------------|
| 核心交互 | 点击Tab切换内容 | 地图点击+策略执行+通信反馈 |
| 信息架构 | 6步顺序分析 | 看→选→做→看 循环 |
| 地图角色 | 辅助展示 | 主操作界面（70%面积） |
| 策略执行 | 点击按钮→状态变化 | 生成指令单→推送→现场反馈 |
| 通信能力 | 无 | 双向通信（指令+反馈） |
| 底部面板 | 拥堵趋势时间轴 | 策略执行进度+通信消息流 |
| 无人机 | 总览模式的巡航 | 指挥模式的定点侦察 |
